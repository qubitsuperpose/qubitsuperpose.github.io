2. 백엔드 및 타겟

Qiskit은 양자 장치에 대한 정적 및 동적 정보를 모두 얻을 수 있는 API를 제공합니다. 우리는 Backend 인스턴스를 사용하여 장치와 인터페이스하며, 여기에는 Target 인스턴스가 포함됩니다. Target은 명령어 세트 아키텍처(ISA) 및 이와 연관된 속성이나 제약 조건과 같은 관련 기능을 요약하는 추상 기계 모델입니다.
이러한 백엔드 인스턴스를 사용하여 IBM Quantum® 플랫폼의 [Compute resources](https://quantum.cloud.ibm.com/computers) 페이지에서 볼 수 있는 일부 정보를 얻어 보겠습니다. 먼저, 관심 있는 장치에 대한 백엔드 인스턴스를 생성합니다. 다음에서는 "ibm_kyoto", "ibm_kawasaki" 또는 가장 한가한 Eagle 머신을 선택합니다. QPU에 대한 접근 권한은 다를 수 있으므로, 이에 따라 백엔드 이름을 업데이트하십시오.

```python
service = QiskitRuntimeService()
# backend = service.backend("ibm_kawasaki") # an Eagle, if you have access to ibm_kawasaki
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=127
)  # Eagle
backend.name
```

먼저 기기에 대한 몇 가지 기본적인 (정적) 정보부터 살펴보겠습니다.

```python
print(
    f"""
{backend.name}, {backend.num_qubits} qubits
processor type = {backend.processor_type}
basis gates = {backend.basis_gates}
"""
)
```

```
ibm_strasbourg, 127 qubits
processor type = {'family': 'Eagle', 'revision': 3} 
basis gates = ['ecr', 'id', 'rz', 'sx', 'x']
```

## 3. 큐비트의 특성

이글 장치는 127개의 큐비트를 가지고 있습니다. 그중 일부 큐비트의 특성을 알아보겠습니다.

```python
for qn in range(backend.num_qubits):
```

### 3.2 게이트 및 판독 오류

이제 게이트 오류에 대해 살펴보겠습니다. 먼저 대상 인스턴스의 데이터 구조를 분석해 보겠습니다. 대상 인스턴스는 연산 이름을 키로 하는 딕셔너리입니다.

```python
target = backend.target
target.keys()
```

```
dict_keys(['measure', 'id', 'sx', 'delay', 'x', 'for_loop', 'rz', 'if_else', 'ecr', 'reset', 'switch_case'])
```

그 값들 또한 딕셔너리입니다. 'sx' 연산에 대한 값(딕셔너리)의 몇 가지 항목을 살펴보겠습니다.

```python
for i, qq in enumerate(target["sx"]):

        break

```

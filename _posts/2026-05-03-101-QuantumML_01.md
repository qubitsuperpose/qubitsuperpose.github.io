# 1. 양자 머신러닝 입문 (Introduction to Quantum Machine Learning)

## 1. 개요 및 동기 (Overview and motivation)
양자 머신러닝에 오신 것을 환영합니다! 

*   우리는 어떤 문제가 양자 컴퓨터에서 처음으로 해결된 후, 뒤이어 사람들이 고전 슈퍼컴퓨터에서 이를 해결하는 방법을 찾아내는 과정을 보아왔습니다. 고전 컴퓨팅과 양자 컴퓨팅이 서로를 한계까지 밀어붙이는 이러한 순환은 향후 몇 년 동안 지속될 것입니다.
*   오류 감소 및 사용 가능한 큐비트 수와 같은 분야에서 진전이 이루어진다면, 양자 컴퓨팅이 고전 컴퓨팅보다 증명 가능한 우위를 점할 수 있는 특정 문제들이 존재합니다. 하지만 지금은 여전히 양자에 적합한 데이터 세트와 유용한 양자 특징 맵(feature maps)을 찾는 탐색의 시기입니다.
*   양자 머신러닝(QML)은 양자 컴퓨팅이 기존의 고전적 워크플로우를 증강하거나 보완할 수 있는 흥미로운 분야 중 하나입니다.

머신러닝(ML)은 데이터 세트에 알고리즘을 적용하는 것이며, 따라서 QML은 데이터나 알고리즘 측면, 혹은 두 측면 모두에 양자 역학을 포함할 수 있습니다. 이러한 모든 가능성은 잠재적으로 흥미롭지만, 우리는 주로 고전 데이터에 적용되는 양자 알고리즘에 대한 논의로 범위를 제한할 것입니다. 그 이유 중 하나는 고전 데이터를 사용한 ML 문제가 이미 매우 잘 연구되어 있고 널리 이용 가능하기 때문입니다. 고전 데이터에서 시작하는 문제를 해결하는 것에는 광범위한 관심이 쏠려 있습니다. 또 다른 이유는 QRAM의 부재입니다. 비교적 긴 시간 동안 대량의 양자 데이터를 저장할 수 있는 능력이 없다면, 양자 데이터에서 시작하는 방법은 산업에 적용하기까지 아직 상당히 멀었습니다. 또한 고전 데이터에 효율적인 방식으로 "양자적으로 접근"하는 방법도 불분명합니다. 특히 관심 있는 두 가지 유형의 ML은 라벨이 지정된 데이터 세트를 사용하여 알고리즘을 훈련하는 지도 학습(supervised learning)과 알고리즘이 라벨이 없는 샘플에서 분포에 대해 학습하려고 시도하는 비지도 학습(unsupervised learning)입니다. 예를 들어, 비지도 알고리즘은 동일한 분포에서 새로운 샘플을 생성하는 방법이나 유사한 특성을 가진 그룹으로 샘플을 클러스터링하는 방법을 배울 수 있습니다.

![](https://quantum.cloud.ibm.com/assets-docs-learning/_next/image?url=%2Flearning%2Fimages%2Fcourses%2Fquantum-machine-learning%2Fintroduction%2Fqml-cr-background-sup-unsup.avif&w=3840&q=75)

왼쪽 이미지는 지도 학습에서처럼 레이블이 지정된 두 가지 범주의 데이터를 보여줍니다. 이 경우 범주는 선형적으로 분리 가능합니다. 오른쪽 이미지는 데이터 클러스터를 보여줍니다. 비지도 학습 작업에서는 이러한 데이터에 처음에는 레이블이 지정되지 않으며, 알고리즘은 분포를 학습하여 클러스터를 찾습니다. 알고리즘이 식별할 수 있는 예시 클러스터를 시각화하기 위해 데이터 포인트에 레이블을 지정했습니다. 두 학습 방식의 핵심적인 차이점은 지도 학습은 이미 레이블이 지정된 데이터로 시작하는 반면, 비지도 학습은 최종적으로 레이블이 지정되더라도 레이블이 지정되지 않은 데이터로 시작한다는 점입니다.

머신러닝에 대한 배경지식이 있는 사람들은 이미 많은 해결 방법이 데이터를 고차원 공간으로 매핑하는 것을 포함한다는 것을 알고 있을 것입니다. 특히 커널(kernel) 분야에서 이러한 매핑 기법이 많이 활용됩니다. 간단히 다시 설명하자면, 데이터는 주어진 차원과 동일한 차원 내에서 선, 평면 또는 초평면(간결성을 위해 "초평면"으로 줄여서 부르겠습니다)으로 범주별로 분리될 수 있습니다. 이는 위의 첫 번째 이미지에서 확인할 수 있습니다. 반대로, 데이터가 주어진 차원에서 초평면으로 분리되지 않는 경우도 있습니다. 두 번째 이미지가 그 예입니다. 하지만 데이터에는 여전히 구조가 존재하며, 이러한 구조를 활용하여 고차원 공간으로 매핑하면 해당 고차원 공간에서 데이터를 분리할 수 있습니다. 이는 원형 대칭을 갖는 2차원 데이터를 포물면을 따라 배열된 데이터 포인트가 있는 3차원 공간으로 매핑하는 예시에서 확인할 수 있습니다.

![](https://quantum.cloud.ibm.com/assets-docs-learning/_next/image?url=%2Flearning%2Fimages%2Fcourses%2Fquantum-machine-learning%2Fintroduction%2Fqml-cr-background-2d-3d.avif&w=3840&q=75)

양자 머신러닝(QML)의 일반적인 목표는 저차원 특징 집합에서 고차원 공간으로의 매핑을 찾아 데이터 포인트를 효과적으로 분리하고, 이를 사용하여 새로운 데이터 포인트를 분류하는 것입니다.
하지만 이는 쉬운 일이 아니며, 머신러닝에서 양자 컴퓨팅의 잠재적 유용성에 대한 논의에는 적절한 주의 사항이 수반되어야 합니다. 특히 데이터셋 선택의 미묘한 차이와 유틸리티 규모 달성의 어려움을 다뤄야 합니다. 또한 기존 알고리즘이 이미 효율적이고 잘 처리하는 데이터에서 기존 머신러닝 알고리즘보다 뛰어난 성능을 내는 데 집중하기보다는, 유용할 수 있는 새로운 특징 맵을 탐구하는 데 초점을 맞춰야 합니다.


## 2. 기대치 관리 (Managing expectations)

문헌에 설명된 QML 애플리케이션에 사용되는 많은 데이터 세트는 "특징 공학적(feature engineered)"입니다. 즉, 양자 컴퓨팅이 유용한 좁은 사용 사례를 보여주기 위해 특정 데이터 세트가 선택되거나 생성된다는 의미입니다. 이것이 속임수처럼 느껴진다면 우리는 당면한 과제를 오해하고 있는 것입니다. 일부 양자 특징 맵이 고전 ML 알고리즘보다 모든 또는 많은 분류 작업을 더 효율적이고 확장 가능하게 해결할 수 있게 해주는 것은 **아닙니다**. 오히려 일부 양자 특징 맵(전부는 아님)은 고전적인 특징 맵과 다르게 작동합니다. 따라서 당면한 과제는 복잡한 데이터 구조의 맥락에서 양자 회로를 탐구하는 것입니다. 해결해야 할 구체적인 질문은 다음과 같습니다.

1.  고전적인 대안과 비교하여 어떤 양자 회로가 참신한 방식으로 작동할 가능성이 가장 높은가?
2.  이러한 참신한 양자 회로를 사용하여 가장 잘 탐구할 수 있는 특성을 가진 데이터를 포함하는 실제 문제가 존재하는가?
3.  이러한 양자 회로가 근미래 양자 컴퓨터에서 확장 가능한가?

### 2.1 불충분한 설명 (Insufficient explanation)
양자 컴퓨팅의 강력함을 설명하는 단순화된 표현을 흔히 접하게 됩니다. 그 표현은 대략 다음과 같습니다.

고전 컴퓨터가 정보 비트를 사용하는 것처럼 양자 컴퓨터는 큐비트를 사용합니다. 예를 들어 4비트를 가진 고전 컴퓨터는 2⁴ = 16개의 가능한 상태 중 하나만을 가질 수 있지만, 양자 컴퓨터는 16개의 상태가 모두 중첩된 상태로 동시에 존재할 수 있으며, 이 중첩 상태 전체에 대해 연산을 수행할 수 있습니다. 어떤 경우에는 이러한 특성 덕분에 고차원 공간으로의 매핑을 기반으로 하는 흥미로운 학습 알고리즘을 설계할 수 있습니다.

이 설명은 맞지만, 불충분하고 다소 오해의 소지가 있습니다. 이에 대해서는 아래에서 자세히 설명하겠습니다. 또한 복소수 계수와 실수 계수의 차이점을 강조하는 표현도 있습니다.

시스템이 서로 다른 상태에 있을 확률이 정해져 있는 확률적 고전 시스템은 다음과 같이 설명할 수 있습니다.
<p>
$$
|s\rangle = a|0000\rangle+b|0001\rangle+c|0010\rangle+... a, b, c \in \mathbb{R}
$$
</p>

이러한 시스템에서 계수 $a$, $b$, $c$ 등은 양의 실수일 때만 의미를 갖습니다. 양자 컴퓨터의 상태는 복소수일 수 있는 확률 진폭으로 기술됩니다.
<p>
$$
|\psi \rangle = A|0000\rangle+B|0001\rangle+C|0010\rangle+... A, B, C \in \mathbb{C}
$$
</p>

위의 진술들은 매우 신중하게 작성되었으므로 모두 참입니다(겉보기에 유사해 보이는 많은 진술들이 사실은 틀렸습니다). 하지만 이러한 정확한 진술들이 양자 컴퓨팅이 머신러닝에서 발휘하는 강력한 성능을 설명하는 것은 아닙니다. 우선, 양자 컴퓨팅을 머신러닝에 적용하려면 측정이 수반되는데, 큐비트가 동시에 여러 상태에 있는 것을 측정할 수는 없습니다. 큐비트를 $\|\psi\rangle = \frac{1}{\sqrt{2}}\left(\|0\rangle+\|1\rangle\right)$와 같은 중첩 상태로 준비할 수는 있지만, 측정 결과는 $\|0\rangle$ 또는 $\|1\rangle$ 중 하나일 뿐입니다. 따라서 최소한 차원 증가에 대한 설명은 불완전합니다. 더욱이, 커널의 맥락에서 볼 때, 가우스 커널은 무한 차원이기 때문에 양자 컴퓨팅에서 차원이 증가한다는 것만으로는 고전적 대안보다 연산 능력이 우수하다는 충분조건이 될 수 없습니다. 여기에는 미묘한 차이가 있는데, 가우시안 특징 맵은 무한 차원의 매핑 벡터를 계산할 필요성을 없애는 "커널 트릭"과 함께 사용될 때만 유효하다는 점입니다. 하지만 요점은 같습니다.


**얽힌 양자 상태의 고차원성은 지수적 병렬성이 아니며, 머신러닝에서 향상된 성능을 위한 충분조건도 아닙니다.**

이어지는 강의에서 우리는 양자 컴퓨팅의 힘을 탐구하는 것을 돕기 위해 머신러닝 작업에 양자 회로를 통합하는 워크플로우를 제시합니다. 이 과정의 어떤 특징 맵이나 알고리즘도 일반적인 문제에 대해 더 나은 머신러닝 결과를 얻기 위한 빠른 지름길로 제시되지 않습니다. 왜냐하면 그러한 것은 존재하지 않기 때문입니다. 대신 유용한 양자 컴퓨팅을 탐구하는 데 사용할 수 있는 다양한 양자 도구를 제시합니다.

### 2.2 역양자화 (Dequantization)
역양자화(Dequantization)란 주어진 양자 알고리즘을, 일반적으로 확장성을 포함하여, 주어진 작업 집합에 대해 양자 알고리즘과 유사한 성능을 보이는 고전 알고리즘으로 대체하는 것을 의미합니다. 일부 정의에 따르면, 고전 알고리즘은 양자 알고리즘보다 다항식적으로만 느려야 합니다.

최근 몇 년 동안, 초기에는 고전 알고리즘보다 훨씬 빠른 속도를 제공할 것으로 예상되었던 여러 양자 머신러닝(QML) 알고리즘들이 역양자화되었습니다. 이러한 역양자화 과정은 머신러닝에 대한 양자적 접근 방식의 잠재적 이점과 한계에 대한 중요한 통찰력을 제공했습니다.

가장 주목할 만한 역양자화 결과 중 하나는 [Ewin Tang의 추천 시스템 연구](https://arxiv.org/abs/1807.04271)에서 나왔습니다. 탕은 양자 컴퓨터에서만 가능하다고 여겨졌던 속도로 추천 작업을 수행할 수 있는 고전 알고리즘을 발견했습니다. 이 발견은 양자 알고리즘이 이 문제에서 기하급수적인 이점을 가진다는 가정에 이의를 제기했습니다. [Shin et al.](https://journals.aps.org/prresearch/abstract/10.1103/PhysRevResearch.6.023218)의 최근 연구는 변분 양자 기계 학습 모델의 함수 클래스의 비양자화 가능성에 대한 조건을 식별하는 데 중점을 두었습니다.

역양자화에 대한 일반적인 접근 방식 중 하나(물론 유일한 방법은 아니지만)는 데이터 로딩 오버헤드를 고려하는 것입니다. 즉, 고전 데이터에 적용되는 모든 양자 알고리즘은 고전 데이터를 양자 컴퓨터에 인코딩하는 단계를 거칩니다. 양자 알고리즘이 양자 데이터가 이미 존재하는 시작점을 가정한다면, 인코딩에 필요한 시간을 효과적으로 숨길 수 있습니다. 양자 데이터를 가정하는 것이 합리적인 상황도 있지만, 많은 응용 분야에서는 고전 데이터로 시작합니다. 일부 역양자화 사례에서는 인코딩 시간을 포함하고 고전 데이터 로딩을 효율적으로 수행할 수 있는 경우, 양자 알고리즘이 더 이상 고전 알고리즘보다 성능이 뛰어나지 않다는 것을 보여주었습니다.

알고리즘을 역양자화할 수 없다고 해서 모든 고전 알고리즘보다 더 효율적이거나 확장성이 뛰어나다는 의미는 아닙니다. 극단적인 예를 들어보겠습니다. 크기가 k인 집합에서 가장 큰 j개의 요소를 선택하는 알고리즘을 생각해 보세요. 쇼어의 알고리즘을 사용하여 k개의 각 요소를 소인수분해한 다음, 소인수를 이용하여 가장 큰 요소를 찾는 양자 알고리즘을 작성할 수 있습니다. 이러한 알고리즘은 역양자화가 불가능할 가능성이 높지만, 동일한 방식으로 가장 큰 요소를 선택하는 데 있어 기존 알고리즘보다 효율성이 현저히 떨어집니다(불필요한 인수분해 부분은 제외).


## 3. 존재 증명 (Existence proof)
2021년, IBM Quantum® 연구원인 Yunchao Liu, Srinivasan Arunachalam, 그리고 Kristan Temme는 Nature지에 [지도 학습에서의 엄격하고 견고한 양자 속도 향상](https://www.nature.com/articles/s41567-021-01287-z)이라는 논문을 발표했습니다. 앞서 언급한 주의사항을 고려하여, 본 연구에서는 (1) 고전적으로 어려운 문제로 알려져 있고, (2) 양자 알고리즘이 속도 향상을 보여주기에 적합한 분류 문제를 신중하게 선택했습니다.

이 논문은 이산 로그를 기반으로 한 데이터 분류 문제를 다룹니다. 
논문을 인용하자면, 큰 소수 p와 $$\mathbb{Z}^*_p = {1, 2, . . . , p − 1}$$의 생성자 $g$에 대해, 어떤 고전 알고리즘도 입력 $x \in \mathbb{Z}^*_p$에 대한 $\text{log}_g(x)$를 $p$를 표현하는 데 필요한 비트 수 $n = \lceil{\text{log}_2(p)}\rceil$에 대한 다항식 시간 내에 계산할 수 없다는 것이 널리 받아들여지는 추측이다. 이와 대조적으로, [쇼어의 알고리즘](https://epubs.siam.org/doi/10.1137/S0097539795293172)은 이산 로그 문제를 다항식 시간 내에 해결하는 것으로 알려져 있다. 따라서 이러한 문제 선택은 위의 기준, 즉 고전적 난이도(역양자화가 어려울 가능성이 높음)와 양자 알고리즘에 적합한 것으로 알려진 조건을 동시에 충족합니다.

저자들은 분류 문제를 신중하게 선택함으로써 양자 커널 방법(아래에서 간략히 설명하고 이후 강의에서 자세히 다룰 예정)을 사용하여 종단 간(end-to-end) 및 강건성(robust) 측면에서 지수적인 속도 향상을 보여줄 수 있었습니다. 여기서 "종단 간"이란 고전적 데이터에서 시작한다는 가정을 의미하며, 저자들은 이 경우 데이터 인코딩 시간을 포함합니다. "강건성"이란 양자 알고리즘을 사용하여 분류 대상 데이터를 넓은 간격으로 분리함으로써 유한 샘플링 오류와 같은 실제 환경 요인에 대해 분류 성공률이 강건하다는 것을 의미합니다.

요컨대, 양자 커널을 통해 지수적인 속도 향상을 얻을 수 있는 문제가 존재합니다. 하지만 현재 과학 수준에서는 이러한 문제는 양자 알고리즘에 적합할 것이라는 관찰 또는 이론적 근거에 기반하여 선택됩니다. 고전 컴퓨터가 이미 상당히 잘 수행하는 머신러닝 작업에서 양자 속도 향상을 기대하는 것은 현실적이지 않습니다.

이러한 양자 활용 가능성을 탐구할 이상적인 사례를 찾아내는 것은 이 강의를 수강하는 학습자들에게 막중한 책임입니다. 그리고 이는 본 강좌에서 다룰 수 있는 과제가 아닙니다. 그러한 탐구는 여러분과 같은 연구자들로 구성된 IBM 양자 네트워크 전체의 몫입니다. 본 강좌에서는 여러분이 전문 분야에서 양자 컴퓨팅의 활용 가능성을 탐색할 수 있도록 QML 워크플로우와 인코딩 전략을 시연할 것입니다.

이 소개를 통해 양자 머신러닝에 대해 다음과 같은 몇 가지 사항이 명확해졌기를 바랍니다.

1. 양자 알고리즘은 고전적으로 어렵고 양자 알고리즘에 매우 적합한 특정 문제에 대해 고전 알고리즘보다 기하급수적인 속도 향상을 제공할 수 있습니다.

2. 양자 컴퓨팅에서 얽힌 상태의 고차원성은 중요하지만, 고전 알고리즘보다 우위를 점하기 위한 충분한 요인은 아닙니다.

3. 양자 알고리즘에 적합한 문제를 찾는 것은 매우 어려운 과제이며, 본 강좌의 학습자들이 상당 부분 담당하게 될 것입니다.

본격적으로 코드를 구성 요소별로 나누고 배경 지식을 탐구하기 전에, 전체적인 방향을 파악하는 것이 도움이 될 수 있습니다. 아래 코드 셀은 양자 커널 메서드의 간단한 예시를 보여줍니다. 구체적으로, 하나의 커널 행렬 요소를 계산합니다. 커널 메서드나 양자 커널에 익숙하지 않은 사용자도 이 코드에 겁먹을 필요는 없습니다. 이 과정의 여러 강의에서 이러한 코드 셀에서 정확히 어떤 작업이 수행되는지 자세히 분석할 것입니다.

이 코드를 통해 우리는 유틸리티 규모의 양자 컴퓨팅 접근 방식인 Qiskit 패턴을 동시에 소개합니다. 이 프레임워크는 매우 일반적이며 대부분의 문제에 적용할 수 있는 네 단계로 구성됩니다(단, 일부 작업 흐름에서는 특정 단계를 여러 번 반복해야 할 수도 있습니다).

### 7.1 Qiskit 패턴:

* 1단계: 고전적 입력을 양자 문제로 변환
* 2단계: 양자 실행에 맞게 문제 최적화
* 3단계: Qiskit 런타임 프리미티브를 사용하여 실행
* 4단계: 분석/후처리

아래 셀에서는 각 단계에 대한 간략한 설명만 제공하며, 더 자세한 내용을 학습하려면 해당 강의를 참조.

```python
# Import some qiskit packages required for setting up our quantum circuits.
from qiskit.circuit import Parameter, ParameterVector, QuantumCircuit
from qiskit.circuit.library import unitary_overlap

# Import StatevectorSampler as our sampler.
from qiskit.primitives import StatevectorSampler

# Step 1: Map classical inputs to a quantum problem:

# Start by getting some appropriate data.
# The data imported below consist of 128 rows or data points.
# Each row has 14 columns that correspond to data features, and a 15th column with a label (+/-1).
!wget https://raw.githubusercontent.com/qiskit-community/prototype-quantum-kernel-training/main/data/dataset_graph7.csv

# Import some required packages, and write a function to pull some
# training data out of the csv file you got above.
import pandas as pd
import numpy as np


def get_training_data():
    """Read the training data."""
    df = pd.read_csv("dataset_graph7.csv", sep=",", header=None)
    training_data = df.values[:20, :]
    ind = np.argsort(training_data[:, -1])
    X_train = training_data[ind][:, :-1]

    return X_train


# Prepare training data
X_train = get_training_data()

# Empty kernel matrix
num_samples = np.shape(X_train)[0]

# Prepare feature map for computing overlap between two data points.
# This could be pre-built feature maps like ZZFeatureMap, or a custom quantum circuit,
# as shown here.
num_features = np.shape(X_train)[1]
num_qubits = int(num_features / 2)
entangler_map = [[0, 2], [3, 4], [2, 5], [1, 4], [2, 3], [4, 6]]
fm = QuantumCircuit(num_qubits)
training_param = Parameter("θ")
feature_params = ParameterVector("x", num_qubits * 2)
fm.ry(training_param, fm.qubits)
for cz in entangler_map:
    fm.cz(cz[0], cz[1])
for i in range(num_qubits):
    fm.rz(-2 * feature_params[2 * i + 1], i)
    fm.rx(-2 * feature_params[2 * i], i)

# Pick two data points, here 14 and 19, and assign the features to the circuits as parameters.
x1 = 14
x2 = 19
unitary1 = fm.assign_parameters(list(X_train[x1]) + [np.pi / 2])
unitary2 = fm.assign_parameters(list(X_train[x2]) + [np.pi / 2])

# Create the overlap circuit
overlap_circ = unitary_overlap(unitary1, unitary2)
overlap_circ.measure_all()
overlap_circ.draw("mpl", scale=0.6, style="iqp")

# Step 2: Optimize problem for quantum execution

# Use Qiskit Runtime service to get the least busy backend for running on real quantum computers.

# overlap_ibm = pm.run(overlap_circ)

# Step 3: Execute using Qiskit Runtime Primitives

# Specify the number of shots to use.
num_shots = 10_000

## Evaluate the problem using statevector-based primitives from Qiskit
sampler = StatevectorSampler()
counts = (
    sampler.run([overlap_circ], shots=num_shots).result()[0].data.meas.get_int_counts()
)

# Step 4: Analyze and post-processing

# Find the probability of 0.
counts.get(0, 0.0) / num_shots
```

# 5. SQD for energy estimation of a chemistry Hamiltonian

이번 수업에서는 SQD를 적용하여 분자의 기저 상태 에너지를 추정할 것입니다.

특히, 4단계 Qiskit 패턴 접근법을 사용하여 다음 주제들을 다룰 것입니다:

1. 1단계: 문제를 양자 회로와 연산자에 매핑
   * $N_2$ 분자의 해밀토니안을 설정합니다.
   * 화학에서 영감을 받고 하드웨어 친화적인 국부 단위 클러스터 야스트로(LUCJ) $\[1\]$(#references)를 설명합니다.
2. 2단계: 목표 하드웨어에 최적화
   * 하드웨어 실행을 위해 ansatz의 게이트 수와 레이아웃을 최적화합니다.
3. 3단계: 목표 하드웨어에서 실행
   * 최적화된 회로를 실제 QPU에서 실행하여 부분공간의 샘플을 생성합니다.
4. 4단계: 결과 후처리
   * 자기일관적 구성 복구 루프 $\[2\]$(#references)를 소개합니다.
     * 전체 비트열 샘플 집합을 후처리하며, 이전 반복에서 계산된 입자 수와 평균 궤도 점유에 대한 사전 지식을 사용합니다.
     * 복구된 비트열로부터 확률적으로 부분 샘플 배치를 생성합니다.
     * 각 샘플링된 부분공간에 대해 분자 해밀토니안을 투영하고 대각화합니다.
     * 모든 배치에서 찾은 최소 기저 상태 에너지를 저장하고 평균 궤도 점유를 업데이트합니다.

수업 전반에 걸쳐 여러 소프트웨어 패키지를 사용할 것입니다.

* 분자를 정의하고 해밀토니안을 설정하기 위해 `PySCF`
* LUCJ ansatz를 구성하기 위해 `ffsim` 패키지
* 하드웨어 실행을 위해 ansatz를 트랜스파일링하기 위해 `Qiskit`
* QPU에서 회로를 실행하고 샘플을 수집하기 위해 `Qiskit IBM Runtime`
* 부분공간 투영과 행렬 대각화를 사용한 구성 복구와 기저 상태 에너지 추정을 위해 `Qiskit addon SQD`

## 1. 문제를 양자 회로 및 연산자로 매핑합니다.


### 분자 Hamiltonian

분자 해밀토니안은 다음과 같은 일반적인 형태를 가진다:

<p>
$$
\hat{H} = \sum_{ \substack{pr\\\sigma} } h_{pr} \, \hat{a}^\dagger_{p\sigma} \hat{a}_{r\sigma}
+
\sum_{ \substack{prqs\\\sigma\tau} }
\frac{(pr|qs)}{2} \,
\hat{a}^\dagger_{p\sigma}
\hat{a}^\dagger_{q\tau}
\hat{a}_{s\tau}
\hat{a}_{r\sigma}
$$
</p>


$$\hat{a}^\dagger_{p\sigma}/\hat{a}_{p\sigma}$$
는 $p$번째 기저함수 요소와 스핀 $\sigma$에 결부된 페르미온 생성/소멸 연산자이다. $h_{pr}$과 $(pr\|qs)$는 각각 1체 및 2체 전자 적분이다. pySCF를 사용하여, 분자를 정의하고 기저함수 `6-31g`에 대한 해밀토니안의 1체 및 2체 적분을 계산할 것이다.

```python
import warnings
import pyscf
import pyscf.cc
import pyscf.mcscf

warnings.filterwarnings("ignore")

# Specify molecule properties
open_shell = False
spin_sq = 0

# Build N2 molecule
mol = pyscf.gto.Mole()
mol.build(
    atom=[["N", (0, 0, 0)], ["N", (1.0, 0, 0)]],  # Two N atoms 1 angstrom apart
    basis="6-31g",
    symmetry="Dooh",
)

# Define active space
n_frozen = 2
active_space = range(n_frozen, mol.nao_nr())

# Get molecular integrals
scf = pyscf.scf.RHF(mol).run()
num_orbitals = len(active_space)
n_electrons = int(sum(scf.mo_occ[active_space]))
num_elec_a = (n_electrons + mol.spin) // 2
num_elec_b = (n_electrons - mol.spin) // 2
cas = pyscf.mcscf.CASCI(scf, num_orbitals, (num_elec_a, num_elec_b))
mo = cas.sort_mo(active_space, base=0)
hcore, nuclear_repulsion_energy = cas.get_h1cas(mo)  # hcore: one-body integrals
eri = pyscf.ao2mo.restore(1, cas.get_h2cas(mo), num_orbitals)  # eri: two-body integrals

# Compute exact energy for comparison
exact_energy = cas.run().e_tot
```

```
converged SCF energy = -108.835236570774
CASCI E = -109.046671778080  E(CI) = -32.8155692383188  S^2 = 0.0000000
```

이번 수업에서는 페르미온 파동 함수를 양자 회로에서 사용할 수 있도록 큐비트 파동 함수로 변환하기 위해 조던-위그너(JW) 변환을 사용합니다. JW 변환은 M개의 공간 오비탈에 있는 페르미온의 포크 공간을 2M개의 큐비트로 이루어진 힐베르트 공간으로 변환합니다. 즉, 하나의 공간 오비탈은 스핀 업("α") 전자와 스핀 다운("β") 전자에 해당하는 두 개의 스핀 오비탈로 나뉩니다. 스핀 오비탈은 전자가 점유되거나 비어 있을 수 있습니다. 일반적으로 오비탈의 개수를 언급할 때는 공간 오비탈의 개수를 의미합니다. 스핀 오비탈의 개수는 공간 오비탈의 개수의 두 배가 됩니다. 양자 회로에서는 각 스핀 오비탈을 하나의 큐비트로 나타냅니다. 따라서 한 세트의 큐비트는 스핀 업("α") 오비탈을 나타내고, 다른 세트의 큐비트는 스핀 다운("β") 오비탈을 나타냅니다. 예를 들어, `6-31g` 기저 함수 세트를 사용하는 N₂ 분자는 16개의 공간 오비탈(즉, 16개의 α 오비탈 + 16개의 β 오비탈 = 32개의 스핀 오비탈)을 가집니다. 따라서 32개의 큐비트로 구성된 양자 회로가 필요합니다(나중에 설명하겠지만, 추가적인 보조 큐비트가 필요할 수도 있습니다). 큐비트는 계산 기저 함수로 측정되어 전자 배치 또는 (슬레이터) 행렬식을 나타내는 비트열을 생성합니다. 이 강의 전체에서 비트열, 배치, 행렬식이라는 용어를 상호 교환적으로 사용합니다. 비트열은 스핀 오비탈의 전자 점유 상태를 알려줍니다. 비트 위치에 1이 있으면 해당 스핀 오비탈이 점유되어 있음을 의미하고, 0이 있으면 스핀 오비탈이 비어 있음을 의미합니다. 전자 구조 문제는 입자 보존 법칙을 따르므로, 고정된 수의 스핀 오비탈만 점유되어야 합니다. $N_2$ 분자는 스핀업($\alpha$) 전자 5개와 스핀다운($\beta$) 전자 5개를 가지고 있습니다. 따라서 $\alpha$ 및 $\beta$ 오비탈을 나타내는 비트열은 $N_2$ 분자의 경우 각각 5개의 $1\text{s}$를 포함해야 합니다.

### 1.1 샘플 생성을 위한 양자 회로: LUCJ 안자츠

이 레슨에서는 양자 상태 준비와 이후 샘플링을 위해 국소 유니터리 결합 클러스터 야스트로프(local unitary coupled cluster Jastrow, LUCJ) $\[1\]$(#references) 안자츠(ansatz)를 사용할 것입니다. 먼저, 전체 UCJ 안자츠의 다양한 구성 요소들과 그것의 국소화 버전에서 이루어지는 근사들을 설명하겠습니다. 다음으로, ffsim 패키지를 사용하여 LUCJ 안자츠를 구성하고, 하드웨어 실행을 위해 Qiskit 트랜스파일러를 이용해 이를 최적화할 것입니다.

UCJ 안자츠는 다음과 같은 형태를 가집니다 (UCJ 연산자의 $L$개 레이어 또는 반복의 곱에 대해).
<p>
$$
|\psi\rangle = \prod_{\mu=1}^{L}{(e^{K^{\mu}} \times {e^{iJ^{\mu}}} \times {e^{-K^{\mu}}})} |\Phi_{0}\rangle
$$
</p>

여기서 $\vert \Phi_{0} \rangle$는 기준 상태(reference state)로, 일반적으로 하트리-폭(Hartree-Fock, HF) 상태로 취합니다. 하트리-폭 상태는 가장 낮은 번호의 궤도들이 점유되어 있는 상태로 정의되므로, HF 상태 준비는 점유된 궤도에 해당하는 큐비트를 1로 설정하기 위해 X 게이트를 적용하는 과정을 포함합니다. 예를 들어, 4개의 공간 궤도와 2개의 업스핀, 2개의 다운스핀에 대한 HF 상태 준비 블록은 다음과 같은 형태일 수 있습니다:

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig1.svg)


UCJ 연산자 ${(e^{K^{(\mu)}} \times {e^{iJ^{(\mu)}}} \times {e^{-K^{(\mu)}}})}$의 단일 반복은 궤도 회전($e^{K^{(\mu)}}$ 및 $e^{-K^{(\mu)}}$) 사이에 끼워진 대각선 쿨롱 진화($e^{iJ^{(\mu)}}$)로 구성됩니다.

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig2.svg)

궤도 회전 블록은 단일 스핀 종류($\alpha$(업스핀)/$\beta$(다운스핀))에 작용합니다. 각 전자 종류에 대해 궤도 회전은 단일 큐비트 $R_{z}$ 게이트 층과 그 뒤에 이어지는 2큐비트 Given's 회전 게이트($XX + YY$ 게이트) 시퀀스로 구성됩니다.

2큐비트 게이트는 인접한 스핀 궤도(최근접 이웃 큐비트)에 작용하므로 SWAP 게이트 없이도 IBM® QPU에서 구현할 수 있습니다.

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig3.svg)


$e^{iJ^{(\mu)}}$는 대각 쿨롱 연산자(diagonal Coulomb operator)라고도 불리며, 세 개의 블록으로 구성된다. 이 중 두 개는 동일한 스핀 섹터에 작용하고 ($e^{iJ_{\alpha \alpha}^{(\mu)}}$ 및 $e^{iJ_{\beta \beta}^{(\mu)}}$), 나머지 하나는 두 스핀 섹터 사이에 작용한다 ($e^{iJ_{\alpha \beta}^{(\mu)}}$).

$e^{iJ^{(\mu)}}$의 모든 블록은 number-number 게이트 $U_{nn}(\phi)$ $\[1\]$(#references)로 구성된다. $U_{nn}(\phi)$ 게이트는 $R_{ZZ}(\frac{\phi}{2})$ 게이트 하나와, 그 뒤에 오는 두 개의 서로 다른 큐비트에 작용하는 단일 큐비트 $Rz(-\frac{\phi}{2})$ 게이트 두 개로 더 세분화될 수 있다.

동일 스핀 성분($J_{\alpha \alpha}$ 및 $J_{\beta \beta}$)은 가능한 모든 큐비트 쌍 사이에 $U_{nn}$ 게이트를 갖는다. 그러나 초전도 QPU는 제한적인 연결성(connectivity)을 가지므로, 인접하지 않은 큐비트 사이의 게이트를 구현하려면 큐비트를 스왑(swap)해야 한다.

예를 들어, $N = 4$개의 공간 궤도(spatial orbitals)에 대한 다음 $e^{iJ_{\alpha \alpha}^{(\mu)}}$ (또는 $e^{iJ_{\beta \beta}^{(\mu)}}$) 블록을 생각해 보자. 선형 큐비트 연결성의 경우, 마지막 세 개의 게이트는 인접하지 않은 큐비트 사이에 작용하기 때문에 직접 구현할 수 없다 (예를 들어, Q0와 Q2는 직접 연결되어 있지 않다). 따라서 이들을 인접하게 만들기 위해 SWAP 게이트가 필요하다 (다음 그림은 $3$개의 SWAP 게이트를 사용하는 예시를 보여준다).

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig4.svg)

다음으로, $J_{\alpha \beta}$는 서로 다른 스핀 영역에 있는 동일한 인덱스의 오비탈 사이의 게이트를 구현합니다(예: $0\alpha$와 $0\beta$ 사이). 마찬가지로, 큐비트가 QPU에서 물리적으로 인접하지 않은 경우 이러한 게이트에도 SWAP이 필요합니다.

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig5.svg)


위에서 논의한 바와 같이, UCJ 안자츠는 인접하지 않은 큐비트 간의 상호작용으로 인해 SWAP 게이트가 필요하기 때문에 하드웨어 실행에 몇 가지 어려움이 있습니다. UCJ 안자츠의 로컬 변형인 LUCJ는 대각 쿨롱 연산자에서 일부 $U_{nn}$ 게이트를 제거함으로써 이러한 문제를 해결합니다.

동일한 전자 종류 블록($J_{\alpha \alpha}$ 및 $J_{\beta \beta}$)에서, LUCJ 버전에서는 최근접 이웃 연결에 적합한 $U_{nn}$ 게이트만 유지하고 인접하지 않은 큐비트 사이의 게이트를 제거합니다. 다음 그림은 인접하지 않은 게이트를 제거한 후의 LUCJ 블록을 보여줍니다.

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig6.svg)

다음으로, 서로 다른 전자 종(species) 사이에서 작동하는 $J_{\alpha \beta}$ 블록의 LUCJ 버전은 디바이스 토폴로지에 따라 다른 형태를 가질 수 있습니다.

여기서도 LUCJ 버전은 호환되지 않는 게이트를 제거합니다. 아래 그림은 그리드(grid), 육각형(hexagonal), 헤비헥스(heavy-hex), 선형(linear)을 포함한 서로 다른 큐비트 토폴로지에 대한 $J_{\alpha \beta}$ 블록의 변형들을 보여줍니다.

* **정사각형(Square)**: 모든 $\alpha$ 및 $\beta$ 오비탈 사이에서 SWAP 없이 $U_{nn}$ 게이트를 가질 수 있으며, 따라서 어떠한 $U_{nn}$ 게이트도 제거할 필요가 없습니다.
* **헤비헥스(Heavy-hex)**: $\alpha$-$\beta$ 상호작용은 매 4번째 인덱스(예: 0번째, 4번째, 8번째)의 스핀 오비탈 사이에서 유지되며, 이는 *보조(ancilla)* 큐비트에 의해 매개됩니다. 즉, $\alpha$ 및 $\beta$ 오비탈을 나타내는 선형 체인 사이에 보조 큐비트가 필요합니다. 이러한 배치는 제한된 수의 SWAP만을 필요로 합니다.
* **육각형(Hexagonal)**: $\alpha$와 $\beta$가 두 개의 인접한 선형 체인으로 배치될 때, 0번째, 2번째, 4번째 인덱스 오비탈과 같이 하나 걸러 하나씩의 오비탈이 최근접 이웃(nearest neighbor)이 됩니다.
* **선형(Linear)**: 오직 하나의 $\alpha$ 오비탈과 하나의 $\beta$ 오비탈만 연결되며, 이는 $J_{\alpha \beta}$ 블록이 단 하나의 게이트만을 가짐을 의미합니다.

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig7.svg)

UCJ 연산자에서 게이트를 제거하여 LUCJ 버전을 구성하면 하드웨어 호환성은 향상되지만 표현력이 다소 떨어집니다. 따라서 LUCJ 연산자를 사용할 때는 수정된 UCJ 연산자($L$)를 더 많이 반복해야 할 수도 있습니다.

### 1.2 LUCJ ansatz 초기화

LUCJ는 매개변수화된 앙상블이므로 하드웨어 실행 전에 매개변수를 초기화해야 합니다. 앙상블을 초기화하는 한 가지 방법은 고전적인 결합 클러스터 단일 및 이중 여기(CCSD) 방법에서 얻은 `t1` 및 `t2` 진폭을 사용하는 것입니다. 여기서 `t1` 진폭은 단일 여기 연산자의 계수이고, `t2` 진폭은 이중 여기 연산자의 계수입니다.

`t1` 및 `t2` 진폭을 사용하여 LUCJ 앙상블을 초기화하면 괜찮은 결과를 얻을 수 있지만, 앙상블 매개변수에 대한 추가 최적화가 필요할 수 있습니다.


```python
# Get CCSD t2 amplitudes for initializing the ansatz
ccsd = pyscf.cc.CCSD(
    scf, frozen=[i for i in range(mol.nao_nr()) if i not in active_space]
)
ccsd.run()

t1 = ccsd.t1
t2 = ccsd.t2
```

```
E(CCSD) = -109.0398256929733  E_corr = -0.20458912219883
```


### 1.3 `ffsim`을 사용하여 LUCJ ansatz 구성하기

위 문서에서 계산한 `t1`과 `t2` 진폭을 사용하여 ansatz를 생성하고 초기화하기 위해 [ffsim](https://github.com/qiskit-community/ffsim/tree/main) 패키지를 사용할 것입니다. 우리의 분자는 닫힌-껍질(closed-shell) Hartree-Fock 상태를 가지고 있으므로, UCJ ansatz의 스핀-균형(spin-balanced) 변형인 [UCJOpSpinBalanced](https://qiskit-community.github.io/ffsim/api/ffsim.html#ffsim.UCJOpSpinBalanced)를 사용할 것입니다.

IBM 하드웨어는 heavy-hex 토폴로지를 가지고 있으므로, 큐비트 상호작용을 위해 $\[1\]$(#references)에서 사용되었고 위에서 설명한 *지그재그(zig-zag)* 패턴을 채택할 것입니다. 이 패턴에서는 동일한 스핀을 가진 오비탈(큐비트)들이 선형(line) 토폴로지로 연결됩니다(빨간색과 파란색 원). heavy-hex 토폴로지로 인해, 서로 다른 스핀을 가진 오비탈들은 매 4번째 오비탈마다, 즉 0번째, 4번째, 8번째 등등에서 연결을 가집니다(보라색 원).

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig8.svg)


```python
import ffsim
from qiskit import QuantumCircuit, QuantumRegister

n_reps = 2
alpha_alpha_indices = [(p, p + 1) for p in range(num_orbitals - 1)]
alpha_beta_indices = [(p, p) for p in range(0, num_orbitals, 4)]

ucj_op = ffsim.UCJOpSpinBalanced.from_t_amplitudes(
    t2=t2,
    t1=t1,
    n_reps=n_reps,
    interaction_pairs=(alpha_alpha_indices, alpha_beta_indices),
)

nelec = (num_elec_a, num_elec_b)

# create an empty quantum circuit
qubits = QuantumRegister(2 * num_orbitals, name="q")
circuit = QuantumCircuit(qubits)

# prepare Hartree-Fock state as the reference state and append it to the quantum circuit
circuit.append(ffsim.qiskit.PrepareHartreeFockJW(num_orbitals, nelec), qubits)

# apply the UCJ operator to the reference state
circuit.append(ffsim.qiskit.UCJOpSpinBalancedJW(ucj_op), qubits)
circuit.measure_all()
# circuit.decompose().draw("mpl", scale=0.5, fold=-1)
```

반복 레이어를 포함하는 LUCJ 안자츠는 인접한 블록들을 병합함으로써 최적화할 수 있습니다. `n_reps=2`인 경우를 생각해 보면, 가운데 있는 두 개의 궤도 회전 블록을 하나의 궤도 회전 블록으로 병합할 수 있습니다. `ffsim` 패키지에는 이러한 인접 블록 병합을 통해 회로를 최적화하는 `ffsim.qiskit.PRE_INIT`라는 패스 관리자가 있습니다.

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig9.svg)

* 위에서 설명한 지그재그 패턴(그 사이에 보조 큐비트(ancilla qubit)가 있는 두 개의 선형 체인)을 따르는 대상 하드웨어의 물리 큐비트(`initial_layout`)를 선택합니다. 이 패턴으로 큐비트를 배치하면 게이트 수가 더 적은 효율적인 하드웨어 호환 회로를 얻을 수 있습니다.

* 회로에 pass manager를 실행합니다.

```python
print(f"Gate counts (w/ pre-init passes): {isa_circuit.count_ops()}")
```

## 4. 후처리 결과

SQD 워크플로의 후처리 부분은 다음 다이어그램을 사용하여 요약할 수 있습니다.

![](https://quantum.cloud.ibm.com/learning/images/courses/quantum-diagonalization-algorithms/sqd2/sqd2-fig10.svg)

계산 기저(computational basis)에서 LUCJ 앤자츠를 샘플링하면 잡음이 포함된 구성(configuration)들의 집합 $\tilde{\mathcal{\chi}}$가 생성되며, 이는 후처리(post-processing) 과정에서 사용된다. 이 과정에는 (자세한 내용은 뒤에서 설명할) *구성 복구(configuration recovery)* 라는 방법이 포함되며, 이를 통해 전자 수가 올바르지 않은 구성을 확률적으로 교정한다. 이후 올바른 전자 수를 갖는 구성들만으로 이루어진 $\tilde{\mathcal{\chi}}_{R}$를 다시 부분 샘플링(subsample*하고, 각 고유한 구성의 출현 빈도에 따라 여러 배치로 분배한다. 각 샘플 배치는 하나의 부분공간($\mathcal{S^{(k)}}$)을 정의한다. 다음으로 분자 해밀토니안 $H$를 각 부분공간으로 사영한다.

<p>
$$
H_{\mathcal{S}^{(k)}} = P_{\mathcal{S}^{(k)}} H *{\mathcal{S}^{(k)}} \text{ with } P*{\mathcal{S}^{(k)}} = \sum_{x \in \mathcal{S}^{(k)}} \vert x \rangle \langle x \vert
$$
</p>

그런 다음 각 사영된 해밀토니안 $H_{\mathcal{S}^{(k)}}$를 고유값 해법기(Eigensolver)에 입력하여 대각화하고, 고유값과 고유벡터를 계산하여 고유상태를 재구성한다. 이 강의에서는 `qiskit-addon-sqd` 패키지를 사용하여 해밀토니안을 사영하고 대각화하며, 이 패키지는 대각화를 위해 PySCF의 Davidson 방법을 사용한다.

<p>
$$
H_{\mathcal{S}^{(k)}} \vert \psi^{(k)} \rangle = E^{(k)} \vert \psi^{(k)} \rangle
$$
</p>

그런 다음 각 배치에서 가장 낮은 고유값(에너지)을 수집하고, 평균 오비탈 점유수(orbital occupancy) $\text{n}$도 계산한다. 평균 점유수 정보는 구성 복구 단계에서 잡음이 포함된 구성을 확률적으로 교정하는 데 사용된다.

다음으로 자기일관적 구성 복구(self-consistent configuration recovery) 반복 과정을 자세히 설명하고, 앞서 언급한 단계들을 구현하여 $N_2$ 해밀토니안의 바닥상태 에너지를 추정하는 구체적인 코드 예제를 제시한다.



### 4.1 구성 복구: 개요

비트열(bitstring, 슬레이터 행렬식(Slater determinant))의 각 비트는 하나의 스핀 오비탈(spin orbital)을 나타낸다. 비트열의 오른쪽 절반은 스핀-업 오비탈을 나타내고, 왼쪽 절반은 스핀-다운 오비탈을 나타낸다. `1`은 해당 오비탈이 전자로 점유되어 있음을 의미하고, `0`은 해당 오비탈이 비어 있음을 의미한다. 올바른 입자 수(스핀-업 전자와 스핀-다운 전자 모두)는 사전에 알고 있다고 가정한다. 전자 수가 $N_x$인(즉, 비트열에 `1`이 $N_x$개 있는) 행렬식 $x$를 생각하자. 올바른 입자 수는 $N$이다. 만약 $N_x \neq N$이면, 해당 비트열은 노이즈에 의해 손상되었다는 것을 알 수 있다. 자기 일관적 구성(self-consistent configuration) 루틴은 평균 오비탈 점유도 정보를 활용하여 $\|N_x - N\|$개의 비트를 확률적으로 뒤집음으로써 비트열을 복원하려고 시도한다. 평균 오비탈 점유도($n$)는 각 오비탈이 전자로 점유될 가능성이 얼마나 되는지를 나타낸다. 만약 $N_x < N$이면 전자 수가 부족하므로 일부 `0`을 `1`로 뒤집어야 하며, 반대의 경우도 마찬가지이다.

`i`번째 스핀 오비탈에 대해 비트를 뒤집을 확률은 $\|x\[i\] - avg_{occupancy}\[i\]\|$로 둘 수 있다. $\[2\]$(#references)에서는 수정된 ReLU 함수를 이용한 가중 확률을 사용하여 비트를 뒤집었다.

<p>
$$
\begin{align}
    w(y) = \begin{cases}
    \delta \frac{y}{h} & \text{if }  y \leq h\\ \nonumber
    \delta + (1 - \delta) \frac{y - h}{1 - h} & \text{if } y > h
\end{cases}
\end{align}
$$
</p>

여기서 $h$는 ReLU 함수의 "모서리(corner)" 위치를 정의하며, 매개변수 $\delta$는 그 모서리에서의 ReLU 함수 값을 정의한다. $\delta = 0$이면 $w$는 실제 ReLU 함수가 되고, $\delta > 0$이면 *수정된(modified)* ReLU*함수가 된다. 논문에서는 $\delta = 0.01$과 $h =$ 알파(또는 베타) 입자 수 / 알파(또는 베타) 스핀 오비탈 수 $= N/M$ (충전율, filling factor)를 사용하였다.



평균 오비탈 점유도($n$)는 사전에 알려져 있지 않다. 바닥 상태 추정의 첫 번째 반복(iteration)은 두 스핀 종 모두에서 올바른 입자 수를 갖는 구성들로 시작한다. 첫 번째 반복 이후에는 바닥 상태의 추정값을 얻게 되며, 이 추정값을 이용하여 $n$의 첫 번째 추정값을 구성할 수 있다. 이 추정된 $n$을 사용하여 구성들을 복원하고, 다음 반복의 바닥 상태 추정을 수행하며, 자기 일관적으로 $n$의 추정값을 점진적으로 개선한다. 이 과정은 종료 조건이 만족될 때까지 반복된다.

다음 예를 생각해 보자. $N = 2$이고 $x = \|1000\rangle$ ($N_x = 1$)이라고 하자. 입자 수를 올바르게 맞추기 위해 `0` 중 하나를 `1`로 뒤집어야 하며, 가능한 선택지는 `1100`, `1010`, `1001`이다. 비트를 뒤집을 확률에 따라 이들 중 하나가 복원된 구성(recovered configuration) (즉, 올바른 입자 수를 갖는 비트열)으로 선택된다.

첫 번째 반복에서 두 개의 배치(batch)를 실행하였고, 각 배치에서 추정된 바닥 상태가 다음과 같다고 하자.

<p>
$$
\begin{align}\nonumber
\text{Batch0: } \vert \psi \rangle &= 0.8 \times \vert 1001 \rangle + 0.6 \times \vert 0110 \rangle \ \nonumber \\
\text{Batch1: } \vert \psi \rangle &= \frac{1}{\sqrt{3}} \left( \vert 1001 \rangle + \vert 0101 \rangle + \vert 0110 \rangle \right) \nonumber
\end{align}
$$
</p>

계산 기저 상태(computational basis state)와 그 진폭(amplitude)을 이용하여 각 스핀 오비탈(큐비트)에서의 전자 점유 확률(간단히 점유도(occupancy))을 계산할 수 있다(확률 = \|amplitude\|$^2$임에 유의하라). 아래에서는 추정된 바닥 상태에 등장하는 각 비트열에 대해 큐비트별 점유도를 표로 나타내고, 각 배치의 전체 오비탈 점유도를 계산하였다. Qiskit의 비트 순서 규약에 따라 가장 오른쪽 비트는 큐비트-0(Q0)을 나타내고, 가장 왼쪽 비트는 Q3을 나타낸다는 점에 유의하라.

위에서 계산한 평균 오비탈 점유도를 이용하여 구성 $x = \vert 1000 \rangle$의 서로 다른 오비탈에 대한 비트 뒤집기 확률을 구할 수 있다. Q3가 나타내는 오비탈은 이미 점유되어 있으므로 뒤집을 필요가 없으며, 따라서 p(flip)을 $0$으로 둔다. 나머지 점유되지 않은 오비탈들에 대해서는 각각의 비트 뒤집기 확률이 $\vert x\[i\] - \text{n}\[i\] \vert$가 된다. 또한 p(flip)과 함께, 앞에서 설명한 수정된 ReLU 함수를 사용하여 비트 뒤집기에 대응하는 확률 가중치도 계산한다.

비트 뒤집기 확률 ($x = \vert 1000 \rangle$, $\delta = 0.01$, $h = N/M = 2/4 = 0.50$)

|                                            |  Q3 |  Q2  |   Q1  |  Q0  |
| :----------------------------------------: | :-: | :--: | :---: | :--: |
| p(flip) ($\vert x[i] - \text{n}[i] \vert$) |  0  | 0.51 |  0.35 | 0.65 |
|                 w(p(flip))                 |  0  | 0.03 | 0.007 | 0.31 |



마지막으로, 위에서 계산한 가중 확률을 사용하여 점유되지 않은 Q2, Q1, Q0 오비탈 중 하나를 뒤집을 수 있다. 위의 값에 따르면 Q0가 가장 높은 확률로 뒤집히며, 가능한 복원된 구성은 $\vert \text{1001} \rangle$이 될 수 있다.

완전한 자기일관적 구성 복원(self-consistent configuration recovery) 과정은 다음과 같이 요약할 수 있다.

**첫 번째 반복:**
양자 컴퓨터가 생성한 비트열(구성(configuration) 또는 슬레이터 행렬식(Slater determinant))이 집합 $\widetilde{\chi}$를 이룬다고 가정하자. 이 집합에는 각 스핀 부문에서 입자 수가 올바른 구성($\widetilde{\chi}\*{correct}$)과 올바르지 않은 구성($\widetilde{\chi}\*{incorrect}$)이 모두 포함되어 있다.

1. ($\widetilde{\chi}_{correct}$)의 구성들을 무작위로 샘플링하여 부분공간 사영을 위한 벡터 배치 $(\mathcal{S}^{(1)}, \cdots, \mathcal{S}^{(K)})$를 생성한다. 배치의 개수와 각 배치의 샘플 수는 사용자가 정의하는 매개변수이다. 각 배치의 샘플 수가 많을수록 부분공간의 차원이 커지고, 대각화 계산의 비용도 증가한다. 반면, 샘플 수가 너무 적으면 바닥 상태의 지지 벡터(support vectors)를 놓쳐 잘못된 추정으로 이어질 수 있다.
2. 각 배치에 대해 고유상태 해법기(eigenstate solver)(즉, 부분공간으로의 사영 및 대각화)를 실행하여 근사 고유상태 $\|\psi^{(1)}\rangle, \cdots, \|\psi^{(K)}\rangle$를 얻는다.
3. 근사 고유상태들로부터 $n$의 첫 번째 추정값을 구성한다.

**이후 반복:**

1. $n$을 사용하여 $\widetilde{\chi}\*{incorrect}$에 있는 잘못된 입자 수를 가진 구성들을 수정한다. 이를 $\widetilde{\chi}\*{correct_new}$라고 하자. 그러면 $\widetilde{\chi}\*{recovered} (\widetilde{\chi}\*{R}) = \widetilde{\chi}\*{correct} \cup \widetilde{\chi}\*{correct_{new}}$가 올바른 입자 수를 갖는 새로운 구성들의 집합을 이룬다.
2. $\widetilde{\chi}_{R}$를 샘플링하여 배치 $\mathcal{S}^{(1)}, \cdots, \mathcal{S}^{(K)}$를 생성한다.
3. 새로운 배치를 사용하여 고유상태 해법기를 실행하고, 바닥 상태의 새로운 추정값 $\|\psi^{(1)}\rangle, \cdots, \|\psi^{(K)}\rangle$를 생성한다.
4. 근사 고유상태들로부터 $n$의 개선된 추정값을 구성한다.
5. 종료 조건이 만족되지 않으면 단계 `2.1`로 돌아간다.



### 4.2 바닥 상태 추정

먼저, 후처리를 위해 카운트(counts)를 비트열(bitstring) 행렬과 확률 배열로 변환합니다.

행렬의 각 행은 하나의 고유한 비트열을 나타냅니다. Qiskit에서는 큐비트가 비트열의 오른쪽부터 인덱싱되므로, 열 `0`은 큐비트 `N-1`을 나타내고, 열 `N-1`은 큐비트 `0`을 나타냅니다. 여기서 `N`은 큐비트의 개수입니다.

알파 오비탈은 열 인덱스 범위 `(N, N/2]`(오른쪽 절반)로 표현되며, 베타 오비탈은 열 인덱스 범위 `(N/2, 0]`(왼쪽 절반)로 표현됩니다.

```python
from qiskit_addon_sqd.counts import counts_to_arrays

# Convert counts into bitstring and probability arrays
bitstring_matrix_full, probs_arr_full = counts_to_arrays(counts)
```

이 기법에 중요한 몇 가지 사용자 제어 옵션이 있습니다.

* `iterations`: 자체 일관성 구성 복구 반복 횟수
* `n_batches`: 고유 상태 솔버 호출 시 사용되는 구성 배치 수
* `samples_per_batch`: 각 배치에 포함할 고유 구성 수
* `max_davidson_cycles`: 각 고유 상태 솔버에서 실행되는 최대 데이비슨 사이클 수

```python
import numpy as np
from qiskit_addon_sqd.configuration_recovery import recover_configurations
from qiskit_addon_sqd.fermion import (
    bitstring_matrix_to_ci_strs,
    solve_fermion,
)
from qiskit_addon_sqd.subsampling import postselect_and_subsample

rng = np.random.default_rng(24)
# SQD options
iterations = 5

# Eigenstate solver options
n_batches = 5
samples_per_batch = 500
max_davidson_cycles = 300

# Self-consistent configuration recovery loop
e_hist = np.zeros((iterations, n_batches))  # energy history
s_hist = np.zeros((iterations, n_batches))  # spin history
occupancy_hist = []
avg_occupancy = None
for i in range(iterations):
    print(f"Starting configuration recovery iteration {i}")
    # On the first iteration, we have no orbital occupancy information from the
    # solver, so we begin with the full set of noisy configurations.
    if avg_occupancy is None:
        bs_mat_tmp = bitstring_matrix_full
        probs_arr_tmp = probs_arr_full

    # If we have average orbital occupancy information, we use it to refine
    # the full set of noisy configurations.
    else:
        bs_mat_tmp, probs_arr_tmp = recover_configurations(
            bitstring_matrix_full,
            probs_arr_full,
            avg_occupancy,
            num_elec_a,
            num_elec_b,
            rand_seed=rng,
        )

    # Create batches of subsamples. We postselect here to remove configurations
    # with incorrect hamming weight during iteration 0, since no config recovery was performed.
    batches = postselect_and_subsample(
        bs_mat_tmp,
        probs_arr_tmp,
        hamming_right=num_elec_a,
        hamming_left=num_elec_b,
        samples_per_batch=samples_per_batch,
        num_batches=n_batches,
        rand_seed=rng,
    )

    # Run eigenstate solvers in a loop. This loop should be parallelized for larger problems.
    e_tmp = np.zeros(n_batches)
    s_tmp = np.zeros(n_batches)
    occs_tmp = []
    coeffs = []
    for j in range(n_batches):
        strs_a, strs_b = bitstring_matrix_to_ci_strs(batches[j])
        print(f"  Batch {j} subspace dimension: {len(strs_a) * len(strs_b)}")
        energy_sci, coeffs_sci, avg_occs, spin = solve_fermion(
            batches[j],
            hcore,
            eri,
            open_shell=open_shell,
            spin_sq=spin_sq,
            max_davidson=max_davidson_cycles,
        )
        energy_sci += nuclear_repulsion_energy
        e_tmp[j] = energy_sci
        s_tmp[j] = spin
        occs_tmp.append(avg_occs)
        coeffs.append(coeffs_sci)

    # Combine batch results
    avg_occupancy = tuple(np.mean(occs_tmp, axis=0))

    # Track optimization history
    e_hist[i, :] = e_tmp
    s_hist[i, :] = s_tmp
    occupancy_hist.append(avg_occupancy)
```

### 4.3 결과에 대한 논의

첫 번째 그래프는 몇 번의 반복(iteration) 후에 바닥 상태 에너지가 약 24 mH 이내로 추정됨을 보여줍니다 (화학적 정확도는 보통 1 kcal/mol $\approx$ 1.6 mH로 인정됩니다). 두 번째 그래프는 마지막 반복 후 각 공간 오비탈(spatial orbital)의 평균 점유율을 보여줍니다. 우리의 해(solution)에서 스핀 업 및 스핀 다운 전자 모두 높은 확률로 처음 5개의 오비탈을 점유하고 있음을 확인할 수 있습니다.

추정된 바닥 상태 에너지가 준수한 편이지만, 화학적 정확도 한계($\pm \approx 1.6$ mH) 이내에는 들지 못합니다. 이러한 격차는 투영(projection) 및 대각화(diagonalization)를 위해 위에서 사용한 하위 공간(subspace)의 차원이 작기 때문일 수 있습니다. `samples_per_batch=500`을 사용했으므로 하위 공간은 최대 $500$개의 벡터로 스팬(span)되며, 이는 바닥 상태 지지(ground state support)에서 나오는 벡터들이 누락된 상태입니다. `samples_per_batch` 매개변수를 늘리면 더 많은 고전적 컴퓨팅 자원과 실행 시간이 소요되는 대신 정확도가 향상될 것입니다.

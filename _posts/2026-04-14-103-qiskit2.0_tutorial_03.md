## 1. 동적 회로 구성

개요: 이 노트북은 동적 회로를 구성하는 방법을 다룹니다.

*   `if` 문, 멀티비트 클래식 레지스터 및 고급 표현식을 사용한 클래식 피드 포워드 및 제어 흐름
*   클래식 제어 흐름과 관련된 Qiskit 런타임 제한 사항

**클래식 피드포워드 및 제어 흐름**

* 측정 결과(클래식 값)를 이용해서 이후의 양자 연산을 조건적으로 수행하는 것(양자 → 고전 → 다시 양자 흐름)

* 클래식 피드포워드를 사용하면 양자 회로가 측정 결과를 기반으로 결정을 내릴 수 있어, 이후의 연산이 이전 측정 결과에 의존하는 적응형 양자 알고리즘을 가능하게 합니다. 몇 가지 사용 사례가 확인 되었습니다. 예를 들어:
    - 효율적인 양자 상태 준비
    - 효율적인 장거리 엔탱글먼트

* 클래식 피드 포워드 및 동적 회로에 대한 더 많은 정보는 여기서 확인할 수 있습니다.

### 1. 클래식 조건을 사용한 IF 문

`if_test` 메서드는 클래식 레지스터 값을 기반으로 조건부 연산을 허용합니다. 이는 측정 기반 양자 컴퓨팅, 적응형 알고리즘 및 오류 수정에 사용될 수 있습니다.

`else` 블록도 `with` 문에 추가할 수 있으며, 이는 `if` 블록이 실행되지 않을 때 실행됩니다.

```python
# Create quantum and classical registers
qubits = QuantumRegister(2)
clbits = ClassicalRegister(2)
circuit = QuantumCircuit(qubits, clbits)
(q0,q1) = qubits
(c0,c1) = clbits
 
# Step 1: Create superposition and measure
circuit.h(q0)                    # Apply Hadamard: |0⟩ → (|0⟩ + |1⟩)/√2
circuit.measure(q0, c0)          # Measure qubit and store result in classical bit
 
# Step 2: Conditional operation based on measurement result
with circuit.if_test((c0, 1)) as else_:
    circuit.x(q1)               # X gate is applied on q1 if c0 == 1
with else_:
    circuit.h(q1)               # H gate is applied on q1 if c0 == 0

# Step 3: Measure c1 to see the effect of conditional operation
circuit.measure(q1, c1)

 
print("Dynamic Circuit with Classical Conditional:")
circuit.draw("mpl")
 
# - If first measurement gives |1⟩ (c0=1): X gate applied, second measurement gives c1=1
# - If first measurement gives |0⟩ (c0=0): H gate is applied on q1, c1 has 50% propability of being 1 and 50% of being 0

# Output {'00': 25%, '10': 25%, '11': 50%}
```

### 2. 멀티비트 클래식 레지스터 조건

동일한 조건부 로직을 멀티비트 클래식 레지스터에 적용할 수 있으며, 이를 통해 멀티큐비트 측정 결과를 기반으로 더 복잡한 의사 결정을 가능하게 합니다.

```python
# Create quantum and classical registers
qubits = QuantumRegister(4)
clbits = ClassicalRegister(4)
circuit = QuantumCircuit(qubits, clbits)
(q0,q1,q2,q3) = qubits
(c0,c1,c2,c3) = clbits
 
# Step 1: Create superposition and measure
circuit.h([q0,q1])                 # Apply Hadamard: |00⟩ → (|00⟩ + |01⟩ + |10⟩ + |11⟩)/2
circuit.measure([q0,q1] ,[c0,c1]) # Measure qubits and store result in classical bits
 
# Step 2: Conditional operation based on measurement result
with circuit.if_test((clbits, 0b0000)):
    circuit.x(q2)               # X gate is applied on q2 if c1c0 == 00
    circuit.x(q3)               # x gate is applied on q3 if c1c0 == 00
with circuit.if_test((clbits, 0b0001)):
    circuit.x(q3)               # X gate is applied on q3 if c1c0 == 01    
with circuit.if_test((clbits, 0b0010)):
    circuit.x(q2)               # X gate is applied on q2 if c1c0 == 10
    
    
# Step 3: Measure c1 to see the effect of conditional operation
circuit.measure([q2,q3], [c2,c3])

 
print("Dynamic Circuit for multiple bits register")
circuit.draw("mpl")

# 빈 원(○)은 해당 classical 비트가 0일 때 조건이 참

# - If first measurement gives |00⟩ : X gate applied on q2 and q3, second measurement gives c3c2=11
# - If first measurement gives |01⟩ : X gate applied on q3, second measurement gives c3c2=10
# - If first measurement gives |10⟩ : X gate applied on q2, second measurement gives c3c2=01
# - If first measurement gives |11⟩ : No gate applied, second measurement gives c3c2=00
```

### 3. 고급 클래식 표현식

Qiskit 는 `expr` 모듈을 사용하여 복잡한 클래식 표현식을 지원하며, 여러 클래식 비트와 그 관계를 기반으로 한 정교한 조건부 로직을 가능하게 합니다.

```python
# Create registers
qr = QuantumRegister(4, 'q')
cr = ClassicalRegister(3, 'c')   # For first three qubits
cr2 = ClassicalRegister(1, 'c2') # For the fourth qubit
circuit = QuantumCircuit(qr, cr, cr2)

# Create superposition on first three qubits
circuit.h(qr[0])
circuit.h(qr[1])
circuit.h(qr[2])
circuit.measure(qr[0:3], cr)  # Measure first three qubits

# Create complex condition: XOR of all three classical bits
# This checks the parity (even/odd number of 1's)
_condition = expr.bit_xor(expr.bit_xor(cr[0], cr[1]), cr[2])

# Apply X gate to fourth qubit if parity is odd (XOR result is 1)
with circuit.if_test(_condition):
    circuit.x(qr[3])

# Measure the fourth qubit
circuit.measure(qr[3], cr2)

circuit.draw("mpl")

# Execute the circuit to see results
backend = GenericBackendV2(num_qubits=len(circuit.qubits), control_flow=True)
transpiled_circuit = transpile(circuit, backend)
result = backend.run(transpiled_circuit).result()
print("\nMeasurement results:")
print(result.get_counts())
```

### 4. 동적 회로를 지원하는 백엔드 찾기

모든 백엔드가 동적 회로를 지원하는 것은 아닙니다. 아래 코드는 IBM Quantum 계정에서 이 기능을 지원하는 백엔드를 확인합니다.

```python
# Initialize the service (update credentials if not saved)

```

### 5. 클래식 제어 흐름에 대한 Qiskit 런타임 제한 사항

Qiskit 에서 클래식 조건부를 사용할 때 중요한 제약 사항:

*   비트 제한 : `if_test` 문의 피연산자는 32 비트 이하여야 하며, 클래식 레지스터를 비교하는 경우 32 비트 미만이어야 합니다.
*   브로드캐스팅 : Qiskit 런타임은 한 번에 60 비트 이상을 "제어 로직에 데이터 전송"으로 브로드캐스트할 수 없습니다 (서로 다른 브로드캐스트를 관리하려면 배리어를 사용하십시오).
*   네스팅 불가 : 중첩 조건부 (다른 `if_test` 내부의 `if_test`) 는 지원되지 않습니다.
*   조건부 내 Reset/Measure 불가 : 조건부 내의 Reset 또는 측정 연산은 지원되지 않습니다.
*   산술 연산 불가 : 클래식 표현식 내의 산술 연산은 지원되지 않습니다.
*   `for`, `while` (문법은 존재하나 실제 장비에서 지원 제한적)및 `switch` 명령은 지원되지 않습니다.

이러한 제한 사항들은 복잡한 클래식 피드포워드 회로를 설계할 때 고려하는 것이 중요합니다.

## 2. 파라미터화된 회로 구성

개요: 이 노트북은 파라미터화된 회로를 구성하는 방법을 다룹니다.

Qiskit 의 파라미터화된 회로는 파라미터를 조정하여 다양한 작업에 적응할 수 있는 유연한 양자 알고리즘을 생성하는 데 사용됩니다. 우리는 다음을 식별하는 것으로 시작합니다:

*   `Parameter` 클래스 및 그 속성과 메서드
*   양자 회로 및 여기서 파라미터를 사용하는 방법
*   `Parameter` 표현식 및 언제 사용하는지

```python
import numpy as np
from qiskit.circuit import QuantumCircuit, Parameter
```

### 1. Parameter

Qiskit 에서 `Parameter` 클래스는 양자 회로에서 사용할 수 있는 기호 변수를 생성할 수 있게 해줍니다. 이러한 파라미터는 회로 실행 때까지 바인딩되지 않은 상태로 유지되어 파라미터화된 양자 회로의 생성을 가능하게 합니다.

```python
 # create the parameter  symbolic variable named 'phi' 
phi = Parameter('phi')
qc = QuantumCircuit(1)
 
# use parameter in place of numeric values in quantum gates
# The circuit remains unbound until parameters are assigned
qc.rx(phi, 0)
qc.draw('mpl')
```

파라미터는 회로가 생성된 후 값이 할당됩니다. 그런 다음 동일한 회로를 다른 파라미터 값으로 여러 번 실행할 수 있습니다.

```python
# bind the parameters after circuit to create a bound circuit
# it can be assigned on the same circuit or a copy of it
qc = qc.assign_parameters({phi: np.pi})
qc.measure_all()
print("Parameterized circuit:")
print("phi = π")
qc.draw('mpl')
```

#### 1. 속성

| 속성 | 설명 | 참고 |
|-----------|-------------|----------------|
| `name` | 파라미터 이름 | |
| `uuid` | 파라미터 식별자 | |

```python
print(f"Parameter name: {phi.name}")
print(f"Parameter UUID: {phi.uuid}")
```

#### 2. 메서드

*   `abs` : 파라미터의 절대값.
*   `arccos` : 파라미터의 아크코사인.
*   `arcsin` : 파라미터의 아크사인.
*   `arctan` : 파라미터의 아크탄젠트.
*   `assign` : 하나의 파라미터를 값에 할당.
*   `bind` : 파라미터를 값에 바인딩.
*   `conjugate` : 파라미터의 복소수 켤레.
*   `cos` : 파라미터의 코사인.
*   `exp` : 파라미터의 지수.
*   `gradient` : 입력 파라미터에 대한 미분.
*   `is_real` : 파라미터가 실수인지 확인.
*   `log` : 파라미터의 자연 로그.
*   `numeric` : 파라미터를 수치 값으로 캐스팅.
*   `sign` : 파라미터의 부호 반환.
*   `sin` : 파라미터의 사인.
*   `subs` : 대체 파라미터로 새 표현식을 반환.
*   `sympify` : 파라미터의 SymPy 동등물.
*   `tan` : 파라미터의 탄젠트.

```python
print(f"Sin: {phi.sin()}")
# phi.sin()은 계산 결과가 아니라 "sin(phi)라는 표현식"을 반환
print(f"Cos: {phi.cos()}")
print(f"tan: {phi.tan()}")
```

### 2. Quantum Circuit

`QuantumCircuit` 클래스는 Qiskit 에서 양자 프로그램의 기본 구성 요소입니다. 이는 양자 게이트 및 연산의 컬렉션을 나타냅니다.

파라미터는 다양한 큐비트와 게이트 유형에서 사용될 수 있으며, 또한 회로는 여러 파라미터를 포함할 수 있습니다.

아래 예시는 여러 파라미터가 있는 회로를 보여주며, 회로의 모든 바인딩되지 않은 파라미터를 얻으려면 `parameters` 속성을 사용할 수 있습니다.

```python
# Create a multi-parameter circuit
theta = Parameter('θ')
gamma = Parameter('γ')

qc2 = QuantumCircuit(2)
qc2.ry(theta, 0)
qc2.rz(gamma, 1)
qc2.cx(0, 1)

print("Multi-parameter circuit:")
print(f"Circuit parameters: {qc2.parameters}")
qc2.draw('mpl')
```

`dict` 유형을 사용하여 여러 파라미터를 값에 바인딩합니다.

```python
# Multiple ways to bind parameters
param_dict = {theta: np.pi/2, gamma: np.pi/4}

# assign_parameters (returns new circuit)
bound_circuit = qc2.assign_parameters(param_dict)


print("Bound circuit values:")
print(f"theta = π/2, gamma = π/4")
bound_circuit.draw('mpl')
```

### 3. Parameter Expression

`ParameterExpression` 은 파라미터의 수학적 표현식을 생성할 수 있게 하여 복잡한 파라미터 관계와 제약을 허용합니다. 이들은 표준 수학 연산을 사용하여 생성되며, 또한 표현식에는 삼각 함수, 로그 및 기타 연산이 포함될 수 있습니다.

```python
from qiskit.circuit import Parameter

# Create base parameters
x = Parameter('x')
y = Parameter('y')

# Create mathematical expressions
expr1 = x + y
expr2 = 2 * x - y**2
expr3 = np.sin(x) + np.cos(y)

print(f"Expression 1: {expr1}")
print(f"Expression 2: {expr2}")
print(f"Expression 3: {expr3}")
print(f"Parameters in expr1: {expr1.parameters}")
```

파라미터 표현식은 양자 게이트에서 직접 사용될 수 있습니다. 회로가 표현식에서 모든 기본 파라미터를 자동으로 추출하기 때문에 종속 파라미터가 있는 회로를 생성할 수 있습니다.

```python
# Create circuit using parameter expressions
alpha = Parameter('α')
beta = Parameter('β')

expr = 2 * alpha - beta

qc3 = QuantumCircuit(1)
qc3.rx(alpha + beta, 0)  # Use parameters as an expression  as rotation angle
qc3.ry(expr, 0) # Use created expression

print("Circuit with parameter expressions:")
print(f"All parameters: {qc3.parameters}")
qc3.draw('mpl')
```

더 복잡한 표현식을 생성할 수 있으며, 이 경우 `bind()` 메서드를 사용하여 파라미터 값을 표현식에 할당합니다.

```python
# Create and evaluate complex expressions
expr = (x + y) * np.cos(x) - y**2
print(f"Complex expression: {expr}")

# Bind parameters and evaluate
param_values = {x: 1.0, y: 2.0}
bound_expr = expr.bind(param_values)
numeric_value = float(bound_expr)

print(f"Bound expression: {bound_expr}")
print(f"Numeric value: {numeric_value}")
```

theta = Parameter("θ")

**트랜스파일러 단계**

트랜스파일레이션은 양자 회로를 실제 양자 하드웨어에서 실행 가능하도록 변환하는 과정입니다. 이는 다음을 포함합니다:
- 논리 큐비트를 물리 큐비트에 매핑.
- 연결성 제약을 만족하기 위해 SWAP 게이트 추가.
- 게이트를 하드웨어 네이티브 기반 게이트로 변환.
- 회로 깊이 및 게이트 수 최적화.
- 유휴 시간을 최소화하기 위한 작업 스케줄링.

###  1. 트랜스파일러 단계

Qiskit 의 트랜스파일러는 이러한 단계를 통해 회로를 처리합니다:
1.  초기화  - 사용자 정의 최적화 및 다중 큐비트 게이트 분해
2.  레이아웃  - 가상 큐비트를 물리 큐비트에 매핑
3.  라우팅  - 연결성을 위한 SWAP 게이트 삽입
4.  변환  - 게이트를 하드웨어 기반으로 변환
5.  최적화  - 회로 깊이 및 게이트 수 감소
6.  스케줄링  - 타이밍 및 지연 삽입

#### 1. 초기화 단계

초기화 단계는 다음을 통해 트랜스파일을 위해 회로를 준비합니다:
- 사용자 정의 최적화 적용.
- 다중 큐비트 게이트 (>2 큐비트) 를 1 및 2 큐비트 게이트로 분해.
- 후속 단계를 위해 회로 설정.

#### 2. 레이아웃 단계

레이아웃 단계는 필요한 SWAP 게이트 수를 최소화하는 것을 목표로 타겟 백엔드의 물리 큐비트에 가상 큐비트를 매핑합니다. 다양한 레이아웃 전략은 다음과 같습니다:

-  TrivialLayout : 간단한 순차 매핑 (큐비트 0→0, 1→1 등)
-  VF2Layout : 서브그래프 동형 사상을 사용하여 최적의 매핑을 찾습니다.
-  DenseLayout : 가장 연결성이 높은 큐비트 부분집합을 찾습니다.
-  SabreLayout : 반복적인 양방향 라우팅을 사용합니다.
-  CSPLayout : 제약 충족 문제 접근 방식.

#### 3. 라우팅 단계

라우팅 단계는 회로를 하드웨어의 연결성 그래프에서 실행 가능하게 만들기 위해 SWAP 게이트를 삽입합니다. 이 단계는 기본적으로 레이아웃 단계와 함께 계산되지만 논리적으로는 서로 distinct 합니다.

다양한 라우팅 방법:
-  SabreSwap : 가장 일반적으로 사용되며 휴리스틱 기반 라우팅.
-  BasicSwap : 간단하지만 효율성은 떨어짐.
-  LookaheadSwap : 더 정교하지만 계산 비용이 높음.

### 3. 트랜스파일러 패스

#### 1. 레이아웃

-  TrivialLayout : n 개의 회로 큐비트를 장치 큐비트 0, .., n-1 에 할당하여 레이아웃을 선택합니다.
-  DenseLayout : 가장 연결성이 높은 큐비트 부분집합을 찾아 레이아웃을 선택합니다.
-  SabreLayout : 입력 회로의 반복적인 양방향 라우팅을 통해 레이아웃을 선택합니다.
-  CSPLayout : 가능하다면 백트래킹을 사용하여 CSP 로 레이아웃을 선택합니다.
-  VF2Layout: 결합 그래프에서 회로 레이아웃을 선택하는 과정으로, 부분 그래프 동형성 문제를 VF2++로 해결합니다.

#### 2. 라우팅

-  BasicSwap : swap 게이트를 추가하여 DAGCircuit 을 coupling_map 에 매핑합니다 (최소한의 노력으로).
-  Commuting2qGateRouter : 하나 이상의 교환 게이트를 결합 맵으로 스왑 라우트합니다.
-  LayoutTransformation : 주어진 (부분) 순열에 대한 Swap 회로를 회로에 추가합니다.
-  LookaheadSwap : SWAP 삽입을 통해 입력 회로를 백엔드 토폴로지에 매핑합니다.(트리탐색-정밀)
-  SabreSwap : SWAP 삽입을 통해 입력 회로를 백엔드 토폴로지에 매핑합니다.(휴리스틱-근사)
-  StarPreRouting : star to lineat 프리 라우팅을 실행합니다.

#### 3. 기반 변경

-  BasisTranslator : 주어진 EquivalenceLibrary 에서 변환 세트를 검색하여 게이트를 타겟 기반으로 변환합니다.
-  Decompose : 분해 규칙을 사용하여 회로의 게이트를 확장합니다.
-  TranslateParameterizedGates : 파라미터화된 게이트를 지원되는 기반 세트로 변환합니다.
-  Unroll3qOrMore: 회로에 2q 또는 1q 게이트만 포함될 때까지 3q+ 게이트를 재귀적으로 확장합니다.
-  UnrollCustomDefinitions : 사용자 정의가 있는 명령을 언롤합니다.

#### 4. 최적화

-  Collect1qRuns : 단일 큐비트 서브서킷을 수집합니다.
-  Collect2qBlocks : 두 큐비트 서브서킷을 수집합니다.
-  CommutativeCancellation : 교환 관계를 통해 중복 (자기 수반) 게이트를 취소합니다.
-  ConsolidateBlocks : 연속적인 게이트의 각 블록을 단일 Unitary 노드로 대체합니다.
-  InverseCancellation : 백투백으로 발생할 때 서로 역인 특정 게이트를 취소합니다.
-  Optimize1qGates : 단일 큐비트 u1, u2, u3 게이트 체인을 하나의 게이트로 결합하여 최적화합니다.
-  Optimize1qGatesDecomposition : 단일 큐비트 게이트 체인을 하나의 게이트로 결합하여 최적화합니다.
-  Optimize1qGatesSimpleCommutation : 구성 요소를 교환하고 결과를 재합성하여 2Q 게이트에 의해 중단된 1Q 게이트 문자열을 최적화합니다.

#### 5. 스케줄링

-  ALAPScheduleAnalysis : ALAP 스케줄링 패스로, 명령의 중지 시간을 가능한 한 늦게 스케줄합니다.
-  ASAPScheduleAnalysis : ASAP 스케줄링 패스로, 명령의 시작 시간을 가능한 한 빨리 스케줄합니다.
-  ConstrainedReschedule : 노드 시작 시간을 하드웨어 정렬에 conforms 하도록 업데이트하는 Rescheduler 패스입니다.
-  ContextAwareDynamicalDecoupling : 게이트 및 큐비트 컨텍스트를 고려하여 X-sequence 동적 디커플링을 구현합니다.
-  InstructionDurationCheck : reschedule 을 위한 지속 시간 유효성 검사 패스입니다.
-  PadDelay : 유휴 시간을 Delay 명령으로 패딩합니다.
-  PadDynamicalDecoupling : 동적 디커플링 삽입 패스입니다.
-  SetIOLatency : 입력 회로에 IOLatency 정보를 설정합니다.
-  TimeUnitConversion : 다음 시간 인식 패스에서 사용할 시간 단위를 선택하고 모든 회로 시간 단위를 해당 단위와 일치하도록 만듭니다.

Qiskit 회로 계층 구조:

-   CircuitInstruction : `Operation` 과 그 피연산자인 `Qubit` 또는 `Clbit` 로 구성됩니다.

회로에는 실시간으로 평가되는 클래식 표현식이 포함될 수 있습니다:

-   Expr : 실시간으로 평가되는 표현식.

#### 2. Creating Instruction SubClasses

사용자 정의 instruction 또는 gate 를 생성하는 가장 쉬운 방법은 단순히 정의를 QuantumCircuit 으로 빌드한 다음 `to_instruction()` 또는 `to_gate()` 를 사용하는 것입니다. 결과는 더 큰 회로의 `QuantumCircuit.append()` 에 직접 전달될 수 있습니다.

보다 복잡한 instruction 의 경우 또는 광범위하게 사용될 경우, 사용자 정의 gate 또는 instruction 을 생성할 수 있습니다.

기본 클래스인 `Instruction` , `Gate` 및 `ControlledGate` 는 하위 클래스로 생성할 수 있으며, instruction 또는 gate 에 대한 회로 구현을 제공하기 위해 `_define()` 메서드를 재정의해야 합니다. 효율적인 최적화가 있는 경우 `inverse()` , `control()` 또는 `power()` 와 같은 다른 메서드도 재정의할 수 있습니다.

```python
# Example  Custom Two-Qubit Gate Definition
class RXZGate(Gate):
    """A custom two-qubit gate implementing an RZX-like operation."""
    
    def __init__(self, theta): 
        # Qiskit 의 기본 Gate 클래스의 설정을 불러옵니다.
        # Initialize with our name, number of qubits and parameters.
        super().__init__("rxz", 2, [theta])
 
    def _define(self):
        # _define 은 하드웨어 실행을 위한 것이고, __array__ 는 시뮬레이션/수학 계산을 위한 것
        # 이 커스텀 게이트를 실제 하드웨어에서 실행할 수 있는 기본 게이트들로 분해 (Decomposition)
        # base definition is an RZXGate, applied "backwards".
        defn = QuantumCircuit(2)
        defn.rzx(self.params[0], 1, 0)  # Note: qubit order swapped
        self._definition = defn
 
    def inverse(self, annotated=False):
        # We have an efficient representation of our inverse,
        # so we'll override this method.
        return RXZGate(-self.params[0])
 
    def power(self, exponent: float):
        # Also we have an efficient representation of power.
        return RXZGate(exponent * self.params[0])
 
    def __array__(self, dtype=None, copy=None):
        # 행렬 표현 
        if copy is False:
            raise ValueError("unable to avoid copy while creating an array as requested")
        cos = math.cos(0.5 * self.params[0])
        isin = 1j * math.sin(0.5 * self.params[0])
        # 이 게이트의 수학적 행렬 (Unitary Matrix)을 반환
        # 실제 양자 컴퓨터가 아닌, 컴퓨터에서 시뮬레이션 (예: AerSimulator) 을 할 때는 게이트를 행렬로 변환하여 상태 벡터에 곱해야 
        return np.array([
            [cos, -isin, 0, 0],
            [-isin, cos, 0, 0],
            [0, 0, cos, isin],
            [0, 0, isin, cos],
        ], dtype=dtype)

# Test the custom gate
theta = Parameter("theta")
custom_gate = RXZGate(theta)
print("Custom RXZGate created successfully!")
print(f"Gate name: {custom_gate.name}")
print(f"Number of qubits: {custom_gate.num_qubits}")
print(f"Parameters: {custom_gate.params}")
```

#### 5. Apply Pauli twirling to a circuit

무작위화를 사용하여 코히어런트 오류를 확률론적 오류로 형성하는 양자 오류 억제 기술로, 연산 전후에 무작위 Pauli 게이트를 적용한 다음 많은 논리적으로 동등한 회로의 결과를 결합합니다.

Qiskit 에는 특정 2 큐비트 게이트 주변에 pauli twirling 을 적용하는 데 사용할 수 있는 `pauli_twirl_2q_gates` 함수가 있습니다.

효과: 특정 방향의 시스템 오류 (일관성 오류) 가 평균화되어 사라지고, 남은 것은 단순한 무작위 오류뿐이 됩니다. 이렇게 하면 **오류율을 정확히 측정 (Randomized Benchmarking)**하거나 **오류를 보정 (Error Mitigation)**하기 훨씬 쉬워집니다.(추후에 오류보정, PEC)

qc.cx(0, 1)  # This might cancel with the first CX
qc.cx(1, 2)
qc.measure([0, 1, 2], [0, 1, 2])

print("Original circuit:")

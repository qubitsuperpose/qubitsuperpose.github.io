# 7. 결과 검색 및 분석

## 1. 실험 결과 검색 및 관리

이 노트북은 Qiskit Primitives 작업의 결과를 검색하고 관리하는 데 사용되는 객체와 메서드에 대한 실용적인 가이드를 제공합니다.

**다루는 주요 개념:**
- **목표 1:** `SamplerPubResult` 객체 이해하기
- **목표 2:** 작업 결과를 디스크에 저장하고 검색하기
- **목표 3 & 4:** `RuntimeJob` 및 `BasePrimitiveJob` 클래스의 속성과 메서드 탐색하기

모든 예시는 `AerSimulator` 를 사용하여 로컬에서 실행되도록 설계되었으며, 클라우드 특정 섹션은 명확하게 표시되어 있습니다.

**설정: 간단한 Sampler 작업 실행**

결과 객체를 탐색하기 위해 먼저 작업을 실행해야 합니다. 간단한 벨 회로를 생성하고 로컬 `BackendSamplerV2` 로 실행해 보겠습니다.

```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.primitives import BackendSamplerV2 as Sampler
import numpy as np
import time
import json

# Helper function to create a Bell state circuit
def create_bell_circuit():
    """Create a Bell state (maximally entangled state) circuit."""
    qc = QuantumCircuit(2)
    qc.h(0)  # Apply Hadamard to qubit 0
    qc.cx(0, 1)  # Apply CNOT with control=0, target=1
    return qc

# 1. Create a circuit and backend
circuit = create_bell_circuit()
circuit.measure_all()
local_backend = AerSimulator()

# 2. Instantiate a local Sampler and run a job
sampler = Sampler(backend=local_backend)
job = sampler.run([(circuit,)], shots=1024)

# 3. Get the result object
# For local jobs, this is usually instantaneous.
result = job.result()
print("Successfully ran a local Sampler job.")
print(f"Job ID: {job.job_id()}")
```

### 1. `SamplerPubResult` 이해하기

V2 Primitives 작업에서 결과를 가져오면 `PrimitiveResult` 객체를 받게 되는데, 이는 `PubResult` 객체들의 리스트입니다 (제출한 각 회로, 즉 "PUB" 당 하나씩). 단일 작업에 대한 `PubResult` 를 살펴보겠습니다.

#### 1. 속성: `data` 와 `metadata`

`PubResult` 는 두 가지 주요 속성을 가집니다:
- `.data`: 핵심 과학적 출력을 포함하는 객체 (예: Sampler 의 경우 비트문자열, Estimator 의 경우 기대값).
- `.metadata`: 작업이 어떻게 실행되었는지에 대한 정보를 포함하는 딕셔너리 (예: 샷 수).

```python
# Get the result for the first (and only) PUB
pub_result = result[0]

# Access the .data attribute
# For a Sampler, this is a DataBin object with a 'meas' field for measurements.
bitstring_data = pub_result.data.meas

# Access the .metadata attribute
metadata = pub_result.metadata

print(f"--- PubResult for Job '{job.job_id()}' ---")
print(f"\nMetadata: {metadata}")

# The data object has its own methods to access the results in different formats
print(f"\nFirst 5 Bitstrings: {bitstring_data.get_bitstrings()[:5]}")
print(f"\nCounts Dictionary: {bitstring_data.get_counts()}")
```

#### 2. 메서드: `join_data()` (이런 메서드는 없다)

단일 작업에서 여러 개의 PUB (여러 회로) 를 실행한 경우, `join_data()` 를 사용하여 결과를 결합할 수 있습니다.

```python
# Run a job with two different circuits
circuit1 = create_bell_circuit()
circuit1.measure_all()

circuit2 = create_bell_circuit()
circuit2.x(0)  # Add X gate to flip the state
circuit2.measure_all()

job_multi = sampler.run([(circuit1,), (circuit2,)], shots=512)
result_multi = job_multi.result()

print("Results from first circuit:")
print(f"  Counts: {result_multi[0].data.meas.get_counts()}")

print("\nResults from second circuit:")
print(f"  Counts: {result_multi[1].data.meas.get_counts()}")

# Note: join_data() is used to combine DataBin objects with the same structure
# For demonstration, we can access individual PubResults and their data separately
```


### 2. 작업 저장 및 검색

실제 하드웨어에서 장시간 실행되는 작업의 경우, 노트북에서 대기하고 싶지 않을 것입니다. 작업을 제출하고 그 ID 를 얻은 후 나중에 결과를 검색할 수 있습니다. 이 섹션에서는 결과를 디스크에 저장하고 로드하는 방법도 보여줍니다.

#### 1. 결과를 디스크에 저장하기

`PrimitiveResult` 객체를 나중에 분석할 수 있도록 JSON 파일로 쉽게 저장할 수 있습니다. 이는 결과를 공유하거나 작업을 다시 실행하지 않고도 후처리를 수행하는 데 유용합니다.

```python
# We need the Qiskit Runtime JSON encoder to handle the specific data types
from qiskit_ibm_runtime import RuntimeEncoder

# 'result' is the object we got from job.result() earlier
with open("sampler_result.json", "w") as file:
    json.dump(result, file, cls=RuntimeEncoder)

print("Result object successfully saved to 'sampler_result.json'")
```

#### 2. 디스크에서 결과 로드하기

```python
# Use the Qiskit Runtime JSON decoder to correctly reconstruct the objects
from qiskit_ibm_runtime import RuntimeDecoder

with open("sampler_result.json", "r") as file:
    loaded_result = json.load(file, cls=RuntimeDecoder)

print("Result object successfully loaded from 'sampler_result.json'")
print(f"\nLoaded Result Object Type: {type(loaded_result)}")

# You can now access the data just as before
print(f"\nCounts from loaded data: {loaded_result[0].data.meas.get_counts()}")
```


#### 3. 클라우드 서비스에서 작업 검색하기 (선택사항)

**참고:** 이 섹션은 구성된 `qiskit-ibm-runtime` 계정이 필요합니다. 계정이 없으면 이러한 셀에서 `AccountNotFoundError` 가 발생하며, 이는 예상된 동작입니다. `try...except` 블록이 이를 잡아내고 노트북이 계속 실행되도록 합니다.

```python
import datetime

try:
    from qiskit_ibm_runtime import QiskitRuntimeService
    
    # This line will only work if you have an account saved locally
    service = QiskitRuntimeService()
    
    three_months_ago = datetime.datetime.now() - datetime.timedelta(days=90)
    jobs_in_last_three_months = service.jobs(created_after=three_months_ago)
    
    print(f"Found {len(list(jobs_in_last_three_months))} jobs from the last 90 days.")
    if jobs_in_last_three_months:
        print("Showing the first three:")
        for i, job in enumerate(jobs_in_last_three_months[:3]):
            print(f"  {i+1}. Job {job.job_id()} - Status: {job.status()}")

except Exception as e:
    print(f"Cloud job retrieval skipped: {e}")
    print("This is expected if you don't have an IBM Quantum account configured.")
```

#### 4. ID 로 특정 작업 검색하기

작업의 ID 를 알고 있다면 직접 검색할 수 있습니다. 이는 작업을 제출하고 그 ID 를 저장한 후 나중에 결과를 가져올 때 유용합니다.

```python
try:
    from qiskit_ibm_runtime import QiskitRuntimeService
    
    service = QiskitRuntimeService()
    
    # Get the most recent successful job for demonstration
    successful_jobs = [j for j in service.jobs(limit=100) if j.status() == "DONE"]
    
    if successful_jobs:
        successful_job = successful_jobs[0]
        job_id = successful_job.job_id()
        print(f"Found successful job: {job_id}")
        
        # Retrieve the job by its ID
        retrieved_job = service.job(job_id)
        retrieved_result = retrieved_job.result()
        
        print(f"Successfully retrieved job {job_id}")
        print(f"Result: {retrieved_result}")
    else:
        print("No successful jobs found in recent history.")
        
except Exception as e:
    print(f"Job retrieval skipped: {e}")
    print("This is expected if you don't have an IBM Quantum account configured.")
```

### 3. `RuntimeJob` 과 `BasePrimitiveJob`

작업을 실행하면 `Job` 객체를 반환받습니다. 로컬 `BackendPrimitives` 의 경우 이는 `PrimitiveJob` 입니다. 런타임 서비스의 경우 `RuntimeJob` 입니다. 두 클래스 모두 `BasePrimitiveJob` 에서 상속받으며, 작업 상태를 관리하고 쿼리하기 위한 공통 메서드 세트를 공유합니다.

**중요 참고사항:** 일부 메서드는 `RuntimeJob` (클라우드 작업) 에서만 사용할 수 있으며, `PrimitiveJob` (로컬 작업) 에서는 사용할 수 없습니다. 어떤 것이 어떤 것인지 명확하게 표시해 두겠습니다.

#### 1. 작업 관리 예시

로컬 시뮬레이터에서 작업을 실행하고 작업 객체의 메서드를 사용하여 상태를 모니터링해 보겠습니다. 로컬 작업의 경우 이는 매우 빠르게 발생하지만, 실제 하드웨어에서는 작업이 `QUEUED` 및 `RUNNING` 상태에 잠시 머무를 수 있습니다.

```python
from qiskit.circuit.library import EfficientSU2
from qiskit.transpiler import generate_preset_pass_manager
from qiskit.providers.jobstatus import JobStatus

# Create a slightly more complex circuit to give the simulator a moment of work
circuit = EfficientSU2(10, reps=4, entanglement='linear')
circuit.measure_all()
params = np.random.rand(circuit.num_parameters)

# Transpile the circuit
pm = generate_preset_pass_manager(optimization_level=1, backend=local_backend)
isa_circuit = pm.run(circuit)

# Submit the job
job = sampler.run([(isa_circuit, params)])

print(f"Submitted job with ID: {job.job_id()}")

# --- Polling for Job Status ---
# This loop mimics how you would check on a long-running cloud job.
max_checks = 50
checks = 0
while not job.in_final_state() and checks < max_checks:
    # JobStatus is an Enum with values like QUEUED, RUNNING, DONE, ERROR, CANCELLED
    status = job.status()
    print(f"Job status is currently: {status}")
    time.sleep(0.1)  # Wait a moment before checking again
    checks += 1

final_status = job.status()
print(f"\nJob finished with final status: {final_status}")

# --- Accessing Results ---
if job.done():
    result = job.result()
    print("Job completed successfully!")
    print(f"Sample counts: {result[0].data.meas.get_counts()}")
else:
    # For local jobs, errors are raised immediately when calling result()
    # Cloud jobs have additional methods like errored() and error_message()
    status = job.status()
    print(f"Job is in state: {status}")
```

#### 2. 주요 작업 속성 및 메서드

`job` 객체에는 많은 유용한 메서드와 속성이 있습니다. 가장 중요한 것들은 다음과 같습니다:

**PrimitiveJob (로컬) 과 RuntimeJob (클라우드) 모두에서 사용 가능한 메서드:**
- `job.status()`: 작업 상태를 반환합니다 (예: `JobStatus.RUNNING`).
- `job.in_final_state()`: 작업이 완료, 오류 발생 또는 취소된 경우 `True` 를 반환합니다.
- `job.done()`: 작업이 성공적으로 완료된 경우에만 `True` 를 반환합니다.
- `job.running()`: 작업이 현재 실행 중인 경우 `True` 를 반환합니다.
- `job.cancelled()`: 작업이 취소된 경우 `True` 를 반환합니다.
- `job.result()`: 작업이 최종 상태가 될 때까지 실행을 **블록**한 후 `PrimitiveResult` 객체를 반환합니다.
- `job.job_id()`: 작업의 고유 문자열 ID 를 반환합니다.
- `job.cancel()`: 대기 중이거나 실행 중인 작업을 취소하려고 시도합니다.

**RuntimeJob (클라우드 작업) 에서만 사용 가능한 메서드:**
- `job.errored()`: 작업이 실패한 경우 `True` 를 반환합니다 (로컬 PrimitiveJob 에서는 사용 불가).
- `job.error_message()`: 작업이 실패한 경우 오류 메시지를 반환합니다.
- `job.backend()`: 백엔드 객체를 반환합니다 (모든 작업 유형에서 작동하지 않을 수 있음).
- `job.queue_info()`: 대기열 위치에 대한 정보.
- `job.queue_position()`: 대기열에서의 작업 위치.
- `job.logs()`: 작업 로그를 검색합니다.
- `job.metrics()`: 작업의 성능 메트릭.
- `job.update_tags()`: 작업 태그를 업데이트합니다.

**RuntimeJob 특정 속성:**
- `job.creation_date`
- `job.tags`
- `job.session_id`
- `job.usage_estimation`

```python
# --- Demonstration of job methods that work on ALL job types ---

# We can get the job ID
print(f"Job ID: {job.job_id()}")

# Check if the job is in a final state (should be True now)
print(f"Is job in a final state? {job.in_final_state()}")

# Check specific status flags (these work on all jobs)
print(f"Is job done? {job.done()}")
print(f"Is job running? {job.running()}")
print(f"Is job cancelled? {job.cancelled()}")

# Note: errored() is only available on RuntimeJob
# For local jobs, check status directly:
print(f"Job status: {job.status()}")
```

### 4. 요약

이 노트북에서는 다음 내용을 다루었습니다:

1. **PubResult 구조**: 결과를 포함하는 `data` 및 `metadata` 속성
2. **결과 저장/로드**: `RuntimeEncoder` 와 `RuntimeDecoder` 를 사용하여 결과를 디스크에 지속화하는 방법
3. **작업 검색**: IBM Quantum 에서 작업을 검색하는 방법 (클라우드 특정)
4. **작업 관리**: PrimitiveJob (로컬) 과 RuntimeJob (클라우드) 의 차이점 이해
   - 둘 다에서 작동하는 공통 메서드
   - 모니터링 및 오류 처리를 위한 클라우드 특정 메서드

이러한 도구들은 특히 작업이 장시간 대기열에 있을 수 있는 실제 양자 하드웨어를 사용할 때, 장시간 실행되는 양자 실험을 관리하는 데 필수적입니다.

## 2. 프로바이더 이해 및 작업 모니터링

이 노트북은 `Provider` , `Backend` 및 `Job` 클래스를 포함하는 Qiskit 프로바이더 모델에 대한 실용적인 가이드를 제공합니다.

**목표:** 이러한 구성 요소가 어떻게 함께 작동하는지 이해하기 위해 간단한 커스텀 인메모리 백엔드를 구축할 것입니다. 이 "가짜" 백엔드는 실제 양자 계산을 실행하지는 않지만 Qiskit 인터페이스를 올바르게 구현하여 작업을 제출하고 결과를 검색하는 전체 워크플로우를 볼 수 있게 합니다.

### 1. 프로바이더, 백엔드 및 작업 추상 클래스

Qiskit 프로바이더 모델은 Qiskit 이 다양한 실행 리소스 (백엔드) 와 상호 작용하는 방식을 정의하는 프레임워크입니다.

*   `Provider` : 진입점입니다. 백엔드 그룹에 대한 접근을 관리하고 제공합니다 (예: `qiskit-ibm-provider` 는 IBM 의 백엔드에 대한 접근을 제공합니다).
*   `BackendV2` : 백엔드를 위한 현대적인 인터페이스입니다. 백엔드의 기능 (그것의 `target` ) 을 설명하며 회로를 실행하기 위한 `.run()` 메서드를 가지고 있습니다.
*   `Job` : `backend.run()` 에 의해 반환되는 객체입니다. 제출된 작업을 나타내며 상태 (`.status()`) 를 모니터링하고 최종 결과 (`.result()`) 를 검색하는 데 사용됩니다.

#### 1. 단계 1: 커스텀 `Job` 구현

우리는 작업의 수명 주기를 시뮬레이션하는 간단한 `MyJob` 클래스를 생성하는 것으로 시작할 것입니다. 이는 짧은 시간 동안 "RUNNING" 상태인 척한 다음 미리 정의된 가짜 결과를 반환합니다.

```python
import time
import uuid
from qiskit.providers import JobV1 as Job, JobStatus, JobError
from qiskit.result import Result

class MyJob(Job):
    """A simple, in-memory job class for demonstration."""
    def __init__(self, backend, job_id, circuit):
        super().__init__(backend, job_id)
        self._status = JobStatus.QUEUED
        self._circuit = circuit
        self._result = self._create_fake_result()
        print(f"-> Job {self._job_id} created for circuit '{self._circuit.name}'.")

    def _create_fake_result(self):
        """Creates a static, fake result object."""
        fake_counts = {'00': 512, '11': 512} # Simulate a perfect Bell state
        results_dict = {
            'backend_name': self._backend.name,
            'backend_version': self._backend.version,
            'job_id': self._job_id,
            'qobj_id': str(uuid.uuid4()),
            'success': True,
            'results': [
                {
                    'shots': 1024,
                    'success': True,
                    'header': {'name': self._circuit.name},
                    'data': {'counts': fake_counts}
                }
            ]
        }
        return Result.from_dict(results_dict)

    def submit(self):
        """Simulates submitting the job. It will move to RUNNING then DONE."""
        self._status = JobStatus.RUNNING
        print(f"-> Job {self._job_id} is now {self.status().name}.")
        # Simulate some execution time
        time.sleep(0.5)
        self._status = JobStatus.DONE
        print(f"-> Job {self._job_id} is now {self.status().name}.")

    def result(self):
        """Return the fake result."""
        if self._status is not JobStatus.DONE:
            raise JobError("Job has not completed yet.")
        return self._result

    def status(self):
        """Return the current job status."""
        return self._status

print("MyJob class defined successfully!")
```


#### 2. 단계 2: 커스텀 `BackendV2` 구현

다음으로 `MyBackend` 를 생성할 것입니다. 그 주요 작업은 기능 ( `target` ) 과 우리의 `MyJob` 객체를 반환하는 `.run()` 메서드를 정의하는 것입니다.

**중요:** `BackendV2` 는 백엔드에 대한 기본 옵션을 정의하는 `_default_options()` 메서드를 구현해야 합니다.

```python
from qiskit.circuit import Measure, Reset, Parameter
from qiskit.circuit.library import SXGate, CXGate, UGate
from qiskit.providers import BackendV2, Options
from qiskit.transpiler import Target

class MyBackend(BackendV2):
    """A simple custom backend that returns our fake MyJob."""
    def __init__(self, provider=None):
        super().__init__(name="my_custom_backend", backend_version="1.0")
        self._provider = provider
        
        # --- Define the Backend's Target ---
        # The target describes exactly what gates and operations are supported.
        self._target = Target()
        theta = Parameter("theta")
        phi = Parameter("phi")
        lam = Parameter("lam")
        
        # Add supported instructions with their properties (on all qubits 0-4)
        self._target.add_instruction(UGate(theta, phi, lam), {(i,): None for i in range(5)})
        self._target.add_instruction(SXGate(), {(i,): None for i in range(5)})
        self._target.add_instruction(CXGate(), {(0, 1): None, (1, 2): None, (2, 3): None, (3, 4): None})
        self._target.add_instruction(Measure(), {(i,): None for i in range(5)})
        self._target.add_instruction(Reset(), {(i,): None for i in range(5)})
        
        print("-> MyBackend initialized.")

    @property
    def target(self):
        return self._target

    @property
    def max_circuits(self):
        return 1
    
    @classmethod
    def _default_options(cls):
        """Return default options for this backend."""
        return Options(shots=1024)

    def run(self, circuit, **kwargs):
        """'Run' a circuit by creating and submitting a MyJob instance."""
        job_id = str(uuid.uuid4())
        job = MyJob(backend=self, job_id=job_id, circuit=circuit)
        job.submit()
        return job

print("MyBackend class defined successfully!")
```


#### 3. 단계 3: 커스텀 `Provider` 구현

마지막으로 `MyProvider` 클래스는 백엔드를 얻기 위한 진입점 역할을 합니다. 우리는 더 이상 사용되지 않는 기본 클래스에서 상속하지 않고 간단한 프로바이더 클래스를 생성할 것입니다.

```python
class MyProvider:
    """A simple provider that gives access to MyBackend."""
    def __init__(self):
        self._backends = {"my_custom_backend": MyBackend(provider=self)}
        print("-> MyProvider initialized.")
    
    def get_backend(self, name=None):
        """Get a backend by name."""
        if name is None:
            # Return the first backend if no name specified
            return list(self._backends.values())[0]
        if name in self._backends:
            return self._backends[name]
        raise KeyError(f"Backend '{name}' not found")
        
    def backends(self, name=None, **kwargs):
        """Return the list of available backends."""
        if name:
            return [self._backends[name]] if name in self._backends else []
        return list(self._backends.values())

print("MyProvider class defined successfully!")
```

### 3. 세션 (런타임 특정 개념)

`Session` 은 여러 작업을 효율적으로 실행하기 위해 백엔드에 대한 전용 연결을 관리하는 `qiskit-ibm-runtime` 의 개념입니다.

#### 1. 주요 포인트:

- 세션은 IBM Quantum Runtime 서비스에 특정됩니다
- 여러 작업에 대해 백엔드 시간을 예약할 수 있습니다
- 동일한 세션의 후속 작업에 대한 대기열 시간을 줄입니다
- 우리의 커스텀 `BackendV2` 구현은 세션을 지원하지 않습니다 (런타임 서비스가 아닙니다)

#### 2. 세션 속성:
- `service` : QiskitRuntimeService 인스턴스
- `session_id` : 세션에 대한 고유 식별자

#### 3. 세션 메서드:
- `backend()` : 세션의 백엔드 얻기
- `cancel()` : 세션의 모든 작업 취소
- `close()` : 세션 닫기
- `details()` : 세션 세부 정보 얻기
- `from_id()` : 기존 ID 에서 세션 생성
- `status()` : 세션 상태 얻기
- `usage()` : 리소스 사용 통계 얻기

```python
# Example of how Sessions would be used (requires IBM Quantum account)
print("Example Session Usage (Cloud-only):")
print("="*60)
print("""
# With IBM Quantum Runtime:
from qiskit_ibm_runtime import QiskitRuntimeService, Session, SamplerV2

service = QiskitRuntimeService()
backend = service.backend("ibm_brisbane")

# Create a session to run multiple jobs
with Session(service=service, backend=backend) as session:
    sampler = SamplerV2(session=session)
    
    # Run multiple jobs within the same session
    job1 = sampler.run([circuit1])
    job2 = sampler.run([circuit2])
    job3 = sampler.run([circuit3])
    
    # Get results
    result1 = job1.result()
    result2 = job2.result()
    result3 = job3.result()
    
    # Check session info
    print(f"Session ID: {session.session_id}")
    print(f"Session status: {session.status()}")
    print(f"Session usage: {session.usage()}")

# Session automatically closes when exiting the 'with' block
""")
```

# 5. 양자 변분 회로와 양자 신경망

이번 강의에서는 데이터 분류 작업을 위한 여러 변분 양자 회로, 이른바 변분 양자 분류기(variational quantum classifiers, VQC)를 구현합니다. 한때는 VQC의 하위 집합을 고전 신경망에 빗대어 양자 신경망(quantum neural networks, QNN)이라고 부르는 것이 일반적이었습니다. 실제로 합성곱 층(convolution layers)과 같이 고전 신경망에서 차용한 구조가 VQC에서 중요한 역할을 하는 경우도 있습니다. 이처럼 유사성이 강한 경우에는 QNN이라는 표현이 유용한 설명이 될 수 있습니다. 하지만 매개변수화된 양자 회로(parameterized quantum circuit)가 반드시 신경망의 일반적인 구조를 따를 필요는 없습니다. 예를 들어, 모든 데이터를 첫 번째(입력) 층에 적재할 필요는 없습니다. 첫 번째 층에서 일부 데이터를 적재하고, 몇 개의 게이트를 적용한 뒤 추가 데이터를 적재할 수도 있습니다(이러한 과정을 데이터 "리업로딩(reuploading)"이라고 부릅니다). 따라서 QNN은 매개변수화된 양자 회로의 하위 집합으로 생각해야 하며, 고전 신경망과의 유사성에 얽매여 유용한 양자 회로를 탐구하는 데 제약을 받아서는 안 됩니다.

이번 강의에서 다루는 데이터셋은 수평 및 수직 줄무늬를 포함하는 이미지들로 구성되어 있으며, 우리의 목표는 처음 보는 이미지들을 선의 방향에 따라 두 범주 중 하나로 분류하는 것입니다. 우리는 이를 VQC를 통해 수행할 것입니다. 진행하면서 계산을 개선하고 확장할 수 있는 방법들도 다룰 것입니다. 여기서 다루는 데이터셋은 고전적인 방법으로 매우 쉽게 분류할 수 있습니다. 이는 문제의 양자적인 부분에 집중하고, 데이터셋의 속성이 양자 회로의 어떤 부분으로 변환될 수 있는지를 살펴보기 위해 단순함을 기준으로 선택된 것입니다. 고전 알고리즘이 이렇게 효율적으로 작동하는 단순한 경우에 양자 속도 향상을 기대하는 것은 타당하지 않습니다.

이번 강의를 마치면 다음을 할 수 있어야 합니다:

* 이미지의 데이터를 양자 회로에 적재하기
* VQC(또는 QNN)를 위한 앤자츠(ansatz)를 구성하고, 문제에 맞게 조정하기
* VQC/QNN을 훈련시키고 이를 사용하여 테스트 데이터에 대한 정확한 예측 수행하기
* 문제를 확장하고, 현재 양자 컴퓨터의 한계를 인식하기


## 1. 데이터 생성

데이터를 구성하는 것부터 시작하겠습니다. 데이터 세트는 종종 Qiskit 패턴 프레임워크의 일부로 명시적으로 생성되지는 않습니다. 그러나 데이터 유형과 준비는 양자 컴퓨팅을 머신 러닝에 성공적으로 적용하는 데 매우 중요합니다. 아래 코드는 설정된 픽셀 크기를 가진 이미지 데이터 세트를 정의합니다. 이미지의 한 전체 행 또는 열에 값 $\pi/2$가 할당되고, 나머지 픽셀에는 구간 $(0,\pi/4)$에서 무작위 값이 할당됩니다. 이 무작위 값들이 우리 데이터의 노이즈입니다. 이미지가 어떻게 생성되는지 이해했는지 확인하기 위해 코드를 훑어보세요. 나중에 이미지의 크기를 확장할 것입니다.


```python
# This code defines the images to be classified:

import numpy as np

# Total number of "pixels"/qubits
size = 8
# One dimension of the image (called vertical, but it doesn't matter). Must be a divisor of `size`
vert_size = 2
# The length of the line to be detected (yellow). Must be less than or equal to the smallest
# dimension of the image (`<=min(vert_size,size/vert_size)`
line_size = 2


def generate_dataset(num_images):
    images = []
    labels = []
    hor_array = np.zeros((size - (line_size - 1) * vert_size, size))
    ver_array = np.zeros((round(size / vert_size) * (vert_size - line_size + 1), size))

    j = 0
    for i in range(0, size - 1):
        if i % (size / vert_size) <= (size / vert_size) - line_size:
            for p in range(0, line_size):
                hor_array[j][i + p] = np.pi / 2
            j += 1

    # Make two adjacent entries pi/2, then move down to the next row. Careful to avoid the "pixels"
    # at size/vert_size - linesize, because we want to fold this list into a grid.

    j = 0
    for i in range(0, round(size / vert_size) * (vert_size - line_size + 1)):
        for p in range(0, line_size):
            ver_array[j][i + p * round(size / vert_size)] = np.pi / 2
        j += 1

    # Make entries pi/2, spaced by the length/rows, so that when folded,
    # the entries appear on top of each other.

    for n in range(num_images):
        rng = np.random.randint(0, 2)
        if rng == 0:
            labels.append(-1)
            random_image = np.random.randint(0, len(hor_array))
            images.append(np.array(hor_array[random_image]))

        elif rng == 1:
            labels.append(1)
            random_image = np.random.randint(0, len(ver_array))
            images.append(np.array(ver_array[random_image]))
            # Randomly select 0 or 1 for a horizontal or vertical array, assign the corresponding
            # label.

        # Create noise
        for i in range(size):
            if images[-1][i] == 0:
                images[-1][i] = np.random.rand() * np.pi / 4
    return images, labels


hor_size = round(size / vert_size)
```

위 코드는 이미지에 세로선(+1) 또는 가로선(-1)이 포함되어 있는지 여부를 나타내는 레이블을 생성했음을 참고하세요. 이제 sklearn을 사용하여 100개의 이미지로 구성된 데이터 세트를 학습 세트와 테스트 세트로 분할합니다(해당 레이블도 함께). 여기서는 데이터 세트의 70%를 학습에 사용하고 나머지 30%는 테스트에 사용합니다.

```python
from sklearn.model_selection import train_test_split

train_images, test_images, train_labels, test_labels = train_test_split(
    images, labels, test_size=0.3, random_state=246
)
```

## 2. 변분 양자 분류기: 첫 번째 시도

### 2.1 Qiskit 패턴 1단계: 문제를 양자 회로에 매핑하기

목표는 데이터 벡터/이미지 $\vec{x}$를 올바른 범주에 매핑하는 매개변수 $\theta$를 가진 함수 $f$를 찾는 것이다: $f_\theta(\vec{x}) \rightarrow \pm1$. 이는 서로 다른 목적을 가진 적은 층으로 구성된 VQC를 사용하여 달성될 것이다:

<p>
$$
f_\theta(\vec{x}) = \langle 0|U^{\dagger}(\vec{x})W^\dagger(\theta)OW(\theta)U(\vec{x})|0\rangle
$$
</p>

여기서 $U(\vec{x})$는 인코딩 회로로, 이전 수업에서 보았듯이 다양한 선택지가 있다. $W(\theta)$는 변분적이거나 학습 가능한 회로 블록이며, $\theta$는 학습될 매개변수 집합이다. 이 매개변수들은 양자 회로가 이미지를 가장 잘 분류하는 매개변수 집합을 찾기 위해 고전적 최적화 알고리즘에 의해 변화될 것이다. 이 변분 회로는 때때로 "안자츠(ansatz)"라고 불린다. 마지막으로, $O$는 Estimator 프리미티브를 사용하여 추정될 어떤 관측가능량이다. 층들이 반드시 이 순서로 와야 하거나, 심지어 완전히 분리되어야 한다는 제약은 없다. 기술적으로 동기부여되는 어떤 순서로든 여러 변분 및/또는 인코딩 층을 가질 수 있다.

데이터를 인코딩할 특징 맵(feature map)을 선택하는 것으로 시작한다. 다른 특징 매핑에 비해 회로 깊이를 낮게 유지하기 때문에 `z_feature_map`을 사용할 것이다.

```python
from qiskit.circuit.library import z_feature_map

# One qubit per data feature
num_qubits = len(train_images[0])

# Data encoding
# Note that qiskit orders parameters alphabetically. We assign the parameter prefix "a" to ensure
# our data encoding goes to the first part of the circuit, the feature mapping.
feature_map = z_feature_map(num_qubits, parameter_prefix="a")
```

다음은 원문에 충실하게 번역한 한국어 버전입니다. 수식은 그대로 유지하였습니다.

---

이제 훈련할 ansatz를 결정해야 한다. ansatz를 선택할 때 고려해야 할 사항이 많다. 완전한 설명은 이 입문의 범위를 벗어나므로, 여기서는 몇 가지 고려사항의 범주만 간략히 언급하고자 한다.

1. **하드웨어:** 모든 현대적인 양자 컴퓨터는 고전 컴퓨터에 비해 오류에 취약하고 노이즈에 민감하다. 과도하게 깊은 ansatz(특히 트랜스파일된 이중 큐비트 깊이 측면에서)를 사용하면 좋은 결과를 얻을 수 없다. 관련된 문제로, 양자 컴퓨터는 어떤 큐비트 배치를 가지고 있어서, 일부 물리적 큐비트는 양자 컴퓨터 상에서 인접해 있고 다른 큐비트들은 서로 매우 멀리 떨어져 있을 수 있다는 점이 있다. 인접한 큐비트를 얽히게 하는 것은 깊이를 너무 많이 증가시키지 않지만, 매우 먼 큐비트를 얽히게 하면 깊이가 상당히 증가할 수 있는데, 이는 정보를 얽히게 하기 위해 인접해 있어야 하는 큐비트로 이동시키기 위해 스왑 게이트를 삽입해야 하기 때문이다.
2. **문제:** 문제에 대해 ansatz를 안내할 수 있는 어떤 정보가 있다면, 그것을 활용하라. 예를 들어, 이 수업의 데이터는 수평선과 수직선의 이미지로 구성되어 있다. 인접한 색상/값들 사이의 어떤 상관관계가 수평선 또는 수직선의 이미지를 식별하는지 생각해볼 수 있다. 인접한 픽셀들 사이의 이러한 상관관계에 해당하는 ansatz의 특성은 무엇일까? 이 점은 이 수업의 뒷부분에서 더 기술적으로 다시 살펴볼 것이다. 하지만 지금은 단순히 인접한 픽셀에 해당하는 큐비트들 사이에 얽힘과 CNOT 게이트를 포함하는 것이 좋은 아이디어처럼 보인다고 말하자. 더 큰 그림에서, 문제가 실제로 양자 회로를 사용하여 가장 잘 해결되는지, 아니면 고전 알고리즘으로도 똑같이 잘 수행할 수 있는지 여부를 고려하라.
3. **매개변수의 수:** 회로에서 각각 독립적으로 매개변수화된 양자 게이트는 고전적으로 최적화해야 할 공간을 증가시키며, 이로 인해 수렴이 더 느려진다. 하지만 문제의 규모가 커지면, *barren plateaus*를 마주칠 수 있다. 이 용어는 문제의 크기가 증가함에 따라 변분 양자 알고리즘의 최적화 경관이 지수적으로 평평하고 특징 없어지는 현상을 가리킨다. 이는 기울기가 사라지게 하여 알고리즘을 효과적으로 훈련하기 어렵게 만든다[\[1\]](#references). barren plateaus는 VQC/QNN과 같은 변분 양자 알고리즘과 관련이 있다. 매개변수의 수 증가가 barren plateaus를 피하는 데 있어 유일한 고려사항은 아니라는 점에 유의해야 한다; 다른 고려사항으로는 전역 비용 함수와 무작위 매개변수 초기화가 포함된다.

이 수업에서는 ansatz 구성의 좋은 관행에 대한 몇 가지 간단한 예를 살펴볼 것이다. 먼저 아래의 ansatz를 시도해보자. 나중에 다시 돌아와 수정할 것이다.


```python
# Import the necessary packages
from qiskit import QuantumCircuit
from qiskit.circuit import ParameterVector

# Initialize the circuit using the same number of qubits as the image has pixels
qnn_circuit = QuantumCircuit(size)

# We choose to have two variational parameters for each qubit.
params = ParameterVector("θ", length=2 * size)

# A first variational layer:
for i in range(size):
    qnn_circuit.ry(params[i], i)

# Here is a list of qubit pairs between which we want CNOT gates.
# The choice of these is not yet obvious.
qnn_cnot_list = [[0, 1], [1, 2], [2, 3]]

for i in range(len(qnn_cnot_list)):
    qnn_circuit.cx(qnn_cnot_list[i][0], qnn_cnot_list[i][1])

# The second variational layer:
for i in range(size):
    qnn_circuit.rx(params[size + i], i)

# Check the circuit depth, and the two-qubit gate depth
print(qnn_circuit.decompose().depth())
print(
    f"2+ qubit depth: {qnn_circuit.decompose().depth(lambda instr: len(instr.qubits) > 1)}"
)

# Draw the circuit
qnn_circuit.draw("mpl")
```

데이터 인코딩과 변분 회로가 준비되면, 이들을 결합하여 완전한 앙상블을 구성할 수 있습니다. 이 경우, 양자 회로의 구성 요소는 신경망의 구성 요소와 매우 유사하며, $U(\vec{x})$는 이미지에서 입력값을 불러오는 레이어와 가장 유사하고, $W(\theta)$는 가변적인 "가중치" 레이어와 유사합니다. 이러한 유사성 때문에 일부 명명 규칙에서 "qnn"을 채택했지만, 이 유사성이 VQC 탐구에 제한을 두어서는 안 됩니다.

```python
# QNN ansatz
ansatz = qnn_circuit

# Combine the feature map with the ansatz
full_circuit = QuantumCircuit(num_qubits)
full_circuit.compose(feature_map, range(num_qubits), inplace=True)
full_circuit.compose(ansatz, range(num_qubits), inplace=True)
```

이제 관측가능량(observable)을 정의해야 합니다. 이를 비용 함수(cost function)에서 사용할 수 있도록 하기 위함입니다. 우리는 Estimator를 사용하여 이 관측가능량의 기댓값(expectation value)을 얻을 것입니다. 만약 문제에 잘 부합하는 좋은 안자츠(ansatz)를 선택했다면, 각 큐비트는 분류에 관련된 정보를 담고 있을 것입니다. 정보를 더 적은 수의 큐비트로 결합하는 레이어(*합성곱 레이어(convolutional layer)*라고 불림)를 추가하여, (합성곱 신경망(convolutional neural network)에서처럼) 회로 내 큐비트의 일부에서만 측정이 필요하도록 할 수도 있습니다. 또는 각 큐비트로부터 어떤 속성을 측정할 수도 있습니다. 여기서는 후자를 선택하여, 각 큐비트에 대해 `Z` 연산자를 포함시킬 것입니다. $Z$를 선택하는 데 특별히 유일한 이유는 없지만, 다음과 같은 타당한 근거가 있습니다:

* 이것은 이진 분류(binary classification) 작업이며, $Z$의 측정은 두 가지 가능한 결과를 낼 수 있습니다.
* $Z$의 고유값(eigenvalue) ($\pm 1$)은 상당히 잘 분리되어 있으며, 그 결과 estimator의 결과값은 구간 \[-1, +1] 안에 놓이게 되고, 여기서 0을 단순히 컷오프(cutoff) 값으로 사용할 수 있습니다.
* 추가적인 게이트 오버헤드 없이 파울리 Z 기저(Pauli Z basis)에서 측정하는 것은 간단합니다.

따라서 Z는 매우 자연스러운 선택입니다.

```python

```

양자 회로와 추정하고자 하는 관측량이 준비되었습니다. 이제 이 회로를 실행하고 최적화하기 위해 몇 가지가 필요합니다. 먼저 순방향 전달을 실행하는 함수가 필요합니다. 아래 함수는 `input_params`와 `weight_params`를 각각 입력으로 받습니다. `input_params`는 이미지 데이터를 설명하는 정적 매개변수 집합이고, `weight_params`는 최적화할 가변 매개변수 집합입니다.


```python
from qiskit.primitives import BaseEstimatorV2
from qiskit.quantum_info.operators.base_operator import BaseOperator


def forward(
    circuit: QuantumCircuit,
    input_params: np.ndarray,
    weight_params: np.ndarray,
    estimator: BaseEstimatorV2,
    observable: BaseOperator,
) -> np.ndarray:
    """
    Forward pass of the neural network.

    Args:
        circuit: circuit consisting of data loader gates and the neural network ansatz.
        input_params: data encoding parameters.
        weight_params: neural network ansatz parameters.
        estimator: EstimatorV2 primitive.
        observable: a single observable to compute the expectation over.

    Returns:
        expectation_values: an array (for one observable) or a matrix (for a sequence of observables) of expectation values.
        Rows correspond to observables and columns to data samples.
    """
    num_samples = input_params.shape[0]
    weights = np.broadcast_to(weight_params, (num_samples, len(weight_params)))
    params = np.concatenate((input_params, weights), axis=1)
    pub = (circuit, observable, params)
    job = estimator.run([pub])
    result = job.result()[0]
    expectation_values = result.data.evs

    return expectation_values
```

### 2.2 손실 함수

다음으로, 레이블의 예측 값과 계산된 값 간의 차이를 계산하기 위한 손실 함수가 필요합니다. 이 함수는 알고리즘에 의해 예측된 레이블과 올바른 레이블을 가져와 평균 제곱 차이를 반환합니다. 다양한 손실 함수가 있습니다. 여기서 MSE는 우리가 선택한 예입니다.

```python
def mse_loss(predict: np.ndarray, target: np.ndarray) -> np.ndarray:
    """
    Mean squared error (MSE).

    prediction: predictions from the forward pass of neural network.
    target: true labels.

    output: MSE loss.
    """
    if len(predict.shape) <= 1:
        return ((predict - target) ** 2).mean()
    else:
        raise AssertionError("input should be 1d-array")
```

또한, 고전적인 최적화 알고리즘에서 사용할 수 있도록 변수 매개변수(가중치)의 함수인 약간 다른 손실 함수를 정의해 보겠습니다. 이 함수는 입력으로 초기 매개변수만 받으며, 순방향 전달 및 손실에 사용되는 다른 변수들은 전역 매개변수로 설정됩니다. 최적화 알고리즘은 다양한 가중치를 샘플링하고 비용/손실 함수의 출력값을 낮추는 방식으로 모델을 학습시킵니다.


```python
def mse_loss_weights(weight_params: np.ndarray) -> np.ndarray:
    """
    Cost function for the optimizer to update the ansatz parameters.

    weight_params: ansatz parameters to be updated by the optimizer.

    output: MSE loss.
    """
    predictions = forward(
        circuit=circuit,
        input_params=input_params,
        weight_params=weight_params,
        estimator=estimator,
        observable=observable,
    )

    cost = mse_loss(predict=predictions, target=target)
    objective_func_vals.append(cost)

    global iter
    if iter % 50 == 0:
        print(f"Iter: {iter}, loss: {cost}")
    iter += 1

    return cost
```

## 6. 모델 개선하기
### 6.1 1단계 다시 살펴보기

문제를 양자 회로로 매핑할 때, 인접한 픽셀의 정보가 어떻게 클래스를 결정하는지에 대해 명시적으로 생각했어야 합니다. 수평선을 식별하기 위해서는, 각 행에 걸쳐 있는 모든 픽셀에 대해 "픽셀 $i$가 노란색이면, 픽셀 $i+1$도 노란색인가"를 알고 싶습니다. 우리는 또한 수직선에 대해서도 알고 싶습니다. 하지만 분류가 이진(binary)이므로, 그러한 수평선이 감지되지 *않는다*면 그것은 수직선이라고 단순히 말할 수 있다고 상상해볼 수 있습니다. 우리의 이전 변분 회로는 큐비트(따라서 픽셀) 0과 1, 1과 2, 2와 3 사이에 CNOT 게이트를 포함하고 있었습니다. 이는 이미지 상단을 가로지르는 모든 수평선을 다루지만, 수직선을 직접적으로 감지하지는 못하며, 아래쪽 행을 무시하기 때문에 수평선도 완전히 감지하지 못합니다. 모든 수평선을 완전히 감지하려면, 큐비트(픽셀) 4와 5, 5와 6, 6과 7 사이에도 유사한 CNOT 게이트 세트가 필요할 것입니다. 수직선에 해당하는 큐비트들(예: 0과 4, 또는 2와 6) 사이에 CNOT 게이트를 추가하는 것도 유용할 수 있다는 점을 염두에 둘 수 있습니다. 하지만 먼저 수평선이 *있는지* *없는지*를 감지하는 것으로 충분한지 확인해보겠습니다.

```python
# Initialize the circuit using the same number of qubits as the image has pixels
qnn_circuit = QuantumCircuit(size)

# We choose to have two variational parameters for each qubit.
params = ParameterVector("θ", length=2 * size)

# A first variational layer:
for i in range(size):
    qnn_circuit.ry(params[i], i)

# Here is an extended list of qubit pairs between which we want CNOT gates. This now covers all
# pixels connected by horizontal lines.
qnn_cnot_list = [[0, 1], [1, 2], [2, 3], [4, 5], [5, 6], [6, 7]]

for i in range(len(qnn_cnot_list)):
    qnn_circuit.cx(qnn_cnot_list[i][0], qnn_cnot_list[i][1])

# The second variational layer:
for i in range(size):
    qnn_circuit.rx(params[size + i], i)

# Check the circuit depth, and the two-qubit gate depth
print(qnn_circuit.decompose().depth())
print(
    f"2+ qubit depth: {qnn_circuit.decompose().depth(lambda instr: len(instr.qubits) > 1)}"
)

# Combine the feature map and variational circuit
ansatz = qnn_circuit

# Combine the feature map with the ansatz
full_circuit = QuantumCircuit(num_qubits)
full_circuit.compose(feature_map, range(num_qubits), inplace=True)
full_circuit.compose(ansatz, range(num_qubits), inplace=True)
```

```python
# This code defines the images to be classified:

import numpy as np

# Total number of "pixels"/qubits
size = 36
# One dimension of the image (called vertical, but it doesn't matter). Must be a divisor of `size`
vert_size = 6
# The length of the line to be detected (yellow). Must be less than or equal to the smallest
# dimension of the image (`<=min(vert_size,size/vert_size)`
line_size = 6


def generate_dataset(num_images):
    images = []
    labels = []
    hor_array = np.zeros((size - (line_size - 1) * vert_size, size))
    ver_array = np.zeros((round(size / vert_size) * (vert_size - line_size + 1), size))

    j = 0
    for i in range(0, size - 1):
        if i % (size / vert_size) <= (size / vert_size) - line_size:
            for p in range(0, line_size):
                hor_array[j][i + p] = np.pi / 2
            j += 1

    # Make two adjacent entries pi/2, then move down to the next row. Careful to avoid the "pixels"
    # at size/vert_size - linesize, because we want to fold this list into a grid.

    j = 0
    for i in range(0, round(size / vert_size) * (vert_size - line_size + 1)):
        for p in range(0, line_size):
            ver_array[j][i + p * round(size / vert_size)] = np.pi / 2
        j += 1

    # Make entries pi/2, spaced by the length/rows, so that when folded,
    # the entries appear on top of each other.

    for n in range(num_images):
        rng = np.random.randint(0, 2)
        if rng == 0:
            labels.append(-1)
            random_image = np.random.randint(0, len(hor_array))
            images.append(np.array(hor_array[random_image]))
            # Randomly select one of the several rows you made above.
        elif rng == 1:
            labels.append(1)
            random_image = np.random.randint(0, len(ver_array))
            images.append(np.array(ver_array[random_image]))
            # Randomly select one of the several rows you made above.

        # Create noise
        for i in range(size):
            if images[-1][i] == 0:
                images[-1][i] = np.random.rand() * np.pi / 4
    return images, labels


hor_size = round(size / vert_size)
```

### 9.1 마무리

이번 실습에서는 **양자 신경망(Quantum Neural Network, QNN)** 을 사용하여 이미지를 이진 분류(binary classification)하는 전체 워크플로우를 학습했습니다. 각 Qiskit 패턴 단계에서 고려해야 할 핵심 사항은 다음과 같습니다.

**1단계:** 문제를 양자 회로(quantum circuit)로 매핑하기

* 학습 데이터를 불러옵니다. 이는 직접("by hand") 수행하거나 `z_feature_map`과 같은 미리 구현된 특징 맵(feature map)을 사용할 수 있습니다.
* 문제에 적합한 회전(rotation) 레이어와 얽힘(entanglement) 레이어를 포함하는 ansatz를 구성합니다.
* 양자 컴퓨터에서 양질의 결과를 얻을 수 있도록 회로의 깊이(circuit depth)를 지속적으로 확인합니다.

**2단계:** 양자 실행을 위해 문제 최적화하기

* 백엔드(backend)를 선택하며, 일반적으로 가장 작업량이 적은(least busy) 백엔드를 선택합니다.
* 패스 매니저(pass manager)를 사용하여 회로와 관측량(observables)을 모두 선택한 백엔드의 아키텍처에 맞게 트랜스파일(transpile)합니다.
* 회로가 매우 깊거나 넓은 경우에는 여러 번 트랜스파일을 수행한 후, 가장 얕은(shallowest) 회로를 선택합니다.

**3단계:** Qiskit (Runtime) Primitives를 사용하여 실행하기

* 시뮬레이터에서 예비 실험을 수행하여 ansatz를 디버깅하고 최적화합니다.
* IBM® 양자 컴퓨터에서 실행합니다.

**4단계:** 후처리(Post-process) 및 고전적(classical) 형식으로 결과 반환하기

* 학습 데이터와 테스트 데이터에 대해 모델의 정확도(model accuracy)를 계산합니다.
* 고전적 최적화(classical optimization)의 수렴(convergence) 과정을 모니터링합니다.

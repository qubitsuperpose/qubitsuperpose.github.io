## 1. 유틸리티 논문 소개
이 강의에서는, 우리가 비공식적으로 ["the utility paper"](https://www.nature.com/articles/s41586-023-06096-3)라고 부르는 Nature Vol 618, 2023년 6월 15일에 출판된 논문에 등장하는 유틸리티 규모 회로를 실행합니다. 이 논문은 2D 횡장 이싱 모델의 시간 진화를 다룹니다. 특히, 그들은 다음과 같은 해밀토니안의 시간 역학을 고려합니다.
<p>
$$
H = H_{ZZ} + H_X = - J \sum_{(i,j)} Z_i Z_j + h \sum_{i} X_i
$$
</p>

여기서 $J > 0$는 $i < j$인 최근접 이웃 스핀들의 결합 상수이고, $h$는 전체 횡장입니다.

그들은 초기 상태로부터 스핀 동역학을 1차 Trotter 분해를 이용해 시간-진화 연산자의 근사로 시뮬레이션한다.
<p>
$$
\begin{aligned}
 \exp(-i H_{ZZ} \delta t) &= \prod_{(i,j)} \exp (i J \delta t Z_i Z_j)  = \prod_{(i,j)} \mathrm{R}_{Z_i Z_j} ( - 2 J \delta t) \\
 \exp(-i H_X \delta t) &= \prod_{i} \exp (-i h \delta t X_i )  = \prod_{i} \mathrm{R}_{X_i} ( 2 h \delta t)
\end{aligned}
$$
</p>

여기서 진화 시간 $T$은 $T / \delta t$ 개의 Trotter 단계로 이산화되며, $$\mathrm{R}_{Z_i Z_j}(\theta_J)$$와 $$\mathrm{R}_{X_i}(\theta_h)$$는 각각 $ZZ$와 $X$ 회전 게이트이다.

그들은 IBM Quantum® Eagle 프로세서(127-큐비트 장치, heavy-hex 연결성)에서 실험을 수행했으며, 모든 큐비트에 $X$ 상호작용을, 결합 맵의 모든 에지에 $ZZ$ 상호작용을 적용했다. 모든 $ZZ$ 상호작용은 “데이터 의존성” 때문에 동시에 적용될 수 없다. 따라서 그들은 **결합 맵을 색칠하여(color the coupling map)** 레이어로 그룹화했다. 같은 레이어에 속한 것들은 동일한 색을 가지며, 병렬로 적용될 수 있다.

또한, 실험의 단순화를 위해 $\theta_J = -\pi / 2$인 경우에 초점을 맞췄다.

이 논문의 새로운 기여는 statevector 시뮬레이션을 넘어서는 규모의 양자 회로를 구축하고, 노이즈가 있는 양자 컴퓨터에서 이를 실행한 후 신뢰할 수 있는 결과를 추출하는 데 성공했다는 것이다. 즉, 노이즈가 있는 양자 컴퓨터의 실용성을 입증했다. 이 과정에서 그들은 노이즈가 있는 장치의 오류를 완화하기 위해 확률적 오류 증폭(PEA)을 이용한 zero-noise extrapolation(ZNE)을 적용했다.

이후로 우리는 이러한 실험과 회로를 “utility-scale”이라고 부른다.

### 2.2 결합 맵 색상 지정

백엔드의 결합 맵에 색상을 지정하는 방법을 살펴보겠습니다. 이는 $ZZ$개의 상호작용을 레이어로 그룹화하는 데 필요합니다.

먼저 백엔드의 결합 맵을 시각화해 보겠습니다. 현재 모든 IBM Quantum 장치의 결합 맵은 육각형 구조를 띠고 있습니다.

```python
backend = service.least_busy(operational=True, simulator=False)

backend.coupling_map.draw()
```

![](https://quantum.cloud.ibm.com/assets-docs-learning/_next/image?url=%2Flearning%2Fimages%2Fcourses%2Futility-scale-quantum-computing%2Futility-i%2Fextracted-outputs%2F84e11b8e-bc74-4f3a-8538-d4d34d069e0f-0.avif&w=3840&q=75)

결합 맵에 색을 입히기 위해 그래프 및 복잡한 네트워크 작업을 위한 Python 패키지인 [rustworkx](https://www.rustworkx.org/)를 사용합니다. 이 패키지는 여러 가지 색칠 알고리즘을 제공하지만, 모두 휴리스틱 알고리즘이므로 최소 색칠을 보장하지는 않습니다.

하지만 헤비헥스 그래프는 이분 그래프이므로, 이러한 그래프에 대해 최소 색칠을 찾을 수 있는 [graph_bipartite_edge_color](https://www.rustworkx.org/apiref/rustworkx.graph_bipartite_edge_color.html) 패키지를 사용합니다.


```python
def color_coupling_map(backend):
    graph = backend.coupling_map.graph
    undirected_graph = graph.to_undirected(multigraph=False)
    edge_color_map = rx.graph_bipartite_edge_color(undirected_graph)
    if edge_color_map is None:
        edge_color_map = rx.graph_greedy_edge_color(undirected_graph)
    # build a map from color to a list of edges
    edge_index_map = undirected_graph.edge_index_map()
    color_edges_map = {color: [] for color in edge_color_map.values()}
    for edge_index, color in edge_color_map.items():
        color_edges_map[color].append(
            (edge_index_map[edge_index][0], edge_index_map[edge_index][1])
        )
    return edge_color_map, color_edges_map
```

육각형 구조가 복잡한 그래프는 세 가지 색으로 칠해야 합니다. 위의 결합 맵을 통해 이를 확인해 보겠습니다.

```python
edge_color_map, color_edges_map = color_coupling_map(backend)
print(
    f"{backend.name}, {backend.num_qubits}-qubit device, {len(color_edges_map.keys())} colors assigned."
)
```


## 3. 2차원 이징 모델의 트로터화된 시간 진화 문제를 해결합니다.

2차원 이징 모델의 시간 진화에 대한 실용신안 회로를 구성하는 루틴을 정의해 보겠습니다. 이 루틴은 백엔드, 트로터 단계 수를 나타내는 정수, 그리고 장벽 삽입 여부를 제어하는 ​​불리언 값, 이렇게 세 가지 매개변수를 받습니다.

```python
def get_utility_circuit(backend, num_steps: int, barrier: bool = False):
    num_qubits = backend.num_qubits
    _, color_edges_map = color_coupling_map(backend)
    θ_h = Parameter("$\\theta_h$")
    qc = QuantumCircuit(num_qubits)

    for i in range(num_steps):
        qc.rx(θ_h, range(num_qubits))

        for _, edge_list in color_edges_map.items():
            for edge in edge_list:
                qc.append(rzz, edge)

        if barrier:
            qc.barrier()
    return qc
```

이미 구성된 회로에 대해 큐비트 매핑과 라우팅을 수동으로 수행했음을 알려드립니다. 따라서 나중에 회로를 트랜스파일링할 때 트랜스파일러에게 큐비트 매핑과 라우팅을 다시 수행하도록 요청하지 않습니다(요청해서는 안 됩니다). 곧 보시겠지만, 최적화 수준은 1로, 레이아웃 방식은 "trivial"로 설정하여 트랜스파일링을 실행합니다.

다음으로, 구성된 회로에 대한 정보를 빠르게 확인하기 위한 간단한 루틴을 정의합니다.

```python
def get_circuit_info(qc: QuantumCircuit, reps: int = 0):
    qc0 = qc.decompose(reps=reps)
    return (
        f"{qc0.num_qubits} qubits × {qc0.depth(lambda x: x.operation.num_qubits == 2)} layers ({qc0.depth()}-depth)"
        + ",  "
        + f"""Gate breakdown: {", ".join([f"{k.upper()} {v}" for k, v in qc0.count_ops().items()])}"""
    )
```

이 루틴들을 실행해 보겠습니다. 27큐비트 × 15층(트로터 단계 5회)의 회로가 보일 것입니다. 가짜 장치는 28개의 에지를 가지고 있으므로 28*5개의 얽힘 게이지가 있어야 합니다.

## 4. 27큐비트 버전 문제 해결

이제 소규모 실험 버전을 시연해 보겠습니다. 관측량으로 $\langle Z_{13} \rangle$을 사용하는 27큐비트 $\times$ 6층(트로터 단계 2개) 회로를 구축하고, AerSimulator와 가상 장치 모두에서 실행합니다.

물론, 저희는 매핑, 최적화, 실행, 후처리로 구성된 4단계 워크플로인 "Qiskit 패턴"을 따릅니다. 좀 더 구체적으로 설명하면 다음과 같습니다.

* 고전 입력값을 양자 연산에 매핑합니다.

* 양자 연산에 맞게 회로를 최적화합니다.

* 기본 연산을 사용하여 회로를 실행합니다.

* 후처리를 수행하고 결과를 고전 형식으로 반환합니다.

다음에서는 소규모 실험을 위한 회로 생성 단계인 매핑 과정을 보여줍니다. 그다음에는 AerSimulator와 가상 장치에 대한 최적화 및 실행 과정을 각각 보여줍니다. 마지막으로, 후처리 단계에서 결과를 그래프로 나타냅니다.

여러분의 결과가 그림 4b의 "보완되지 않은" 결과와 유사한가요? 실험 당시의 장치와 상태에 따라 결과는 매우 다를 수 있습니다. 결과 자체는 걱정하지 마세요. 우리가 확인할 것은 여러분이 코딩을 올바르게 수행했는지 여부입니다. 만약 올바르게 수행했다면, 축하합니다. 여러분은 실용화 시대의 출발선에 서게 된 것입니다.

실용화 논문에서 언급했듯이, 전 세계 과학자들은 잡음이 존재하는 상황에서도 의미 있는 결과를 도출하기 위해 엄청난 창의력을 발휘해 왔습니다. 이러한 공동 노력의 최종 목표는 양자 우위, 즉 양자 컴퓨터가 기존 컴퓨터보다 산업 현장에서 유용한 문제들을 더 빠르고, 더 높은 정확도로, 또는 더 저렴하게 해결할 수 있는 상태를 의미합니다. 이는 단번에 이루어지는 사건이 아니라, 양자 결과를 기존 컴퓨터로 재현하는 데 걸리는 시간이 점차 길어지다가 결국 양자 우위가 매우 중요해지는 시대가 도래할 것입니다. 양자 우위에 대해 한 가지 분명한 것은, 우리는 실용화 규모의 실험을 통해서만 그 목표를 달성할 수 있다는 것입니다. 이 강좌를 통해 여러분이 도전과 재미로 가득한 이 여정에 동참하게 된다면, 우리는 더할 나위 없이 기쁠 것입니다.

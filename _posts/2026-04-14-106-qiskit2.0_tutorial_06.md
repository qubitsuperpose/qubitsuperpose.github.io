# 6. Use Estimator Primitive

## 1. Estimator 옵션 설정

개요: 이 노트북은 실행 동작 및 오류 완화 기술을 제어하는 다양한 estimator 옵션을 다룹니다.

```python
# Setup: Import necessary libraries
from qiskit_ibm_runtime import QiskitRuntimeService, EstimatorV2 as Estimator, EstimatorOptions as Options
from qiskit import QuantumCircuit
from qiskit.quantum_info import SparsePauliOp
from qiskit_ibm_runtime.options import MeasureNoiseLearningOptions,LayerNoiseLearningOptions,ZneOptions

print("Libraries imported successfully.")
```

### 1. Estimator 옵션

`Options` 클래스는 EstimatorV2 를 위해 `Estimator` 실행에 대한 다양한 구성 설정을 제공합니다. 이는 성능, 정확성 및 리소스 사용에 영향을 미칩니다.

#### 1. 속성

| 속성 | 설명 | 참고 |
|-----------|-------------|----------------|
| `_VERSION` | 내부 버전 추적 | |
| `max_execution_time` | 작업 실행 최대 시간 (초) | |
| `environment` | 실행 환경 메타데이터 | `EnvironmentOptions` 객체 |
| `simulator` | 시뮬레이터 특정 옵션 | `{'noise_model': None, 'seed_simulator': None}` |
| `default_precision` | 기댓값 목표 정밀도 | Float 기본값은 `0.015625` (1/√4096) |
| `default_shots` | 회로 실행 기본 횟수 | 정수 , 기본 값 없음 |
| `resilience_level` | 오류 완화 수준 | `0` (없음) 부터 `2` (중간) |
| `seed_estimator` | 재현 가능한 결과를 위한 랜덤 시드 | 정수 |
| `dynamical_decoupling` | 동적 디커플링 활성화/구성 | `DynamicalDecouplingOptions` 객체 |
| `resilience` | 고급 복원력 구성 | `Options.ResilienceOptionsV2` 객체 |
| `execution` | 실행 특정 설정 | `ExecutionOptionsV2` 객체 |
| `twirling` | 게이트 트월링 구성 | `Options.TwirlingOptions` 객체 |
| `experimental` | 실험적 기능 | 실험적 설정 딕셔너리 |


```python
# Example: Configuring Estimator Options

# Initialize service (assuming credentials is already saved)
service = QiskitRuntimeService()
backend = service.least_busy(operational=True, simulator=False)

# Create a simple test circuit
qc = QuantumCircuit(2)
qc.h(0)
qc.cx(0, 1)
observable = SparsePauliOp("ZZ")

# Create Options object with custom settings
options = Options()

# Set basic execution parameters
options.max_execution_time = 300
options.default_shots = 2048 
options.default_precision = 0.01
options.seed_estimator = 12345

# Set resilience level (0=none, 1=basic, 2=moderate, 3=maximum)
options.resilience_level = 2  # moderate error mitigation

print(f"Max execution time: {options.max_execution_time} seconds")
print(f"Default shots: {options.default_shots}")
print(f"Default precision: {options.default_precision}")
print(f"Resilience level: {options.resilience_level}\n")

# Create Estimator with custom options
estimator = Estimator(mode=backend, options=options)

print("Updating Options\n")

# Create initial options
initial_options = Options()
initial_options.default_shots = 1024
initial_options.max_execution_time = 300

print(f"Initial shots: {initial_options.default_shots}")
print(f"Initial max time: {initial_options.max_execution_time}")

# Update options with new values
initial_options.update(default_shots=4096, max_execution_time=900)

print(f"Updated shots: {initial_options.default_shots}")
print(f"Updated max time: {initial_options.max_execution_time}\n")
```


#### 2. 메서드

**update**

여러 옵션을 한 번에 업데이트합니다. 이는 Estimator 인스턴스를 생성한 후 옵션을 수정하거나 구성 변경을 적용할 때 유용합니다.

```python
options = Options()
options.update(
    default_shots=4096,
    max_execution_time=1200,
    resilience_level=2
)
```


### 2. Twirling 옵션

Twirling(또는 무작위 컴파일링) 은 서로 다르지만 동등한 게이트 시퀀스를 평균화하여 결합 오류를 확률적 오류로 변환하는 기술로, 이는 완화하기 더 쉽습니다.

| 속성 | 설명 | 참고 |
|-----------|-------------|----------------|
| `enable_gates` | 2-qubit 클리포드 게이트 트월링 활성화 여부 | bool, 기본값 `False` |
| `enable_measure` | 측정 명령어에 트월링 활성화 여부 | `True` for Estimator, `False` for Sampler |
| `num_randomizations` | 트월링 시 사용할 무작위 샘플 수 | 정수 또는 'auto' |
| `shots_per_randomization` | 각 무작위 샘플당 실행할 샷 수. | 정수 또는 'auto' |
| `strategy` | 큐비트 트월링 전략 | `active`, `active-circuit`, `active-accum`, `all`, 기본값 `active-accum` |



### 3. Resilience 옵션

Resilience 옵션은 `EstimatorV2` 의 오류 완화 기술을 제어합니다. 이러한 설정은 노이즈가 있는 하드웨어에서 양자 계산의 정확성을 향상시키는 데 도움이 됩니다.

**속성**

| 속성 | 설명 | 참고 |
|-----------|-------------|----------------|
| `measure_mitigation` | 측정 오류 수정 | 기본값 `True` |
| `measure_noise_learning` | 측정 노이즈 모델 학습 | `MeasureNoiseLearningOptions` 객체 |
| `zne_mitigation` | 제로 노이즈 외삽법 활성화 | 기본값 `False` |
| `zne` | ZNE 특정 구성 | `ZneOptions` 객체 |
| `pec_mitigation` | 확률적 오류 취소 활성화 | 기본값 `False` |
| `pec` | PEC 특정 구성 | `PecOptions` |
| `layer_noise_learning` | 계층별 노이즈 학습 | `LayerNoiseLearningOptions` 객체 |
| `layer_noise_model` | 사용자 정의 계층 노이즈 모델 | `NoiseLearnerResult` 또는 `Sequence[LayerError]` |

```python
# Example: Configuring Resilience Options

# Method 1: Using resilience_level
levels = [0, 1, 2]
descriptions = [
    "No error mitigation",
    "Minimal measurement error mitigation",
    "Medium mitigation"
]

for level, desc in zip(levels, descriptions):
    options = Options()
    options.resilience_level = level
    print(f"Level {level}: {desc}")
    
# Method 2: Control with Resilience Options

options = Options()

# Configure individual resilience techniques
# Basic readout correction
options.resilience.measure_mitigation = True
# Learn measurement noise
options.resilience.measure_noise_learning = MeasureNoiseLearningOptions()
# Enable Zero-Noise Extrapolation
options.resilience.zne_mitigation = True
# Disable PEC (computationally expensive)
options.resilience.pec_mitigation = False
# Learn layer-dependent noise
options.resilience.layer_noise_learning = LayerNoiseLearningOptions() 

# Configure ZNE specifically
options.resilience.zne = ZneOptions(noise_factors=[1.0, 2.0, 3.0], extrapolator="exponential")

print("configured resilience settings:")
print(f"  Measurement mitigation: {options.resilience.measure_mitigation}")
print(f"  ZNE mitigation: {options.resilience.zne_mitigation}")
print(f"  PEC mitigation: {options.resilience.pec_mitigation}")
print(f"  ZNE noise factors: {options.resilience.zne.noise_factors}")
print(f"  ZNE extrapolator: {options.resilience.zne.extrapolator}\n")


# Phase 1: minimal resilience
quick_test = Options()
quick_test.resilience_level = 0
quick_test.default_shots = 256
print(f"  Resilience: {quick_test.resilience_level} (none)")
print(f"  Shots: {quick_test.default_shots}")

# Phase 2: moderate resilience
moderate = Options()
moderate.resilience_level = 1
moderate.default_shots = 1024
print(f"  Resilience: {moderate.resilience_level} (basic)")
print(f"  Shots: {moderate.default_shots}")

# Phase 3: High resilience
high = Options()
high.resilience_level = 2
high.default_shots = 4096
print(f"  Resilience: {high.resilience_level} (advanced)")
print(f"  Shots: {high.default_shots}")
```


### 4. ZNE 옵션

Zero-Noise Extrapolation (ZNE) 은 다음과 같은 오류 완화 기술입니다:
1. 증폭된 노이즈 수준에서 회로를 실행합니다.
2. 서로 다른 노이즈 팩터에서 결과를 측정합니다.
3. 제로 노이즈 한계로 외삽합니다.

**속성**

| 속성 | 설명 | 옵션 |
|-----------|-------------|---------|
| `amplifier` | 노이즈 증폭 방법 | `gate_folding`, `gate_folding_front`, `gate_folding_back`, `pea` |
| `noise_factors` | 노이즈 증폭 수준 | List of floats (예: `[1.0, 2.0, 3.0]`) |
| `extrapolator` | 외삽 방법 | `linear`, `exponential`, `double_exponential`, `polynomial_degree_1`, `polynomial_degree_2`, `polynomial_degree_3`, `polynomial_degree_4`, `polynomial_degree_5`, `polynomial_degree_6`, `polynomial_degree_7`, `fallback` |
| `extrapolated_noise_factors` | 피팅에 사용 | noise_factors 에서 파생 |

```python
# Example: Configuring ZNE Options

print("=== Zero-Noise Extrapolation (ZNE) Configuration ===\n")

# First, enable ZNE at the resilience level
options = Options()
options.resilience.zne_mitigation = True

# Configure ZNE parameters
print("ZNE Configuration Examples:\n")

# Basic ZNE with gate folding
options.resilience.zne.amplifier = "gate_folding"
options.resilience.zne.noise_factors = [1.0, 2.0, 3.0]
options.resilience.zne.extrapolator = "linear"

print(f"Case 1:")
print(f"  Amplifier: {options.resilience.zne.amplifier}")
print(f"  Noise factors: {options.resilience.zne.noise_factors}")
print(f"  Extrapolator: {options.resilience.zne.extrapolator}")

# Advanced ZNE for better accuracy
options.resilience.zne.amplifier = "gate_folding"
options.resilience.zne.noise_factors = [1.0, 1.5, 2.0, 3.0, 4.0]
options.resilience.zne.extrapolator = "exponential"

print(f"Case 2:")
print(f"  Amplifier: {options.resilience.zne.amplifier}")
print(f"  Noise factors: {options.resilience.zne.noise_factors}")
print(f"  Extrapolator: {options.resilience.zne.extrapolator}")

# Create ZNE configuration
zne_config = Options()

# Enable and configure ZNE
zne_config.resilience.zne_mitigation = True
zne_config.resilience.zne.amplifier = "gate_folding"
zne_config.resilience.zne.noise_factors = [1.0, 2.0, 3.0, 4.0]
zne_config.resilience.zne.extrapolator = "polynomial_degree_1"  # Polynomial extrapolation


# Set shots and execution time for ZNE
zne_config.default_shots = 2048
zne_config.max_execution_time = 600  # 10 minutes

print("Complete ZNE Configuration:")
print(f"  ZNE enabled: {zne_config.resilience.zne_mitigation}")
print(f"  Amplifier: {zne_config.resilience.zne.amplifier}")
print(f"  Noise factors: {zne_config.resilience.zne.noise_factors}")
print(f"  Extrapolator: {zne_config.resilience.zne.extrapolator}")
print(f"  Shots per factor: {zne_config.default_shots}")
print(f"  Total estimated shots: {zne_config.default_shots * len(zne_config.resilience.zne.noise_factors)}")
print(f"  Max execution time: {zne_config.max_execution_time} seconds")
```

## 2. 추정기(Estimator) 이론적 배경 이해

**개요:** 이 노트북은 추정기 프리미티브를 사용할 때 적용할 수 있는 다양한 오류 완화 및 오류 억제 기법을 다룹니다.
* 오류 억제 대 오류 완화
* 동적 디커플링 (Dynamical Decoupling)
* 파울리 트위링 (Pauli Twirling)
* 트위링된 판독 오류 소거 (Twirled readout error extinction)
* 제로 노이즈 외삽법 (Zero noise extrapolation)
* 확률적 오류 증폭 (Probabilistic error amplification)
* 확률적 오류 취소 (Probabilistic error cancellation)

```python
# Setup: Import necessary libraries
from qiskit_ibm_runtime import QiskitRuntimeService
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import EstimatorV2 as Estimator

print("Libraries imported successfully.")
```

### 1. 프리미티브 이해

프리미티브는 일반적인 양자 컴퓨팅 작업을 추상화하여 양자 알고리즘 개발을 단순화하는 표준화된 인터페이스입니다. Qiskit 은 두 가지 주요 프리미티브를 제공합니다:
1. 샘플러 (Sampler): 양자 회로로부터 측정 결과를 샘플링합니다.
2. 추정기 (Estimator): 관측 가능량의 기댓값을 계산합니다.
프리미티브에 대한 자세한 내용은 이전 섹션 4 및 5 의 노트북 4.2, 5.1 및 5.2 에서 다루었습니다.

### 2. 오류 완화 및 억제

* **오류 완화 (Error Mitigation)**: 결과의 오류를 줄이기 위한 후처리 기법
* **오류 억제 (Error Suppression)**: 회로 실행 중 오류를 방지하기 위해 적용하는 기법

```python
# if account is already saved, this will load it
service = QiskitRuntimeService()

backend = service.least_busy(operational=True, simulator=False)
```

#### 1. 동적 디커플링 (Dynamical Decoupling)

동적 디커플링은 대기 중인 큐비트에 펄스 시퀀스를 삽입하여 큐비트 간의 원치 않는 상호작용을 상쇄시키는 오류 억제 기법입니다. 이 기법은 일부 큐비트에 연산이 없는 간격 (gap) 이 있는 회로에서 주로 유용합니다.

회로에 많은 연산이 포함된 경우, 동적 디커플링을 추가하면 성능이 오히려 저하될 수 있음을 유의하십시오.

```python
# enable dynamical_decoupling
estimator = Estimator(mode=backend)
estimator.options.dynamical_decoupling.enable = True
estimator.options.dynamical_decoupling.scheduling_method = "asap"
estimator.options.dynamical_decoupling.sequence_type = "XpXm"
```


#### 2. 파울리 트위링 (Pauli Twirling)

트위링은 무작위화를 사용하여 코히어런트 오류를 확률적 오류로 변환하는 오류 억제 기법입니다. 파울리 트위링은 파울리 연산을 사용하는 특수한 유형의 트위링입니다.

이는 일반적으로 오류를 억제하려는 게이트 (주로 2-큐비트 게이트) 앞뒤에 무작위 파울리 게이트를 추가함으로써 구현됩니다.

```python
# enable twirling
estimator = Estimator(mode=backend)
estimator.options.twirling.enable_gates = True
estimator.options.twirling.num_randomizations = 32
estimator.options.twirling.shots_per_randomization = 100
estimator.options.twirling.strategy = "all"
```

#### 3. 트위링된 판독 오류 소거 (TREX)

TREX 는 트위링과 판독 오류 보정을 결합한 측정 오류 완화 기법입니다. 이는 측정 게이트를 파울리 X 게이트로 대체하고, 측정을 고전적 비트 플립 게이트로 대체하는 방식으로 작동하며, 이 시퀀스는 노이즈가 없는 측정과 동일합니다.

판독 오류가 발생할 경우, TREX 는 판독 오류 행렬을 대각화하여 역행렬을 쉽게 구할 수 있도록 합니다. 추가적인 보정 회로를 추가해야 하므로 회로에 작은 오버헤드가 추가됩니다.

```python
# enable TREX
estimator = Estimator(mode=backend)
estimator.options.resilience.measure_mitigation = True
estimator.options.resilience.measure_noise_learning.num_randomizations = 32
estimator.options.resilience.measure_noise_learning.shots_per_randomization = 100
```

#### 4. 제로-노이즈 외삽법 (ZNE)

제로-노이즈 외삽법 (ZNE) 은 인위적으로 증가된 노이즈 수준에서 회로를 실행하여 노이즈가 없는 결과를 추정하는 오류 완화 기법입니다. 결과는 서로 다른 노이즈 증폭 인자에서 측정된 후, 수학적 모델을 사용하여 제로-노이즈 한계로 외삽하여 이상적인 결과를 추정합니다.

일반적인 노이즈 증폭 방법에는 게이트 폴딩 (gate folding) 과 PEA(확률적 오류 증폭) 가 있습니다. 외삽에는 선형, 지수 또는 다항식 피팅을 사용할 수 있습니다.

```python
# enable ZNE
estimator = Estimator(mode=backend)
estimator.options.resilience.zne_mitigation = True
estimator.options.resilience.zne.noise_factors = (1, 3, 5)
estimator.options.resilience.zne.extrapolator = "exponential"
```

#### 5. 확률적 오류 증폭 (PEA)

확률적 오류 증폭 (PEA) 은 제로-노이즈 외삽법 (ZNE) 에서 사용되는 노이즈 증폭 방법으로, 게이트 폴딩보다 더 정확합니다. 이는 세 단계로 구성됩니다:
1. 학습: 회로의 각 엔탱글링 게이트 레이어에 대한 트위링된 노이즈 모델을 `LayerNoiseLearningOptions` 를 사용하여 학습합니다.
2. 노이즈 증폭: 회로를 서로 다른 노이즈 인자에서 여러 번 실행합니다.
3. 외삽: 노이즈가 있는 기댓값 결과를 제로-노이즈 한계로 외삽하여 이상적인 결과를 추정합니다.

PEA 는 ZNE 를 위한 노이즈 증폭 방법이므로, PEA 를 사용하려면 ZNE 가 활성화되어 있어야 합니다.

```python
# enable PEA
estimator = Estimator(mode=backend)
estimator.options.resilience.zne_mitigation = True
estimator.options.resilience.zne.amplifier = "pea"
```

#### 6. 확률적 오류 취소 (PEC)

확률적 오류 취소 (PEC) 는 관측 가능량의 기댓값을 추정할 때 사용할 수 있는 오류 완화 기법입니다. 이는 타겟 회로를 노이즈가 있는 회로들의 선형 결합으로 표현함으로써 수행됩니다. 이들은 준확률 분포 (quasi-probability distribution) 를 형성하며, 모든 노이즈가 있는 기댓값을 결합함으로써 이상적인 결과를 계산할 수 있습니다.

PEC 는 샘플링 오버헤드가 회로 깊이에 따라 지수적으로 증가하므로 계산 비용이 높습니다.
`max_overhead` 를 사용하여 오버헤드를 제한할 수 있습니다.

```python
# enable PEC
estimator = Estimator(mode=backend)
estimator.options.resilience.pec_mitigation = True
estimator.options.resilience.pec.max_overhead = 100
```

### 3. 요약

이 노트북에서 다룬 내용:

**추정기 이론 이해:**

1. **오류 억제** 는 회로 실행 중 오류를 방지하기 위해 적용되는 반면, **오류 완화** 는 처리 후 결과의 오류를 줄이기 위해 적용됩니다.
2. **동적 디커플링** 은 대기 중인 큐비트에 펄스 시퀀스를 삽입하여 원치 않는 상호작용을 상쇄합니다.
3. **파울리 트위링** 은 무작위 파울리 게이트를 추가하여 코히어런트 오류를 확률적 오류로 변환합니다.
4. **트위링된 판독 오류 소거** 는 트위링과 판독 오류 보정을 결합하여 판독 오류를 더 쉽게 보정할 수 있도록 합니다.
5. **제로 노이즈 외삽법** 은 회로를 증폭된 노이즈 수준에서 실행하고 제로 노이즈로 외삽하여 노이즈가 없는 결과를 추정합니다.
6. **확률적 오류 증폭** 은 제로 노이즈 외삽법에서 사용되는 노이즈 증폭 방법으로, 노이즈 모델 학습, 증폭, 그리고 외삽의 세 단계로 수행됩니다.
7. **확률적 오류 취소** 는 타겟 회로를 노이즈가 있는 회로들의 선형 결합으로 표현하고, 이러한 노이즈 회로들의 기댓값을 결합하여 이상적인 결과를 얻습니다.

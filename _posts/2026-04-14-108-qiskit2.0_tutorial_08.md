# 8. Operate with OpenQASM

## 1. QASM 타입과 구조

**개요:**

**QASM** (Quantum Assembly Language)은 양자 회로와 연산을 기술하기 위해 설계된 저수준 프로그래밍 언어입니다. Qiskit과 같은 고수준 양자 프로그래밍 언어와 실제 양자 하드웨어 사이의 중간 표현 역할을 합니다.

이 노트북에서는 QASM에서 사용할 수 있는 타입들과 이들 사이의 다양한 캐스팅 규칙을 다룹니다.

### 1. QASM 타입과 캐스팅

#### 1. 식별자(Identifiers)

* **시작 문자**: $\[A-Za-z\]$, 언더스코어(_) 또는 유니코드 문자 카테고리 Lu/Ll/Lt/Lm/Lo/Nl에 속하는 문자
* **연속 문자**: 시작 문자에 더해 10진수 숫자 $\[0-9\]$ 사용 가능
* **예약된 식별자**: 예약된 키워드나 내장 함수를 오버라이드할 수 없음
* **예시**:
    - 유효: `myQubit`, `_temp`, `θ` (그리스 문자 theta), `qubit1`, `result_2`
    - 무효: `qubit@1`, `1qubit`, `qubit`(예약어)

#### 2. 변수(Variables)

* 식별자 규칙에 따라 이름 지음
* 프로그램 내에서 값을 할당할 수 있음
* 클래식 타입은 선언 시 초기화 가능
* 변수 선언과 초기화는 한 번에 하나씩만 수행해야 함

**양자 변수 선언 예시**
```
// Declares 3 single qubits named q0 , q1 and q2
qubit q0;
qubit q1;
qubit q2;
```

**클래식 변수 선언 및 초기화 예시**
```
// 32-bit signed integer, uninitialized
int[32] a;
// 32-bit floating point, initialized to 5.5
float[32] b = 5.5;
// 3-bit register, uninitialized
bit[3] c;
// Boolean variable initialized to false
bool my_bool = false;
```

### 2. 양자 타입(Quantum Types)

#### 1. 큐비트(Qubits)

* **단일 큐비트**: `qubit`
* **큐비트 레지스터**: `qubit[size]` (큐비트 배열)
* **Deprecated**: `qreg`은 이전 버전 호환성을 위해 포함되어 있으며 향후 제거될 예정
* **전역**: 양자 변수는 항상 전역임

**상수 크기를 가진 큐비트 레지스터 예시**

```
include "stdgates.inc";

// Declares 5-qubit register called q1
qubit[5] q1;
// Compile-time constant
const uint SIZE = 4;
uint runtime_u = 2;
// Declares 4-qubit register using constant
qubit[SIZE] q2;

// Apply X gate to first qubit of q1
x q1[0];                  
// Apply Z gate to third qubit of q2 (4-2=2)
z q2[SIZE - 2];
// Apply X gate to third qubit of q1 (index 2)
x q1[runtime_u];

```

#### 2. 물리적 큐비트(Physical Qubits)

* **문법**: `$0`, `$1`, `$2` 등으로 참조
* **선언 불가**: 물리적 큐비트는 하드웨어에 의해 미리 정의됨
* **전역**: 프로그램 전체에서 접근 가능
* **목적**: 양자 하드웨어의 물리적 큐비트를 직접 주소 지정

**물리적 큐비트 주소 지정 예시**

```
// Virtual qubit named gamma
qubit gamma;
// Virtual qubit using Greek letter (same as gamma)
qubit γ;
qubit[20] qubit_array;
// Apply CX gate between physical qubits 0 and 1 (not virtual)
CX $0, $1;
```

### 3. 클래식 스칼라 타입(Classical scalar types)

#### 1. 클래식 비트와 레지스터(Classical Bits and Registers)

* **단일 비트**: `bit`
* **비트 레지스터**: `bit[size]`
* **Deprecated**: `creg`은 향후 제거될 예정

**비트 연산 예시**

```
// 20-bit register
bit[20] bit_array;
// 8-bit register initialized with bit string
bit[8] name = "00001111";
```

#### 2. 정수(Integers)

* **부호 있는**: `int[size]` 또는 `int` (구현에 따라 크기 정의)
* **부호 없는**: `uint[size]` 또는 `uint`
* **비트 연산**: 명시적인 너비(width) 지정이 필요

**정수 연산 및 캐스팅 예시**
```
// 32-bit unsigned integer
uint[32] my_uint = 10;
// 16-bit signed integer
int[16] my_int;
// Explicit cast from uint[32] to int[16]
my_int = int[16](my_uint);
```

#### 3. 부동소수점 수(Floating Point Numbers)

* **크기 지정**: `float[size]`
* **하드웨어 의존**: `float` (대상 아키텍처에 따라 정밀도가 달라짐)

**부동소수점 값 예시**

```
// 32-bit float initialized with pi constant
//float[32] my_float = pi;
float[32] my_float = π; //(π는 문서용 표기인 듯)위와 같이 해야

// Machine-precision float
float my_machine_float = 2.3;
```

#### 4. Void 타입

* **용도**: 값을 반환하지 않는 함수의 반환 타입으로 사용

#### 5. 각도(Angles)

* **형식**: `angle[size]` (1 unit = 2π / 2^size)
* **산술**: 내부적으로 부호 없는 정수 산술을 사용

**각도 표현 예시**

```
// 20-bit angle representing π/2
angle[20] my_angle = π / 2;
// Machine-precision angle
angle my_machine_angle;

// Binary: "1000" (4-bit representation of π)
// 1 unit = 2π/2^4 = 2π/16 = π/8
// π = 8 units = "1000" 
angle[4] my_pi = π;

// Binary: "010000" (6-bit representation of π/2)
// 1 unit = 2π/2^6 = 2π/64 = π /32
// π/2 =16 units = "010000" 
angle[6] my_pi_over_two = π/2;

// Binary: "01110000" (8-bit representation)
// 1 unit = 2π/2^8 = 2π/256 = π/128
// 7*π/8 = 112 untis = "001110000"
angle[8] my_angle = 7 * (π / 8);
```

#### 6. 복소수(Complex Numbers)

* **형식**: `complex[float[size]]`
* **허수 단위**: `im` 접미사 (예: `3.5im`)
* **추출**: `real()`과 `imag()` 함수 사용
* **하드웨어 지원**: 실제 양자 하드웨어에서는 지원되지 않을 수 있음

**복소수 연산 예시**

```
// 64-bit complex number
complex[float[64]] c;
// Assign real and imaginary parts
c = 2.5 + 3.5im;
// Complex expression
complex[float] d = 2.0+sin(π/2) + (3.1 * 5.5 im);
// Extract real part: ~3.0
float d_real = real(d);
// Extract imaginary part: ~17.05
float d_imag = imag(d);
```

#### 7. 불리언 타입(Boolean Types)

* **값**: `true` 또는 `false`
* **캐스팅**: `bit`에서 `bool()`을 사용해 변환 가능

**불리언 연산 예시**

```
bit my_bit = 0;
bool my_bool;
// Convert bit to boolean (0 → false)
my_bool = bool(my_bit);
// Direct assignment
my_bool = true;
```

### 4. 컴파일 타임 상수(Compile Time Constants)

* **키워드**: `const`
* 타입의 너비(width)를 지정할 때 반드시 필요
* 모든 스칼라 리터럴은 암시적으로 const임

**유효한 상수 사용 예시**

```
const uint SIZE = 32;
// 32-qubit register (uses constant)
qubit[SIZE] q1;
// 32-bit integer array
int[SIZE] i1;
```

**런타임 크기 사용 무효 예시**

```
// Runtime variable (not const)
uint runtime_size = 32;
// runtime_size is not a `const` type.
qubit[runtime_size] q2;
int[runtime_size] i2;
```

**상수 표현식 예시**

```
// Constant float
const float[64] f1 = 2.5;
uint[8] runtime_u = 7;

// Valid: f1 is constant, i1 = 2 (truncated)
const int[8] i1 = int[8](f1);
// Valid: u1 = 4 (2 * 2.5 = 5, cast to uint = 4)
const uint u1 = 2 * uint(f1);

// Invalid: float[64] cannot be cast to bit[2]
const bit[2] b1 = bit[2](f1);
// Invalid: runtime_u is not constant
const int[16] i2 = int[16](runtime_u);
```

#### 1. 내장 상수(Built-in Constants)

* **수학 상수**: `π` (pi), `τ` (tau = 2π), `e` (오일러 상수)

#### 2. 내장 상수 표현식 함수(Built-in Constant Expression Functions)

* **삼각함수**: `cos`, `sin`, `tan`, `arccos`, `arcsin`, `arctan`
* **반올림**: `ceiling`, `floor`
* **지수**: `exp`, `log`
* **거듭제곱/나머지**: `mod`, `pow`, `sqrt`
* **비트 조작**: `popcount` (설정된 비트 수), `rotl`, `rotr` (좌/우 회전)


### 5. 리터럴(Literals)

* **정수**:
  - 10진수: `123`, `456`
  - 16진수: `0x1F`, `0xFF`, `0x1_F` (가독성을 위한 언더스코어)
  - 8진수: `0o77`
  - 2진수: `0b1010`, `0B1100`, `0b1_0101_1010`
  - 큰 값: `1_000_000` (가독성을 위한 언더스코어)

* **부동소수점**:
  - 10진수: `3.14`, `.5`, `2.`
  - 과학적 표기: `1e10`, `2e+1`, `2.0E-1`, `3.14e0`

* **불리언**: `true`, `false`

* **비트 문자열**: `"0101"`, `"1111_0000"` (언더스코어 허용)

* **타이밍**:
  - `10ns` (나노초)
  - `2μs` 또는 `2us` (마이크로초)
  - `5ms` (밀리초)
  - `1s` (초)
  - `100dt` (장치별 타이밍 단위)

### 6. 배열(Arrays)

* **정적 크기**: 크기는 컴파일 타임에 결정
* **문법**: `array[base_type, dimension1, dimension2, ...]`
* **제한**: 함수나 게이트 내부에서 선언할 수 없음
* **최대 차원**: 7차원

**배열 선언 및 접근 예시**

```
// 1D array of 5 integers
array[int[32], 5] myArray = {0, 1, 2, 3, 4};
// 3x2 2D array  
array[float[32], 3, 2] multiDim = {
    {1.1, 1.2}, 
    {2.1, 2.2}, 
    {3.1, 3.2}
};

// Access first element: 0
int[32] firstElem = myArray[0];
// Access last element: 4
int[32] lastElem = myArray[4];
// Negative indexing: also 4
int[32] alsoLastElem = myArray[-1];
// Access element at [1,0]: 2.1
float[32] row2col1 = multiDim[1, 0];
```

### 7. 타이밍(Timing)

#### 1. Duration

* **목적**: 시간 간격을 표현
* **단위**: `ns`, `μs`/`us`, `ms`, `s`, `dt` (장치별)
* **연산**: 덧셈, 뺄셈, 비교 가능

**Duration 연산 예시**

```
duration one_second = 1000ms;                 // 1000 milliseconds = 1 second
duration thousand_cycles = 1000dt;            // Device-specific timing
duration two_seconds = one_second + 1s;       // Arithmetic: 1s + 1s = 2s
duration c = durationof({x $3;});            // Measure duration of gate sequence
// Note: dt units depend on the quantum hardware backend
```

#### 2. Stretch

* **서브타입**: 특수화된 duration 타입
* **해상도**: 컴파일 타임에 결정
* **용도**: 회로 실행에 따라 스케일링되는 타이밍에 사용

### 8. 앨리어싱(Aliasing)

* **키워드**: `let` (classical register aliasing시 let 불필요)
* **목적**: 큐비트와 레지스터에 대한 대체 이름을 생성 (복사본이 아닌 참조 생성)

**레지스터 앨리어싱 예시**

```
// 5-qubit register
qubit[5] q;
// Alias for qubits q[1], q[2], q[3]
let myreg = q[1:3];
// Applies X to q[1] (through alias)
x myreg[0];
// Note: myreg[0] refers to q[1], myreg[1] refers to q[2], etc.
```

### 9. 인덱스 집합과 슬라이싱(Index Set and Slicing)

#### 1. 레지스터 연결 및 슬라이싱(Register Concatenation and Slicing)

* **연결**: `++` 연산자 (동일 타입 레지스터)
* **인덱싱**: 단일 인덱스, 쉼표로 구분된 목록, 또는 범위
* **범위 문법**: `a:b` (a부터 b-1까지), `a:c:b` (a부터 b-1까지 c 스텝) 
  * (튜토리얼이 잘못된 듯, qasm3.0에서 슬라이싱 양 끝단은 inclusive)

**레지스터 연산 예시**

```
// 2-qubit register
qubit[2] one;
// 10-qubit register
qubit[10] two;
// 12-qubit alias (2 + 10)
let concatenated = one ++ two;

// First qubit (one[0])
let first = concatenated[0];
// Last qubit (two[9])
let last = concatenated[-1];
// Qubits 0, 3, 5 from 'two'
let qubit_selection = two[{0, 3, 5}];

// First 7 qubits
let sliced = concatenated[0:6];
// Every second qubit (0,2,4,6,8,10)
let every_second = concatenated[0:2:12]; 

// Last 3 qubits (indices 7,8,9)
let last_three = two[-4:-1];
// Concatenate two aliases
let both = sliced ++ last_three;
```

#### 2. 클래식 비트 값 슬라이싱(Classical Bit Value Slicing)

* **적용 대상**: `int`, `uint`, `angle` 타입
* **문법**: 레지스터 슬라이싱과 유사

**정수에 대한 비트 단위 연산 예시**

```
// Decimal 15 = 0xF = 0b0000...1111
int[32] myInt = 15;
// Least significant bit: 1
bit[1] lastBit = myInt[0];
// Most significant bit (sign): 0
bit[1] signBit = myInt[31];
// Most significant bit also using negative index: 0
bit[1] alsoSignBit = myInt[-1];   

// Every second bit starting from 0: bits 0,2,4,...,30
bit[16] evenBits = myInt[0:2:31];
// Bits 16 through 30 (inclusive)
bit[16] upperBits = myInt[-16:-1];
// Bits 31 down to 16 (reverse order)
bit[16] upperReversed = myInt[-1:-16];

// Set bits 4-7 to binary 1010
myInt[4:7] = "1010";
// Result: myInt becomes 0xAF (decimal 175)
```

**배열 요소 비트 접근 예시**

```
array[int[32], 5] intArr = {0, 1, 2, 3, 4};
// Set bit 0 of element 0 to 1
intArr[0][0] = 1;
// Result: intArr[0] becomes 1 (was 0)

// Copy lowest 5 bits of intArr[4] to b
bit[5] b = intArr[4][0:4];
// intArr[4] = 4 = 0b100, so b = "00100" (5 bits)
```

#### 3. 배열 연결 및 슬라이싱(Array Concatenation and Slicing)

* **연결**: `++` 연산자 (새 배열 생성, 데이터 복사)
* **슬라이싱**: 범위 문법을 사용한 서브 배열 추출
  * (이전 튜토리얼과 다르게 설명, qasm3.0에서 슬라이싱 양 끝단은 inclusive)
* **제한**: 연결된 배열을 서브루틴에 직접 전달할 수 없음

**배열 연산 예시**

```
array[int[8], 2] first = {0, 1};
array[int[8], 3] second = {2, 3, 4};

// {0, 1, 2, 3, 4}
array[int[8], 5] concat = first ++ second;
// {0, 1, 0, 1}
array[int[8], 4] selfConcat = first ++ first;

// Extract elements 1-2: {3, 4}
array[int[8], 2] secondSlice = second[1:2];
// Replace slice: second becomes {2, 0, 1}
second[1:2] = first[0:1];

array[int[8], 4] third = {5, 6, 7, 8};
// Complex assignment
selfConcat[0:3] = first[0:1] ++ third[1:2];

// subroutine_call(first ++ third)  // Not allowed: Cannot pass temporary concatenation
// subroutine_call(selfConcat)      // OK: Pass existing array variable
```

**다차원 배열 연산 예시**

```
int[8] scalar;
array[int[8], 2] oneD;
array[int[8], 3, 2] twoD;
array[int[8], 3, 2] anotherTwoD;
array[int[8], 4, 3, 2] threeD;

// Assign to single element
threeD[0, 0, 0] = scalar;
// Assign to 1D slice (first row, first column pair)
threeD[0, 0] = oneD;
// Assign to 2D slice (first 3×2 block)
threeD[0] = twoD;

// ERROR: Shape mismatch (3×2 vs 2)
threeD[0] = oneD;
// ERROR: Shape mismatch (2 vs 1)
// threeD[0, 0] = scalar

// Copy slice: rows 1-2 from anotherTwoD to twoD
twoD[1:2] = anotherTwoD[0:1];
// Copy specific column slice
twoD[1:2, 0] = anotherTwoD[0:1, 1];
```

### 10. 캐스팅 규칙(Casting Specifics)

* **명시적 캐스팅**: `type(value)` 문법을 사용한 제어형 변환
* **정밀도 손실**: 큰 타입에서 작은 타입으로 캐스팅 시 데이터 잘림 발생 가능
* **암시적 승격**: 이항 연산에서 “작은” 타입이 “큰” 타입으로 자동 캐스팅됨
* **타입 계층** (큰 것 → 작은 것): complex > float > int > uint > bool > bit
* **부호 처리**: 부호 있는 타입과 부호 없는 타입 간 캐스팅 시 부호 비트에 주의 필요


#### 1. 허용되는 캐스팅 매트릭스

| From \ To | bool | int | uint | float | bit | angle |
|-----------|------|-----|------|--------|-----|-------|
| **bool**  |  -   |  ✓  |  ✓   |   ✓    |  ✓  |   ✘   |
| **int**   |  ✓   |  -  |  ✓   |   ✓    |  ✓  |   ✘   |
| **uint**  |  ✓   |  ✓  |  -   |   ✓    |  ✓  |   ✘   |
| **float** |  ✓   |  ✓  |  ✓   |   -    |  ✘  |   ✓   |
| **bit**   |  ✓   |  ✓  |  ✓   |   ✘    |  -  |   ✘   |
| **angle** |  ✓   |  ✘  |  ✘   |   ✘    |  ✓  |   -   |


**특수 경우**:
* **qubit**: 다른 어떤 타입과도 캐스팅 불가
* **duration**: 다른 타입과 캐스팅 불가 (나눗셈 연산자를 사용할 경우 float으로만 변환 가능)

**연산에서의 타입 승격 예시**

```
int[8] a = 5;
float[16] b = 2.5;
float[16] c = a + b;  // a is promoted to float[16] before addition
// Result: c = 7.5 (5.0 + 2.5)

uint[4] x = 3;
bit[4] y = "0101";
uint[4] z = x + uint[4](y);  // y must be explicitly cast to uint
// Result: z = 8 (3 + 5)
```

#### 2. Exporter 클래스

Exporter 클래스를 사용하면 내보내기 과정을 세밀하게 커스터마이징할 수 있습니다.

**매개변수**:
* **`includes`**: 포함할 파일 목록 (기본값: `["stdgates.inc"]`)
* **`basis_gates`**: 특정 게이트 세트로 제한하여 하드웨어 호환성 확보
* **`disable_constants`**: 상수 폴딩 최적화 비활성화
* **`alias_classical_registers`**: 클래식 레지스터에 별칭 사용
* **`allow_aliasing`**: 큐비트 별칭에 `let` 문 사용 허용
* **`indent`**: 들여쓰기 문자열 (기본값: `"  "`)
* **`experimental`**: 실험적 기능 활성화
* **`annotation_handlers`**: 회로 어노테이션에 대한 사용자 정의 처리기

```python
# Example with Exporeter class

qc = QuantumCircuit(3)
qc.h(0)
qc.rx(1.57, 1)  # π/2 rotation

exporter = Exporter(
    # Don't include stdgates.inc
    includes=[],
    # Limit to specific gates
    basis_gates=["h", "rx"],
    # 4-space indentation
    indent="    ",
    allow_aliasing=True
)

# Export with custom exporter
qasm_str = exporter.dumps(qc)
print(qasm_str)
```

```python
# Example of importing a complex OpenQASM 3.0 program into Qiskit
program = """
    OPENQASM 3.0;
    include "stdgates.inc";
 
    input float[64] a;
    qubit[3] q;
    bit[2] mid;
    bit[3] out;
 
    let aliased = q[0:1];
 
    gate my_gate(a) c, t {
      gphase(a / 2);
      ry(a) c;
      cx c, t;
    }
    gate my_phase(a) c {
      ctrl @ inv @ gphase(a) c;
    }
 
    my_gate(a * 2) aliased[0], q[{1, 2}][0];
    measure q[0] -> mid[0];
    measure q[1] -> mid[1];
 
    while (mid == "00") {
      reset q[0];
      reset q[1];
      my_gate(a) q[0], q[1];
      my_phase(a - pi/2) q[1];
      mid[0] = measure q[0];
      mid[1] = measure q[1];
    }
 
    if (mid[0]) {
      let inner_alias = q[{0, 1}];
      reset inner_alias;
    }
 
    out = measure q;
"""
circuit = loads(program)
circuit.draw("mpl")
```

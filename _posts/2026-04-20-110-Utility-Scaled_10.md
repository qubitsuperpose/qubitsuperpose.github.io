### 2.1 최적화 수준

`optimization_level`에는 0부터 3까지 네 가지 옵션이 있습니다. 최적화 수준이 높을수록 회로를 최적화하는 데 더 많은 계산 노력이 투입됩니다. 수준 0은 최적화를 수행하지 않으며 선택된 백엔드에서 회로를 실행 가능하게 만들기 위한 최소한의 작업만 수행합니다. 수준 3은 회로를 최적화하기 위해 가장 많은 노력(및 일반적으로 런타임)을 투입합니다. 수준 1이 기본 최적화 수준입니다.

최적화를 적용하지 않은 경우(`optimization_level=0`)와 적용한 경우(`optimization_level=2`)에 회로를 트랜스파일링합니다.
트랜스파일링된 회로의 길이에 큰 차이가 있음을 확인할 수 있습니다.

```python
pm0 = generate_preset_pass_manager(
    optimization_level=0, backend=backend, seed_transpiler=777
)
pm2 = generate_preset_pass_manager(
    optimization_level=2, backend=backend, seed_transpiler=777
)
circ0 = pm0.run(ghz_circ)
circ2 = pm2.run(ghz_circ)
print("optimization_level=0:")
display(circ0.draw("mpl", idle_wires=False, fold=-1))
print("optimization_level=2:")
display(circ2.draw("mpl", idle_wires=False, fold=-1))
```

새로운 합성은 더 얕은 회로를 생성합니다. 그 이유는 무엇일까요?

이는 새로운 회로가 선형으로 연결된 큐비트에 배치될 수 있어 IBM® Brisbane의 heavy-hexagon 결합 그래프에도 배치될 수 있는 반면, 기존 회로는 별 모양의 연결 구조(차수 4의 노드)를 요구하기 때문에 최대 차수가 3인 노드로 이루어진 heavy-hex 결합 그래프에는 배치될 수 없기 때문입니다. 그 결과 기존 회로는 SWAP 게이트를 추가하는 큐비트 라우팅을 필요로 하게 되어 게이트 수가 증가하게 됩니다.

새로운 회로에서 우리가 수행한 작업은 수동으로 수행한 "결합 제약 조건 인식" 회로 합성으로 볼 수 있습니다. 즉, 회로 합성과 회로 매핑을 동시에 수동으로 해결하는 것입니다.

일반적으로 회로 합성은 응용 분야에 따라 달라지며, 소프트웨어가 모든 가능한 응용 분야를 포괄하는 것은 불가능합니다. Qiskit 트랜스파일러는 GHz 상태 준비 회로를 합성하는 기능을 제공하지 않습니다. 이러한 경우, 위에서 설명한 것처럼 수동으로 회로를 합성하는 것을 고려해 볼 만합니다.

이 섹션에서는 다음의 간단한 예제 회로를 사용하여 Qiskit 트랜스파일러의 작동 방식을 자세히 살펴보겠습니다.

```python
# Build a toy example circuit
from math import pi
import itertools
from qiskit.circuit import QuantumCircuit
from qiskit.circuit.library import excitation_preserving

circuit = QuantumCircuit(4, name="Example circuit")
circuit.append(excitation_preserving(4, reps=1, flatten=True), range(4))
circuit.measure_all()

value_cycle = itertools.cycle([0, pi / 4, pi / 2, 3 * pi / 4, pi, 2 * pi])
circuit.assign_parameters(
    [x[1] for x in zip(range(len(circuit.parameters)), value_cycle)], inplace=True
)
circuit.draw("mpl")
```

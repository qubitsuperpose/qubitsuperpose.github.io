# 4. Quantum Kernels

## 1. 양자 커널 소개

"Quantum kernel method(양자 커널 방법)"은 양자 컴퓨터를 사용하여 커널을 추정하는 모든 방법을 의미합니다. 여기서 "kernel(커널)"은 커널 행렬 또는 그 안에 있는 개별 항목을 가리킵니다. 특징 매핑(feature mapping) $\Phi(\vec{x})$은 $\vec{x}\in \mathbb{R}^d$에서 $\Phi(\vec{x})\in \mathbb{R}^{d'}$로의 매핑으로, 보통 $d'>d$이며 이 매핑의 목적은 데이터의 범주를 초평면(hyperplane)으로 분리 가능하게 만드는 것입니다. 커널 함수는 특징 매핑된 공간의 벡터들을 인수로 받아 그들의 내적(inner product)을 반환합니다. 즉, $K:\mathbb{R}^d\times\mathbb{R}^d\rightarrow \mathbb{R}$이며 $K(x,y) = \langle \Phi(x)|\Phi(y)\rangle$입니다. 고전적으로 우리는 커널 함수를 쉽게 평가할 수 있는 특징 매핑에 관심이 있습니다. 이는 종종 $\Phi(x)$와 $\Phi(y)$를 실제로 구성하지 않고도 특징 매핑된 공간에서의 내적을 원래 데이터 벡터들로 표현할 수 있는 커널 함수를 찾는 것을 의미합니다. 양자 커널 방법에서는 특징 매핑을 양자 회로로 수행하고, 해당 회로에 대한 측정과 상대적 측정 확률을 사용하여 커널을 추정합니다.

이번 강의에서는 상당한 얽힘(entanglement)을 사용하는 미리 코딩된 인코딩 회로들의 깊이(depth)를 살펴보고, 이를 우리가 직접 코딩한 회로들의 깊이와 비교할 것입니다. 이는 한 방법을 다른 방법보다 옹호하기 위함이 아닙니다. 미리 코딩된 회로가 너무 깊다고 느끼거나, 직접 만든 회로의 얽힘이 유용하기에 충분하지 않다고 느낄 수도 있습니다. 다시 말하지만, 이는 여러분의 탐색을 돕기 위해 제시된 것입니다.

커널 행렬 추정을 자세히 살펴보기 전에, Qiskit 패턴의 언어를 사용하여 워크플로를 개요로 설명하겠습니다.

### 1.1 Step 1: Map classical inputs to a quantum problem
* Input: Training dataset
* Output: Abstract circuit for calculating a kernel matrix entry

데이터셋이 주어지면, 시작점은 데이터를 양자 회로에 인코딩하는 것입니다. 즉, 우리의 데이터를 양자 컴퓨터의 상태 힐베르트 공간(Hilbert space)으로 매핑해야 합니다. 이는 데이터에 의존적인 회로를 구성함으로써 이루어집니다. 이를 수행하는 방법은 다양하며, 이전 강의에서 여러 옵션을 설명했습니다. 여러분은 데이터를 인코딩하기 위한 자체 회로를 구성하거나, `zz_feature_map`과 같은 미리 만들어진 특징 맵을 사용할 수 있습니다. 이번 강의에서는 둘 다 수행할 것입니다.

단일 커널 행렬 요소를 계산하려면 두 개의 서로 다른 데이터를 인코딩하여 그들의 내적을 추정해야 합니다. 전체 양자 커널 워크플로는 당연히 매핑된 데이터 벡터들 사이의 많은 내적과 고전적 기계학습 방법을 포함하지만, 반복되는 핵심 단계는 단일 커널 행렬 요소의 추정입니다. 이를 위해 데이터 의존적 양자 회로를 선택하고 두 데이터 벡터를 특징 공간으로 매핑합니다.

![Classical\_Review\_background\_kernel\_circuit](https://quantum.cloud.ibm.com/learning/images/courses/quantum-machine-learning/quantum-kernel-methods/classical-review-background-kernel-circuit.avif)

커널 행렬을 생성하는 작업에서 우리는 특히 $\|0\rangle^{\otimes N}$ 상태(모든 $N$ 큐비트가 $\|0\rangle$ 상태에 있는 경우)를 측정할 확률에 특별한 관심을 가집니다. 이를 이해하기 위해, 하나의 데이터 벡터 $\vec{x}_i$를 인코딩하고 매핑하는 회로를 $\Phi(\vec{x}_i)$로, $\vec{x}_j$를 인코딩하고 매핑하는 회로를 $\Phi(\vec{x}_j)$로 쓸 수 있으며, 매핑된 상태를 다음과 같이 표기합니다.

<p>
$$
|\psi(\vec{x}_i)\rangle = \Phi(\vec{x}_i)|0\rangle^{\otimes N}
$$
</p>

<p>
$$
|\psi(\vec{x}_j)\rangle = \Phi(\vec{x}_j)|0\rangle^{\otimes N}.
$$
</p>

이 상태들이 바로 데이터를 고차원으로 매핑한 것이므로, 우리가 원하는 커널 항목은 내적입니다:

<p>
$$
\langle\psi(\vec{x}_j)|\psi(\vec{x}_i)\rangle = \langle 0 |^{\otimes N}\Phi^\dagger(\vec{x}_j)\Phi(\vec{x}_i)|0\rangle^{\otimes N}.
$$
</p>

기본 초기 상태 $\|0\rangle^{\otimes N}$에 $\Phi^\dagger(\vec{x}_j)$와 $\Phi(\vec{x}_i)$ 두 회로를 모두 적용하면, $\|0\rangle^{\otimes N}$ 상태를 측정할 확률은 다음과 같습니다:

<p>
$$
P_0 = |\langle0|^{\otimes N}\Phi^\dagger(\vec{x}_j)\Phi(\vec{x}_i)|0\rangle^{\otimes N}|^2.
$$
</p>

이것이 정확히 우리가 원하는 값입니다(제곱까지). 우리 회로의 측정 레이어는 측정 확률(또는 특정 오류 완화 방법을 사용할 경우 "quasi-probabilities")을 반환합니다. 관심 있는 확률은 영 상태 $\|0\rangle^{\otimes N}$의 확률입니다.

### 1.2 Step 2: Optimize problem for quantum execution
* Input: Abstract circuit, not optimized for a particular backend
* Output: Target circuit and observable, optimized for the selected QPU

이 단계에서는 Qiskit의 `generate_preset_pass_manager` 함수를 사용하여 우리가 실험을 실행할 실제 양자 컴퓨터에 맞게 회로의 최적화 루틴을 지정합니다. `optimization_level=3`으로 설정하면 최고 수준의 최적화를 제공하는 사전 설정 패스 매니저를 사용하게 됩니다. 여기서 "optimization(최적화)"은 실제 양자 컴퓨터에서 회로의 구현을 최적화하는 것을 의미합니다. 이는 추상 양자 회로의 큐비트에 대응하는 물리적 큐비트를 선택하여 게이트 깊이를 최소화하거나, 사용 가능한 오류율이 가장 낮은 물리적 큐비트를 선택하는 등의 고려사항을 포함합니다. 이는 기계학습 문제 자체의 최적화(고전적 옵티마이저 COBYLA 등)와는 직접 관련이 없습니다.

Step 2를 어떻게 구현하느냐에 따라, 행렬 요소에 관련된 각 데이터 쌍이 측정할 서로 다른 회로를 생성하므로 회로를 여러 번 최적화해야 할 수도 있습니다.

### 1.3 Step 3: Execute using Qiskit Runtime Primitives
* Input: Target circuit
* Output: Probability distribution

Qiskit Runtime의 `Sampler` 프리미티브를 사용하여 회로를 샘플링했을 때 얻어지는 상태들의 확률 분포를 재구성합니다. 노이즈가 문제되고 오류 완화 등의 추가 단계가 도입되는 경우 "quasi-probability distribution"이라고 불리는 것을 볼 수 있습니다. 이 경우 모든 확률의 합이 정확히 1이 되지 않을 수 있어 "quasi-probability"라는 용어가 사용됩니다.

### 1.4 Step 4: Post-process, return result in classical format
* Input: Probability distribution
* Output: A single kernel matrix element, or a kernel matrix if repeating

양자 회로에서 $\|0\rangle^{\otimes N}$을 측정할 확률을 계산하고, 사용된 두 데이터 벡터에 해당하는 위치에 커널 행렬을 채웁니다. 전체 커널 행렬을 채우기 위해서는 각 항목마다 양자 실험을 실행해야 합니다. 커널 행렬을 얻으면 `pre-calculated kernels`을 받아들이는 많은 고전적 기계학습 알고리즘에서 이를 사용할 수 있습니다. 예: `qml_svc = SVC(kernel="precomputed")`. 이후 고전적 워크스트림을 사용하여 테스트 데이터에 모델을 적용하고 정확도 점수를 얻을 수 있습니다. 정확도 점수에 만족하지 못하면 특징 맵 등 계산의 여러 측면을 다시 검토해야 할 수 있습니다.

### 1.5 Lesson outline
이번 강의에서는 실제 양자 컴퓨터의 시간을 최적으로 사용하기 위해 위의 단계를 여러 방식으로 수행할 것입니다. 양자 커널 방법을 다음에 적용합니다:

* 특징이 비교적 적은 데이터에 대한 단일 커널 행렬 항목을 실제 백엔드를 사용하여 계산합니다. 각 단계에서 무슨 일이 일어나는지 쉽게 따라갈 수 있도록 하기 위함입니다.
* 특징이 비교적 적은 전체 데이터셋에 대해 시뮬레이션된 백엔드를 사용하여 계산합니다. 이를 통해 양자 워크스트림이 고전적 기계학습 방법과 어떻게 연결되는지 확인할 수 있습니다.
* 특징이 많은 데이터에 대한 단일 커널 행렬 항목을 실제 양자 컴퓨터를 사용하여 계산합니다. IBM® 양자 컴퓨터의 시간을 존중하기 위해 대규모 데이터셋에 대한 전체 커널 행렬은 추정하지 않습니다.


## 2. Single kernel matrix entry

### 2.1 1단계: 고전적 입력을 양자 문제로 매핑하기

먼저 특징(feature)이 몇 개뿐인 데이터셋, 예를 들어 10개를 고려해 보겠습니다. 데이터셋의 크기는 원하는 만큼 커도 상관없습니다. 커널 행렬의 원소를 한 번에 하나씩 계산하기 때문입니다. 최소 두 개의 데이터 포인트가 필요하므로, 먼저 그 정도로 시작하겠습니다(다음 예제에서는 전체 데이터셋을 불러올 것입니다). 필요한 패키지 몇 가지를 import해 보겠습니다:

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# Two mock data points, including category labels, as in training
small_data = [
    [-0.194, 0.114, -0.006, 0.301, -0.359, -0.088, -0.156, 0.342, -0.016, 0.143, 1],
    [-0.1, 0.002, 0.244, 0.127, -0.064, -0.086, 0.072, 0.043, -0.053, 0.02, -1],
]

# Data points with labels removed, for inner product
train_data = [small_data[0][:-1], small_data[1][:-1]]
```

`z_feature_map`을 사용해 볼 수 있습니다.

```python
# from qiskit.circuit.library import zz_feature_map
# fm = zz_feature_map(feature_dimension=np.shape(train_data)[1], entanglement='linear', reps=1)

from qiskit.circuit.library import z_feature_map

fm = z_feature_map(feature_dimension=np.shape(train_data)[1])


unitary1 = fm.assign_parameters(train_data[0])
unitary2 = fm.assign_parameters(train_data[1])
```

위의 두 유니터리 회로는 서론에서 설명한 $U_1$과 $U_2$에 정확히 대응합니다. `unitary_overlap`을 사용하여 이들을 결합할 수 있습니다. 언제나 그렇듯이 회로 깊이를 주의 깊게 살펴봐야 합니다.

```python
from qiskit.circuit.library import unitary_overlap


overlap_circ = unitary_overlap(unitary1, unitary2)
overlap_circ.measure_all()

print("circuit depth = ", overlap_circ.decompose().depth())
overlap_circ.decompose().draw("mpl", scale=0.6, style="iqp")
```

### 2.2 2단계: 양자 컴퓨팅 실행을 위한 문제 최적화

```python
def visualize_counts(probs, num_qubits):
    """Visualize the outputs from the Qiskit Sampler primitive."""
    zero_prob = probs.get(0, 0.0)
    top_10 = dict(sorted(probs.items(), key=lambda item: item[1], reverse=True)[:10])
    top_10.update({0: zero_prob})
    by_key = dict(sorted(top_10.items(), key=lambda item: item[0]))
    xvals, yvals = list(zip(*by_key.items()))
    xvals = [bin(xval)[2:].zfill(num_qubits) for xval in xvals]
    plt.bar(xvals, yvals)
    plt.xticks(rotation=75)

    plt.xlabel("Measured bitstring")
    plt.ylabel("Counts")
    plt.show()


visualize_counts(counts, overlap_circ.num_qubits)
```

이 정보는 고차원 특징 공간(feature space)에서 두 데이터 포인트 사이의 단 하나의 내적(inner product)에 대한 것일 뿐이므로, 우리가 말할 수 있는 것은 그들의 중첩(overlap)이 최대 중첩(1.0)에 비해 상당히 크다는 것뿐입니다. 이는 두 데이터 포인트가 본질적으로 어느 정도 유사하며, 같은 클래스에 속할 가능성이 있다는 지표가 될 수 있습니다. 또는 이 특징 맵(feature map)이 비슷한 데이터는 강한 중첩을, 다른 데이터는 작은 중첩을 가지도록 매핑하는 데 효과적이지 않다는 지표일 수도 있습니다. 어느 쪽이 맞는지 알기 위해서는, 전체 데이터셋에 특징 맵을 적용해보고, 결과로 나온 커널 행렬(kernel matrix)이 클래스들을 높은 정확도로 효과적으로 분리할 수 있는지 확인해야 합니다.

주목할 점은, 우리는 `z_feature_map`을 사용했으며, 이는 매우 낮은 2-큐비트 트랜스파일된 깊이(transpiled depth, 실제로 depth 1)를 보였다는 것입니다. 회로가 너무 깊어지면 필연적으로 많은 노이즈가 발생하고, 이는 특징 맵이 데이터에 잘 맞더라도 0 상태를 측정할 확률을 매우 낮게 만듭니다. 예를 들어, 위 과정을 `zz_feature_map`과 `entanglement='linear', reps=1`로 반복했을 때, 같은 데이터 포인트를 사용했음에도 `dist.get(0,0.0) = 0.0015`라는 결과가 나왔습니다. 이는 `zz_feature_map`에서 발생하는 훨씬 더 큰 회로 깊이와 2-큐비트 깊이 때문입니다. 아래 그림은 해당 계산의 확률 분포를 보여줍니다.

![Bad results from a zz feature map.](https://quantum.cloud.ibm.com/learning/images/courses/quantum-machine-learning/quantum-kernel-methods/zzfeaturemap-bad-results.avif)

같은 카테고리의 몇 가지 데이터 포인트로 실험해보고, 좋은 결과를 얻기 위해 깊이를 얼마나 낮게 유지해야 하는지 확인해보는 것이 좋습니다. 아래는 예외가 있을 수 있는 대략적인 조언입니다. 일반적으로 2-큐비트 트랜스파일된 깊이가 10 이하라면 큰 문제가 없습니다. 50~60 정도는 최신 기술 수준이며, 고급 오류 완화 기법 등 다른 도구들이 필요합니다. 그 중간 범위에서는 데이터 유사성, 특징 맵의 표현력(expressivity), 회로 폭(circuit width), 기타 요인에 따라 결과가 달라질 수 있습니다.

일반적으로 후처리 단계에는 고전적인 머신러닝 프로세스가 포함됩니다. 다음 섹션에서는 이 프로세스를 전체 데이터셋으로 확장하고 고전적인 머신러닝 워크플로우를 보여드리겠습니다.

### 3.1 단계 1: 고전적 입력을 양자 문제로 매핑하기

훈련에는 처음 90개의 데이터 포인트를 사용하고, 다음 30개의 포인트는 테스트에 사용하겠습니다.

```python

df = pd.read_csv("dataset_graph7.csv", sep=",", header=None)

# Prepare training data

train_size = 90
X_train = df.values[0:train_size, :-1]
train_labels = df.values[0:train_size, -1]

# Prepare testing data
test_size = 30
```

우리는 적절한 크기의 커널 행렬과 테스트 행렬을 구성하여 여러 출력을 저장할 준비를 미리 할 것입니다.

```python
# Empty kernel matrix
num_samples = np.shape(X_train)[0]
kernel_matrix = np.full((num_samples, num_samples), np.nan)
test_matrix = np.full((test_size, num_samples), np.nan)
```

이제 양자 회로에서 고전 데이터를 인코딩하고 매핑하기 위한 특징 맵을 생성합니다. 특징 맵은 직접 만들거나 미리 만들어진 것을 사용할 수 있습니다. 아래 특징 맵을 수정하거나 ZFeatureMap으로 되돌아가도 됩니다. 하지만 항상 회로 깊이에 주의해야 합니다. 이전 6큐비트 예제에서 `zz_feature_map`을 사용했을 때 변환된 회로 깊이가 처리하기 어려울 정도로 높았던 것을 기억하십시오. 회로의 규모와 복잡성이 증가함에 따라 회로 깊이는 노이즈가 결과를 압도할 정도로 빠르게 증가할 수 있습니다. 데이터 구조에 대해 어떤 특징 맵 구조가 가장 유용할지 알 수 있는 정보가 있다면, 그 정보를 활용하여 사용자 지정 특징 맵을 직접 만드는 것이 좋습니다.

```python
from qiskit.circuit import Parameter, ParameterVector, QuantumCircuit

# Prepare feature map for computing overlap
num_features = np.shape(X_train)[1]
num_qubits = int(num_features / 2)

# To use a custom feature map use the lines below.
entangler_map = [[0, 2], [3, 4], [2, 5], [1, 4], [2, 3], [4, 6]]

fm = QuantumCircuit(num_qubits)
training_param = Parameter("θ")
feature_params = ParameterVector("x", num_qubits * 2)
fm.ry(training_param, fm.qubits)
for cz in entangler_map:
    fm.cz(cz[0], cz[1])
for i in range(num_qubits):
    fm.rz(-2 * feature_params[2 * i + 1], i)
    fm.rx(-2 * feature_params[2 * i], i)
```

### 3.2 단계 2와 3: 문제 최적화 및 프리미티브를 사용한 실행

우리는 오버랩 회로(overlap circuit)를 구성할 것입니다. 만약 이 예제에서 실제 양자 컴퓨터에서 실행한다면, 이전과 마찬가지로 실행을 위해 회로를 최적화할 것입니다. 하지만 이 경우에는 모든 데이터 포인트를 순회하면서 전체 커널 행렬(kernel matrix)을 계산하려고 합니다. 각 데이터 벡터 $\vec{x}_i$와 $\vec{x}_j$의 쌍에 대해 서로 다른 오버랩 회로를 생성합니다. 따라서 각 데이터 포인트 쌍마다 회로를 최적화해야 합니다. 그래서 단계 2와 3은 여러 반복(iteration)에서 함께 수행됩니다.

아래 코드 셀은 단일 데이터 포인트 쌍에 대해 이전과 정확히 동일한 과정을 수행합니다. 이번에는 단순히 두 개의 `for` 루프 안에 실행되며, 각 계산 결과를 저장하기 위해 마지막에 `kernel_matrix[x_1,x_2] = ...`라는 추가 줄이 있습니다. 커널 행렬의 대칭성을 활용하여 계산 횟수를 1/2로 줄인 점에 주목하세요. 또한 노이즈가 없을 때 그래야 하듯이 대각 원소(diagonal elements)는 단순히 1로 설정했습니다. 구현 방식과 필요한 정밀도에 따라, 대각 원소를 이용해 노이즈를 추정하거나 오류 완화(error mitigation)를 위해 이를 활용할 수도 있습니다.

커널 행렬이 완전히 채워지면, 테스트 데이터에 대해 동일한 과정을 반복하여 test\_matrix를 채웁니다. 이는 사실상 또 다른 커널 행렬이지만, 두 행렬을 구분하기 위해 다른 이름을 부여한 것입니다.


```python
# To use a simulator
from qiskit.primitives import StatevectorSampler

# Remember to insert your token in the QiskitRuntimeService constructor
# to use real quantum computers

# service = QiskitRuntimeService()
# backend = service.least_busy(
#    operational=True, simulator=False, min_num_qubits=fm.num_qubits
# )

num_shots = 10000

# Evaluate the problem using state vector-based primitives from Qiskit.
sampler = StatevectorSampler()

for x1 in range(0, train_size):
    for x2 in range(x1 + 1, train_size):
        unitary1 = fm.assign_parameters(list(X_train[x1]) + [np.pi / 2])
        unitary2 = fm.assign_parameters(list(X_train[x2]) + [np.pi / 2])

        # Create the overlap circuit
        overlap_circ = unitary_overlap(unitary1, unitary2)
        overlap_circ.measure_all()

        # These lines run the qiskit sampler primitive.
        counts = (
            sampler.run([overlap_circ], shots=num_shots)
            .result()[0]
            .data.meas.get_int_counts()
        )

        # Assign the probability of the 0 state to the kernel matrix, and the transposed element
        # (since this is an inner product)
        kernel_matrix[x1, x2] = counts.get(0, 0.0) / num_shots
        kernel_matrix[x2, x1] = counts.get(0, 0.0) / num_shots
    # Fill in on-diagonal elements with 1, again, since this is an inner-product corresponding to
    # probability (or alter the code to check these entries and verify they yield 1)
    kernel_matrix[x1, x1] = 1

print("training done")

# Similar process to above, but for testing data.
for x1 in range(0, test_size):
    for x2 in range(0, train_size):
        unitary1 = fm.assign_parameters(list(X_test[x1]) + [np.pi / 2])
        unitary2 = fm.assign_parameters(list(X_train[x2]) + [np.pi / 2])

        # Create the overlap circuit
        overlap_circ = unitary_overlap(unitary1, unitary2)
        overlap_circ.measure_all()

        counts = (
            sampler.run([overlap_circ], shots=num_shots)
            .result()[0]
            .data.meas.get_int_counts()
        )

        test_matrix[x1, x2] = counts.get(0, 0.0) / num_shots

print("test matrix done")
```

### 3.3 4단계: 후처리, 결과를 기존 형식으로 반환

양자 커널 방법을 통해 커널 행렬과 유사한 형식의 테스트 행렬을 얻었으므로, 이제 기존 머신러닝 알고리즘을 적용하여 테스트 데이터에 대한 예측을 수행하고 정확도를 확인할 수 있습니다. 먼저 Scikit-Learn의 `sklearn.svc`라는 서포트 벡터 분류기(SVC)를 가져옵니다. `kernel = precomputed`를 사용하여 SVC가 미리 계산된 커널을 사용하도록 지정해야 합니다.


```python
# import a support vector classifier from a classical ML package.
from sklearn.svm import SVC

# Specify that you want to use a pre-computed kernel matrix
qml_svc = SVC(kernel="precomputed")
```

이제 `SVC.fit`을 사용하면 커널 행렬과 훈련 레이블을 입력하여 적합도를 얻을 수 있습니다. 그런 다음 `SVC.score`는 테스트 행렬을 사용하여 적합된 모델에 대한 테스트 데이터의 점수를 계산하고 정확도를 반환합니다.

```python
# Feed in the pre-computed matrix and the labels of the training data.
# The classical algorithm gives you a fit.
qml_svc.fit(kernel_matrix, train_labels)

# Now use the .score to test your data, using the matrix of test data,
# and test labels as your inputs.
qml_score_precomputed_kernel = qml_svc.score(test_matrix, test_labels)
print(f"Precomputed kernel classification test score: {qml_score_precomputed_kernel}")
```

Precomputed kernel classification test score: 1.0

훈련된 모델의 정확도가 100%임을 확인했습니다. 이는 훌륭한 결과이며, 양자 커널(QKE)이 효과적임을 보여줍니다. 하지만 이는 양자 우위와는 매우 다릅니다. 기존 커널을 ​​사용했더라도 이 분류 문제를 100% 정확도로 해결할 수 있었을 것입니다. 다양한 데이터 유형과 데이터 간의 관계를 분석하여 양자 커널이 현재 활용 시대에 가장 유용하게 사용될 수 있는 분야를 파악하기 위해서는 더 많은 연구가 필요합니다.
학습자는 이 워크플로의 일부를 수정하고 다양한 양자 특징 맵의 효과를 연구해 볼 수 있습니다. 다음 사항들을 고려해 보세요.

### 4.1 Step 1: 고전적 입력을 양자 문제로 매핑하기

각 데이터 포인트가 42개의 특징을 가진 데이터셋을 시작점으로 가정하겠습니다. 첫 번째 예시와 마찬가지로, 두 개의 데이터 포인트가 필요한 단일 커널 행렬 요소를 계산할 것입니다. 아래의 두 포인트는 42개의 특징과 단일 범주 변수($\pm 1$)를 가지고 있습니다.

```python
# Two mock data points, including category labels, as in training
```

```python
    [2, 5],
    [1, 4],
    [2, 3],
    [4, 6],
    [7, 9],
    [10, 11],
    [9, 12],

    [14, 16],
    [17, 18],
    [16, 19],
    [15, 18],
    [16, 17],
    [18, 20],
]
```

```python
# Use the entangler map above to build a feature map

num_features = np.shape(train_data)[1]
num_qubits = int(num_features / 2)

fm = QuantumCircuit(num_qubits)
training_param = Parameter("θ")
feature_params = ParameterVector("x", num_qubits * 2)
fm.ry(training_param, fm.qubits)
for cz in entangler_map:
    fm.cz(cz[0], cz[1])
for i in range(num_qubits):
    fm.rz(-2 * feature_params[2 * i + 1], i)
    fm.rx(-2 * feature_params[2 * i], i)
```

```python
from qiskit.circuit.library import unitary_overlap

# Assign features of each data point to a unitary, an instance of the general feature map.

unitary1 = fm.assign_parameters(list(train_data[0]) + [np.pi / 2])
unitary2 = fm.assign_parameters(list(train_data[1]) + [np.pi / 2])

# Create the overlap circuit
```

### 4.2 2단계: 양자 실행을 위한 문제 최적화

가장 사용량이 적은 백엔드를 선택한 다음 해당 백엔드에서 실행되도록 회로를 최적화합니다.

```python
# Import needed packages
from qiskit.transpiler.preset_passmanagers import generate_preset_pass_manager
from qiskit_ibm_runtime import QiskitRuntimeService

# Get the least busy backend
service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=fm.num_qubits
)
print(backend)
```

소규모 작업에서는 미리 설정된 패스 관리자가 동일한 회로와 깊이를 안정적으로 반환하는 경우가 많습니다. 그러나 매우 크고 복잡한 회로의 경우, 패스 관리자는 실행할 때마다 다른 트랜스파일링된 회로를 반환할 수 있습니다. 이는 휴리스틱을 사용하기 때문이며, 매우 큰 회로는 가능한 최적화 옵션이 복잡하기 때문입니다. 여러 번 트랜스파일링하여 가장 얕은 회로를 선택하는 것이 유용한 경우가 많습니다. 이는 고전적인 오버헤드만 발생시키고 양자 컴퓨터의 성능을 크게 향상시킬 수 있습니다.

본 연구에서는 유니터리 오버랩 회로를 20번 트랜스파일링하고, 얻어진 회로의 깊이를 살펴봅니다.

### 4.5  스케일링과 향후 연구

커널 방법은 $\|0\rangle$ 상태를 가능한 한 정확하게 측정할 것을 요구합니다. 그러나 게이트 오류와 읽기 오류(readout errors)로 인해, 임의의 큐비트가 $\|1\rangle$ 상태로 잘못 측정될 비영(非零) 확률 $p$가 존재합니다. $\|0\rangle$의 확률이 $100\%$여야 한다는 과도한 단순화를 가정하더라도, 예를 들어 $N$ 비트에 인코딩된 많은 특징의 경우, 모든 비트가 $\|0\rangle$으로 올바르게 측정될 확률은 $(1-p)^N$으로 감소합니다. $N$이 커질수록 이 방법은 점점 덜 신뢰할 수 있게 됩니다. 이러한 어려움을 극복하고 커널 추정을 더욱 많은 특징으로 확장하는 것은 현재 연구가 진행 중인 분야입니다. 이 문제에 대해 더 알아보려면 [Thanasilp, Wang, Cerezo, and Holmes](https://www.nature.com/articles/s41467-024-49287-w)의 연구를 참조하십시오. 우리는 현재 양자 컴퓨터로 무엇을 할 수 있는지 탐구해 볼 것을 권장하며, 오류 정정 시대에 가능해질 일들도 기대해 봅시다.


### 4.6 복습

양자 커널을 계산하는 것은 다음을 포함합니다
* 훈련 데이터 포인트 쌍을 사용하여 커널 행렬 항목을 계산하는 것
* 데이터를 인코딩하고 특성 매핑(feature mapping)을 통해 매핑하는 것
* 실제 양자 컴퓨터/백엔드에서 실행하기 위해 회로를 최적화하는 것

이렇게 얻은 양자 커널은 이번 수업에서처럼 고전적인 머신러닝 알고리즘에서 사용될 수 있습니다.

양자 커널을 사용할 때 유념해야 할 몇 가지 핵심 사항은 다음과 같습니다:
* 해당 데이터셋이 양자 커널 방법으로부터 이득을 얻을 가능성이 있는가?
* 다양한 특성 맵(feature map)과 얽힘(entanglement) 방식을 시도해 보세요.
* 회로 깊이(circuit depth)가 허용 가능한 수준인가?
* 패스 매니저(pass manager)를 여러 번 실행해 보고, 얻을 수 있는 가장 작은 깊이의 회로를 사용하세요.

양자 커널 방법은 양자 친화적인 특성을 가진 데이터셋과 적절한 양자 특성 맵 사이의 알맞은 매칭이 이루어진다면 잠재적으로 강력한 도구가 될 수 있습니다. 양자 커널이 어디에서 유용하게 쓰일 가능성이 높은지 더 잘 이해하려면, [Liu, Arunachalam & Temme (2021)](https://www.nature.com/articles/s41567-021-01287-z) 논문을 읽어보시길 권합니다.

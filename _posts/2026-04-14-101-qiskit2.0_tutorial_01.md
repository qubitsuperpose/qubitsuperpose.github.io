# 1.Perform_Quantum_Operations

## 1. Pauli Operators 정의하기

**개요:**  
이 노트북에서는 Pauli 연산자에 대해 다룹니다. Pauli 연산자를 초기화하는 방법, 속성, 그리고 메서드에 대해 학습합니다.

Pauli 연산자는 양자 컴퓨팅의 기본 구성 요소로, 단일 큐비트 및 다중 큐비트 연산을 나타냅니다. Qiskit에서는 `Pauli` 클래스를 통해 이러한 연산자를 다양한 방법으로 생성하고 조작할 수 있습니다.

```python
# Setup: Import necessary libraries
from qiskit.quantum_info import Pauli, ScalarOp
import numpy as np
from qiskit import QuantumCircuit

print("Libraries imported successfully.")
```

### 1.초기화 방법

#### 1. Pauli 문자열 표현

Pauli 연산자는 문자열을 사용하여 생성할 수 있습니다:

- `I`: 단위(Identity) 연산자
- `X`: Pauli-X (비트 뒤집기) 연산자
- `Y`: Pauli-Y 연산자
- `Z`: Pauli-Z (위상 뒤집기) 연산자

**주의:** Qiskit은 **Little Endian Notation**을 사용합니다. 문자열에서 **가장 왼쪽이 qubit (n-1)**, **가장 오른쪽이 qubit 0**입니다.

위상(phase)은 ` ` (기본값 +1), `-`, `i`, `-i`로 지정할 수 있습니다.

```python
# Create Pauli operators from string representations
p = Pauli('XZ')      # No phase (default +1)
p2 = Pauli('iYY')    # With imaginary phase i
p3 = Pauli('-iZX')   # With negative imaginary phase -i

print(f"P:{p}, P2:{p2}, P3:{p3}")
```

#### 2. 불리언 배열 표현

불리언 numpy 배열을 사용하여 Pauli 연산자를 생성할 수도 있습니다:

- `z` 배열: Z 성분 지정 (True = Z, False = I)
- `x` 배열: X 성분 지정 (True = X, False = I)

각 큐비트 위치에서 실제 Pauli 연산자는 다음과 같이 결정됩니다:
- (z=False, x=False) → I
- (z=False, x=True) → X
- (z=True, x=False) → Z
- (z=True, x=True) → Y (Y = iXZ)

```python
# Define Z and X components using boolean arrays
z = np.array([False,False,True,True])  
x = np.array([False,True,False,True])  

pauli_array_rep = Pauli((z, x))

print(f"Pauli Operator from Boolean Array: {pauli_array_rep}")
```

**참고:** 배열은 왼쪽부터 인덱싱되지만, Qiskit의 문자열 표현은 오른쪽부터 qubit 0으로 시작하므로 출력 순서가 반대로 보입니다.

#### 3. Quantum Circuit 표현

I, X, Y, Z 게이트만 포함된 양자 회로로부터 Pauli 연산자를 추출할 수 있습니다.

```python
# Create a quantum circuit with Pauli gates
qc = QuantumCircuit(4)
qc.id(0)
qc.z(1)
qc.x(2)
qc.y(3)

pauli_quantum_circuit = Pauli(qc)

print(f"Pauli Operator from quantum circuit : {pauli_quantum_circuit}")
```

#### 4. ScalarOp로부터 생성

`ScalarOp`을 사용해 전역 위상(global phase)을 Pauli 연산자에 적용할 수 있습니다.

```python

scalar_op = ScalarOp(dims=(2,2,2,2), coeff=-1j)
# dims=(2,2,2,2): 4개의 qubit (각 qubit은 2차원 Hilbert 공간)을 의미합니다.
# coeff=-1j: -i (허수 단위 i에 -1을 곱한 값)를 전체 연산자에 곱합니다.
pauli_scalarop = Pauli('YXZI')
p = scalar_op.compose(pauli_scalarop)
#`compose()`: 연산자 합성(composition)입니다.
# 수학적으로: `p = scalar_op @ pauli_scalarop` (행렬 곱)
# 의미: -i × (Y ⊗ X ⊗ Z ⊗ I) 가 됩니다.

print(f"Pauli Operator Before applying Scalar : {pauli_scalarop}")
print(f"Pauli Operator from ScalarOp : {p}")
```


### 2. 표현 방법

#### 1. 행렬 표현 (`to_matrix()`)

Pauli 연산자를 행렬 형태로 변환합니다.

```python
p = Pauli('ZX')
p_matrix = p.to_matrix()
print(f"Matrix Format: {p_matrix}")
```

#### 2. 문자열 표현 (`to_label()`)

Pauli 연산자를 읽기 쉬운 문자열 라벨로 변환합니다.

```python
print(f"String format:{str(p)}")
print(f"Label format:{p.to_label()}")
```

### 3. 속성 (Attributes)

`Pauli` 클래스는 다음과 같은 주요 속성을 제공합니다:

- `dim`: 연산자의 차원
- `num_qubits`: 큐비트 수
- `num_clbits`: 고전 비트 수 (Pauli는 항상 0)
- `phase`: 위상 (0:+1, 1:i, 2:-1, 3:-i)
- `x`, `z`: X와 Z 성분의 불리언 배열

```python
print(f"Dimension: {p.dim}")
print(f"Classical bits: {p.num_clbits}, Qubits: {p.num_qubits}")
print(f"X: {p.x}, Z: {p.z}, Phase: {p.phase}")
```

### 4. 메서드 (Methods)

#### 1. adjoint()
Pauli 연산자의 adjoint (복소 켤레 전치)를 반환합니다. Pauli는 Hermitian하므로 본질적으로 자기 자신과 같지만, 위상이 변경될 수 있습니다.

```python
print(f"pauli operator:{p3}, adjoint: {p3.adjoint()}")
```

#### 2. anticommutes()
두 Pauli 연산자가 반가환(anticommute)하는지 확인합니다. (AB = -BA)

```python
p = Pauli('X')      # X
p2 = Pauli('Y')     # Y
print(f"{p} anticommutes with {p2} is {p.anticommutes(p2)}")
```

#### 3. compose()
두 Pauli 연산자의 행렬 곱(합성)을 반환합니다.

```python
print(f"{p} compose with {p2} is {p.compose(p2)}")
```

#### 4. conjugate()
복소 켤레를 반환합니다.
```python
# Get the complex conjugate of the Pauli operator
p = Pauli('ZX')
print(f"Conjugate of {p} is {p.conjugate()}")
print(f"Conjugate of {p2} is {p2.conjugate()}")
```

#### 5. delete()
지정된 큐비트를 삭제한 새로운 Pauli 연산자를 반환합니다.
```python
# Deletes a qubit from the Pauli operator at the specified index
qubit_index = 0
delete_0 = p.delete(qubit_index)

print(f"{p} after deleting qubit {qubit_index} is {delete_0}")
```

#### 6. insert()
특정 위치에 Pauli 연산자를 삽입합니다.
```python
# Inserts a Pauli operator at a specific qubit position into another Pauli operator
qubit_index = 1
insert_0 = delete_0.insert(qubit_index, Pauli('X'))

print(f"{delete_0} after inserting X at qubit {qubit_index} is {insert_0}")
```


#### 7. to_instruction()
Pauli 연산자를 양자 회로에서 사용할 수 있는 Instruction으로 변환합니다.
```python
# Convert Pauli operator to a quantum instruction
p.to_instruction()
```

#### 1. 단일 큐비트 표준 게이트 (One-qubit Standard Gates)

* **HGate**: 단일 큐비트 Hadamard 게이트
* **IGate**: 단위(Identity) 게이트
* **PhaseGate**: Z축을 중심으로 한 단일 큐비트 회전
* **RGate**: cos(ϕ)x + sin(ϕ)y 축을 중심으로 한 θ 각도 회전
* **RXGate**: X축을 중심으로 한 단일 큐비트 회전
* **RYGate**: Y축을 중심으로 한 단일 큐비트 회전
* **RZGate**: Z축을 중심으로 한 단일 큐비트 회전
* **SGate**: S 게이트 (Z^0.5)
* **SdgGate**: S adjoint 게이트 (~Z^0.5)
* **SXGate**: √X 게이트
* **SXdgGate**: √X adjoint 게이트
* **TGate**: T 게이트 (Z^0.25)
* **TdgGate**: T adjoint 게이트 (~Z^0.25)
* **UGate**: ZYZ Euler 각도를 이용한 일반 단일 큐비트 회전
* **U1Gate, U2Gate, U3Gate**: 다양한 Euler 각도 기반 회전
* **XGate**: Pauli-X 게이트
* **YGate**: Pauli-Y 게이트
* **ZGate**: Pauli-Z 게이트

```python
from qiskit.circuit.library import HGate

hgate = HGate()
print(f"Hadamard Gate Matrix:{hgate.to_matrix()}")
```

```python

```

### 4. Generalized Gates

* 표준 게이트를 확장한 형태
* 입력에 따라 큐비트 수가 달라지는 variadic 게이트 포함
* `Instruction` 또는 `Gate`를 상속받는 클래스를 사용하는 것을 권장합니다.

```python
from qiskit.circuit.library import DiagonalGate

diagonal = DiagonalGate([1, 1j])
print(diagonal.num_qubits)

diagonal = DiagonalGate([1, 1, 1, -1])
print(diagonal.num_qubits)
```

**게이트 또는 명령어 표현**

* **대각 게이트(DiagonalGate)**: 일반적인 양자 대각 게이트.
* **순열 게이트(PermutationGate)**: 큐비트를 순열하는 게이트.
* **MCMTGate**: 임의의 단일 제어 대상 게이트에 대한 다중 제어 다중 대상 게이트.
* **MCPhaseGate**: 다중 제어 위상 게이트.
* **MCXGate**: 일반적인 다중 제어 X 게이트.
* **MSGate**: 묄머-쇠렌센 게이트.
* **RVGate**: 임의의 회전축 v를 중심으로 한 회전. 여기서 ∥v∥2​는 라디안 단위의 회전 각도임.
* **파울리 게이트(PauliGate)**: 다중 큐비트 파울리 게이트.
* **선형 함수(LinearFunction)**: n개의 큐비트에 대한 선형 가역 회로.
* **등거리 변환(Isometry)**: mm에서 nn 큐비트까지의 임의의 등거리 변환 분해.
* **유니터리 게이트(UnitaryGate)**: 유니터리 행렬로 정의되는 양자 게이트 클래스.
* **UCGate**: 균일 제어 게이트(다중화 게이트라고도 함).
* **UCPauliRotGate**: 균일 제어 파울리 회전 게이트.
* **UCRXGate**: 균일 제어 파울리-X 회전 게이트.
* **UCRYGate**: 균일 제어 파울리-Y 회전 게이트.
* **UCRZGate**: 균일 제어 파울리-Z 회전 게이트.

**양자 회로 표현**

* **대각선(Diagonal)**: 대각 변환을 구현하는 회로.
* **MCMT**: 임의의 단일 제어 대상 게이트에 대한 다중 제어 다중 대상 게이트.
* **MCMTVChain**: CCX V-체인을 사용한 MCMT 구현.
* **MCXGrayCode**: 그레이 코드를 사용하여 다중 제어 X 게이트를 구현.
* **MCXRecursive** : 재귀를 사용하여 다중 제어 X 게이트를 구현합니다.
* **MCXVChain** : CX 게이트의 V-체인을 사용하여 다중 제어 X 게이트를 구현합니다.
* **Permutation** : n_qubit 큐비트를 순열하는 회로입니다.
* **GMS** : 전역 Mølmer–Sørensen 게이트입니다.
* **GR** : 전역 R 게이트입니다.
* **GRX** : 전역 RX 게이트입니다.
* **GRY(num_qubits, theta)** 전역 RY 게이트입니다.
* **GRZ(num_qubits, phi)** 전역 RZ 게이트입니다.

```python
from qiskit.circuit.library import MCXGate

mcxgate = MCXGate(num_ctrl_qubits=2)
```

```python
from qiskit.circuit.library import MCMTGate

mcmtGate = MCMTGate(gate=xgate,num_ctrl_qubits=3,num_target_qubits=1)
qc= QuantumCircuit(4,0)
qc.compose(mcmtGate).draw('mpl')
```

### 5. Boolean Logic Gates

**게이트 표현**

### 7. Arithmetic Gates
#### 1. 가산 게이트

**게이트 표현**
* **모듈러 가산 게이트(Modular Adder Gate)**: 크기가 n인 두 큐비트 레지스터의 합을 $2^n$으로 나눈 나머지를 계산합니다.
* **하프 가산 게이트(Half Adder Gate)**: 캐리 출력 비트를 포함하여 크기가 같은 두 큐비트 레지스터의 합을 계산합니다.
* **풀 가산 게이트(Full Adder Gate)**: 캐리 입력 및 출력 비트를 포함하여 크기가 n인 두 큐비트 레지스터의 합을 계산합니다.

**양자 회로 표현**
* **드레이퍼 QFTA 가산기(Draper QFTAdder)**: 두 큐비트 레지스터에 대해 제자리 덧셈을 수행하기 위해 QFT를 사용하는 회로입니다.
* **CDKMRippleCarryAdder**: 두 큐비트 레지스터에 대해 제자리 덧셈을 수행하는 리플 캐리 회로입니다.
* **VBERippleCarryAdder**: VBE 리플 캐리 가산기입니다.

### 8. Example - LinearPauliRotations
아래 예시는 Qiskit 라이브러리의 **`LinearPauliRotations`** 클래스를 사용합니다. 제공된 분류에 따르면, 이 컴포넌트는 **`QuantumCircuit`에서 파생된 클래스** 범주에 속합니다.

#### 1. 개요
`LinearPauliRotations`는 특수 양자 회로 라이브러리 컴포넌트입니다. 이 컴포넌트는 상태(제어) 큐비트에 저장된 정수 값의 선형 함수로 회전 각도가 결정되는 제어된 회전을 구현합니다.

#### 2. 수학적 정의
이 회로는 대상 큐비트에 회전 $R_{basis}(\theta)$를 적용하며, 각도 $\theta$는 다음과 같이 계산됩니다.

$\theta = \text{기울기} \times x + \text{오프셋}$

* **$x$**: 상태(제어) 큐비트가 나타내는 정수 값
* **기울기**: 정수 단계당 각도의 변화율.
* **오프셋**: $x = 0$일 때의 시작 각도.
* **기준축**: 회전축(X, Y 또는 $Z$).

#### 3. 스크립트 구현

`quantum_example.py`에서 회로는 다음과 같은 구성으로 인스턴스화됩니다.

* **기준축:** `'Y'` ($R_y$ 회전을 구현합니다.)
* **상태 큐비트:** 2개 (입력 $x$ 역할을 하며, $0, 1, 2, 3$의 정수 값을 허용합니다.)
* **방정식:** $\theta = 0.5x + 0.2$.

```python
lpr_circuit = LinearPauliRotations(
num_state_qubits=2,
slope=0.5,
offset=0.2,
basis='Y'
)
```

#### 4. 구성 요소 분류
분류 목록에 나와 있듯이 이 클래스는 `Gate`가 아닌 `QuantumCircuit`에서 파생됩니다.

* **중요한 이유:** `QuantumCircuit`이기 때문에 CNOT 및 단일 큐비트 회전과 같은 표준 기본 게이트로 구성된 복합 객체입니다.
* **사용법:** 스크립트에서 이 객체는 블록으로 취급되어 `.append()` 메서드를 사용하여 메인 회로에 추가됩니다.

```python
# qc.append(circuit_object, qubit_indices)
qc.append(lpr_circuit, range(num_state_qubits + num_target_qubits))
```

#### 5. 회로 구조
스크립트에서 `lpr_circuit.decompose().draw()`를 호출하면 내부 구조가 나타납니다.
1. **입력:** "상태" 큐비트 레지스터와 "대상" 큐비트 레지스터입니다.
2. **연산:** 일련의 제어된 회전(종종 CNOT 및 단일 큐비트 회전으로 분해됨)을 사용하여 선형 공식에 따라 각 상태 큐비트의 회전 기여도를 효과적으로 "합산"합니다.

```python
try:
    from qiskit import QuantumCircuit
    from qiskit.circuit.library import LinearPauliRotations
except ImportError:
    print("Error: Qiskit is not installed. Please install it using 'pip install qiskit'.")
    exit(1)

def example_linear_pauli_rotations():
    """
    Demonstrates the LinearPauliRotations circuit.
    
    LinearPauliRotations implements a rotation where the angle is a linear function 
    of the state of a control register.
    
    Rotation angle: theta = slope * x + offset
    where x is the integer value of the control register.
    """
    
    # Parameters for the linear function
    slope = 0.5
    offset = 0.2
    
    # Define the number of state qubits (control) and target qubits
    num_state_qubits = 2
    num_target_qubits = 1
    
    # Create the LinearPauliRotations circuit
    # basis='Y' means we are doing Ry rotations
    lpr_circuit = LinearPauliRotations(
        num_state_qubits=num_state_qubits,
        slope=slope,
        offset=offset,
        basis='Y'
    )
    
    print("--- LinearPauliRotations Circuit ---")
    print(lpr_circuit.draw())
    
    # To see the implementation details (decomposition)
    print("\n--- Decomposed Circuit ---")
    print(lpr_circuit.decompose().draw())

    # Example of integrating into a larger circuit
    print("\n--- Integrated into larger circuit ---")
    qc = QuantumCircuit(num_state_qubits + num_target_qubits)
    
    # Initialize state qubits (e.g., to superposition)
    qc.h(range(num_state_qubits))
    
    # Append the LinearPauliRotations circuit
    # The first 'num_state_qubits' are controls, the last 'num_target_qubits' are targets
    qc.append(lpr_circuit, range(num_state_qubits + num_target_qubits))
    
    print(qc.draw())

if __name__ == "__main__":
    try:
        example_linear_pauli_rotations()
    except ImportError:
        print("Error: Qiskit is not installed. Please install it using 'pip install qiskit'.")
```

### 1. Advanced Arithmetic

**고급 빌딩 블록: 기타 산술 함수**

Qiskit의 회로 라이브러리에는 다양한 전문 산술 구성 요소가 포함되어 있습니다. 이러한 게이트와 회로는 양자 알고리즘을 구축하는 데 필수적이며, 두 가지 형태로 존재합니다: 수학적 연산을 설명하여 transpiler가 최적화할 수 있는 추상 `Gate` 표현과, 즉시 사용할 수 있는 알고리즘 블록인 구체적인 `QuantumCircuit` 구현입니다.

**Gate 표현**

- **ExactReciprocalGate**: 정확한 역수 함수 `f(x) = 1/x` 구현
- **IntegerComparatorGate**: 큐비트 레지스터와 고전 정수 간 ≥ 또는 < 비교 수행
- **QuadraticFormGate**: 큐비트 레지스터에 인코딩된 이진 변수에 대한 2차 형식 구현
- **WeightedSumGate**: 가중 합 `Σ(wᵢ * xᵢ)`을 계산하는 게이트 (각 변수에 특정 가중치 곱)

**QuantumCircuit 표현**

- **ExactReciprocal**: 정확한 역수 `QuantumCircuit` 구현
- **IntegerComparator**: 정수 비교기 `QuantumCircuit` 구현
- **QuadraticForm**: 2차 형식 `QuantumCircuit` 구현
- **WeightedAdder**: 가중 합 `QuantumCircuit` 구현

```python
from qiskit.circuit.library import IntegerComparator

# Example 1: Integer Comparator as a Gate
# 2-qubit 숫자(0-3)가 3보다 작은지 비교
qc = IntegerComparator(2, 3, geq=False)

# qc는 2개의 비교 대상 큐비트 + 1개의 결과 큐비트 (총 3큐비트)로 구성된 게이트
qc.add_register(ClassicalRegister(1))
# 결과는 세 번째 큐비트에 저장 (조건이 참이면 플립)
# 0 < 3 => True, 세 번째 큐비트가 플립됨
qc.measure(2,0)
print("Integer Comparator Circuit:")
qc.draw('mpl')
```

### 2. Particular Quantum Circuits

특정 양자 알고리즘 및 응용 프로그램을 위한 특수 회로입니다. 이러한 기능은 `Gate` 및 `QuantumCircuit` 표현으로도 제공됩니다.

- **fourier_checking**: 푸리에 검사 알고리즘 회로
- **hidden_linear_function**: 숨겨진 선형 함수 문제를 해결하는 회로
- **iqp**: 순간 양자 다항 시간(IQP) 회로
- **random_iqp**: 무작위 순간 양자 다항 시간(IQP) 회로
- **quantum_volume**: 양자 부피 측정에 사용되는 양자 부피 모델 회로
- **phase_estimation**: 유니터리 연산자 U의 고유값 $e^{2\pi i\phi}$의 위상 `ϕ`를 추정하는 위상 추정 회로입니다.
- **grover_operator**: 그로버 탐색 알고리즘의 그로버 연산자를 구성합니다.
- **unitary_overlap**: 두 유니터리 연산자 U2†U1의 중첩을 반환하는 회로입니다.
- **GraphStateGate**: 그래프 상태를 나타내는 게이트입니다.
- **PauliEvolutionGate**: 파울리 게이트로 구성된 연산자의 시간 진화를 구현합니다.
- **HamiltonianGate**: 해밀턴 연산자의 진화를 게이트로 표현하는 클래스입니다.

### 1. 기저 인코딩 (Basis Encoding)

모든 특성(feature)이 4비트로 저장된다고 가정합니다.

5 → 이진수 0101

7 → 이진수 0111

0 → 이진수 0000

이 비트 문자열들은 세 개의 4큐비트 집합에 할당되므로, 전체 12큐비트 기저 상태는 $\|0101 0111 0000\rangle$ 이 됩니다. 여기서 처음 4개의 큐비트는 첫 번째 특성을, 다음 4개의 큐비트는 두 번째 특성을, 마지막 4개의 큐비트는 세 번째 특성을 나타냅니다. 아래 코드는 데이터 벡터 (5, 7, 0)을 양자 상태로 변환하며, 다른 한 자리 특성들에도 일반화하여 적용할 수 있습니다.

```python
from qiskit import QuantumCircuit

# 인코딩할 데이터 포인트
x = 5  # 이진수: 0101
y = 7  # 이진수: 0111
z = 0  # 이진수: 0000

# 각각을 4비트 이진 리스트로 변환
x_bits = [int(b) for b in format(x, "04b")]  # [0,1,0,1]
y_bits = [int(b) for b in format(y, "04b")]  # [0,1,1,1]
z_bits = [int(b) for b in format(z, "04b")]  # [0,0,0,0]

# 모든 비트를 합침
all_bits = x_bits + y_bits + z_bits  # [0,1,0,1,0,1,1,1,0,0,0,0]

# 12큐비트 양자 회로 초기화
qc = QuantumCircuit(12)

# 비트가 1인 위치에 x 게이트 적용
for idx, bit in enumerate(all_bits):
    if bit == 1:
        qc.x(idx)

qc.draw("mpl")
```

### 2. 진폭 인코딩 (Amplitude Encoding)

진폭 인코딩에서는 데이터 벡터의 각 특성값이 서로 다른 양자 상태의 진폭으로 저장됩니다. 예시로, 예제 데이터셋 $\text{X}_\text{ex}$의 첫 번째 벡터 $\vec{x}^{(1)} = (4,8,5)$를 진폭 인코딩을 사용하여 인코딩해 보겠습니다. 결과 벡터를 정규화하면 다음과 같습니다:

$\sum_{i=1}^N \left\|x^{(1)}_i\right\|^2$ = $4^2+8^2+5^2$ = 105 = $\left\|\alpha\right\|^2 \rightarrow \alpha = \sqrt{105}$

그리고 결과 2큐비트 양자 상태는 다음과 같습니다:

$
\|\psi(\vec{x}^{(1)})\rangle = \frac{1}{\sqrt{105}}(4\|00\rangle+8\|01\rangle+5\|10\rangle+0\|11\rangle)
$

위 예시에서 벡터의 특성 수 $N=3$은 2의 거듭제곱이 아닙니다. $N$이 2의 거듭제곱이 아닌 경우, $2^n\geq N$을 만족하는 큐비트 수 $n$을 선택하고 진폭 벡터를 정보가 없는 상수(여기서는 0)로 채워줍니다. 따라서 $N$개의 특성에 대한 진폭 인코딩에는 $n \ge \mathrm{log}_2(N)$개의 큐비트가 필요합니다.

Qiskit에서는 `initialize` 메서드를 사용하여 진폭 인코딩을 쉽게 구현할 수 있습니다:

```python
import math

desired_state = [
    1 / math.sqrt(105) * 4,
    1 / math.sqrt(105) * 8,
    1 / math.sqrt(105) * 5,
    1 / math.sqrt(105) * 0,
]

qc = QuantumCircuit(2)
qc.initialize(desired_state, [0, 1])
#큐비트 0, 1을 |00⟩ 상태에서 시작하는 게 아니라, 위에서 정의한 상태로 세팅

qc.draw(output="mpl")
```

### 3. 각도 인코딩 (Angle Encoding)

각도 인코딩에서:

*   각 특성값은 해당 큐비트 $Q_k$에 매핑됩니다: $\vec{x}^{(j)}_k \rightarrow Q_k$. 큐비트들은 곱 상태로 유지됩니다.
*   한 번에 하나의 수치값이 인코딩됩니다 (데이터 포인트의 전체 특성 집합이 아니라).
*   $N$개의 데이터 특성에 $n$개의 큐비트가 필요합니다($n\leq N$). 대부분의 경우 등호가 성립합니다. $n<N$이 가능한 경우는 다음 몇 섹션에서 살펴보겠습니다.
*   결과 회로는 일정한 깊이(depth)를 가집니다 (전치(transpilation) 전 깊이는 일반적으로 1).

각도 인코딩은 임의의 회전 게이트를 사용하여 구현할 수 있습니다. Y축 회전을 사용하여 데이터를 인코딩하면 실수값 양자 상태가 생성된다는 추가적인 특징이 있으며, 이는 특정 응용에 유용할 수 있습니다. 데이터는 $\vec{x}^{(j)}_k \in (0,2\pi]$ 범위로 재조정되어야 정보 손실 및 기타 원치 않는 효과를 방지할 수 있습니다.

아래 Qiskit 코드는 초기 상태 $\|0\rangle$에서 단일 큐비트를 회전시켜 데이터 값 $\vec{x}^{(j)}_k=\frac{1}{2}\pi$를 인코딩합니다.

```python
from qiskit.quantum_info import Statevector
from math import pi

qc = QuantumCircuit(2)
state1 = Statevector.from_instruction(qc)
qc.ry(1, 0)  #0번 큐비트를 y축 기준으로 1 radian 만큼 회전
qc.ry(-1.2, 1)  #1번 큐비트를 -1.2 radian 만큼 회전
qc.draw("mpl")
```

수학적으로:
$R_y(\theta)\|0\rangle = \cos\left(\frac{\theta}{2}\right)\|0\rangle + \sin\left(\frac{\theta}{2}\right)\|1\rangle$

$j$번째 데이터 벡터 $\vec{x}^{(j)} = (x_1,...,x_N)$의 $N$개 특성을 $n$개의 큐비트 회전 각도로 인코딩할 때, 인코딩된 곱 상태는 다음과 같습니다:

$\|\vec{x}^{(j)}\rangle$ = $$\bigotimes^N_{k=1}$$ $R_Y(2\vec{x}^{(j)}_k)\|0\rangle$ = $$\bigotimes^N_{k=1}$$ $\cos(\vec{x}^{(j)}_k)\|0\rangle + \sin(\vec{x}^{(j)}_k)\|1\rangle$



### 4. 위상 인코딩 (Phase Encoding)

위상 인코딩 절차는 각 특성값을 해당 큐비트의 위상 $\phi$에 매핑합니다: $\vec{x}^{(j)}_k \rightarrow Q_k$. 전체적으로 위상 인코딩은 아다마르(Hadamard) 레이어를 포함하여 회로 깊이가 2이므로, 효율적인 인코딩 방식입니다. 위상 인코딩된 다중 큐비트 상태($N=n$개의 특성에 $n$개의 큐비트)는 곱 상태입니다:

$\|\vec{x}^{(j)}\rangle$ = $$\bigotimes_{k=1}^{N}$$ $P_k(\phi = \vec{x}^{(j)}_k)\|+\rangle^{\otimes N}$ = ${\textstyle\frac{1}{\sqrt{2^N}}}$ $$\bigotimes_{k=1}^{N}$$ $\big(\|0\rangle + e^{i\vec{x}^{(j)}_k}\|1\rangle\big).$

아래 Qiskit 코드는 먼저 아다마르 게이트로 단일 큐비트의 초기 상태를 준비한 다음, 위상 게이트를 사용하여 데이터 포인트 $\vec{x}^{(j)}_k=\frac{1}{2}\pi$를 위상 인코딩합니다.

```python
from qiskit.quantum_info import Statevector
from math import pi
qc = QuantumCircuit(1)
qc.h(0)  # 아다마르 게이트로 상태를 블로흐 적도로 회전
state1 = Statevector.from_instruction(qc)

qc.p(pi / 2, 0)  # 위상 게이트로 pi/2 각도 회전
state2 = Statevector.from_instruction(qc)
#Statevector는 "0일 확률 + 1일 확률 + 위상 정보"를 알려줌(α,β)

states = state1, state2
print(f"state1:{state1}, state2:{state2}")

qc.draw("mpl", scale=1)
```

### 5. 고밀도 각도 인코딩 (Dense Angle Encoding)

고밀도 각도 인코딩(DAE)은 각도 인코딩과 위상 인코딩의 조합입니다. DAE는 두 개의 특성값을 단일 큐비트에 인코딩할 수 있습니다: $\theta$는 Y축 회전으로, $\phi$는 Z축 회전으로. 두 개의 데이터 특성을 하나의 큐비트에 인코딩하면 인코딩에 필요한 큐비트 수가 2배 감소합니다. 더 많은 특성으로 확장하면, 데이터 벡터 $\vec{x} = (x_1,...,x_N)$을 다음과 같이 인코딩할 수 있습니다:

$\|\vec{x}\rangle$ = $$\bigotimes_{k=1}^{N/2}$$ $\cos(\frac{x_{2k-1}}{2})\|0\rangle + e^{i x_{2k}}\sin(\frac{x_{2k-1}}{2})\|1\rangle$

DAE는 여기서 사용된 삼각함수 대신 두 특성의 임의 함수로 일반화할 수 있습니다. 이를 **일반 큐비트 인코딩(general qubit encoding)**이라고 합니다.

DAE의 예시로, 아래 코드는 특성 $x_1=\theta = 3\pi/8$과 $x_2=\phi = 7\pi/4$의 인코딩을 시각화합니다.

```python
qc = QuantumCircuit(1)
state1 = Statevector.from_instruction(qc)
qc.ry(3 * pi / 8, 0)
state2 = Statevector.from_instruction(qc)
qc.rz(7 * pi / 4, 0)
state3 = Statevector.from_instruction(qc)
states = state1, state2, state3

qc.draw("mpl", scale=1)
```

### 6. Efficient SU2

상태벡터의 진폭은 복소수이므로, 단일 큐비트 상태는 4개의 실수 매개변수를 가지며(정규화 조건에 의해 제약), 2큐비트 상태는 8개, 그 이상도 마찬가지입니다. Efficient SU2 방식은 더 적은 큐비트로 훨씬 더 많은 데이터 포인트를 인코딩할 수 있습니다. 예를 들어, 반복(repetition)이 1인 2큐비트 상태는 다음과 같습니다:<br>
<p>
$$
\begin{aligned}
|\psi\rangle_{\text{final}} &= \cos(\theta_0/2)
\left(\cos(\theta_4/2)|0\rangle+\sin(\theta_4/2)e^{i\theta_6}|1\rangle\right)\\
&\quad\otimes \left[ \cos(\theta_1/2)
    \left( \cos(\theta_5/2)|0\rangle+\sin(\theta_5/2)e^{i \theta_7}|1\rangle\right)
+\sin(\theta_1/2)e^{i\theta_3}\left(-\sin(\theta_5/2)|0\rangle+\cos(\theta_5/2)e^{i \theta_7}|1\rangle\right)
    \right]\\
&\quad+\sin(\theta_0/2)e^{i\theta_2}
\left(-\sin(\theta_4/2)|0\rangle+\cos(\theta_4/2)e^{i\theta_6}|1\rangle\right)\\
&\quad\otimes \left[
\sin(\theta_1/2)e^{i\theta_3}\left(\cos(\theta_5/2)|0\rangle+\sin(\theta_5/2)e^{i \theta_7}|1\rangle\right)
+\cos(\theta_1/2)e^{i\theta_3}\left(-\sin(\theta_5/2)|0\rangle+\cos(\theta_5/2)e^{i \theta_7}|1\rangle\right)
\right]
\end{aligned}
$$
</p>

```python
from qiskit.circuit.library import efficient_su2

circuit = efficient_su2(num_qubits=2, reps=3, insert_barriers=True)
circuit.draw(output="mpl")
```

단, 2개의 큐비트로 더 많은 매개변수를 인코딩할 수 있지만, 상태가 더 이상 유일하지 않습니다.

### 7. Z 피처 맵 (Z Feature Map)

$Z$ 피처 맵(ZFM)은 위상 인코딩의 자연스러운 확장으로 해석할 수 있습니다. ZFM은 아다마르 게이트와 위상 게이트가 번갈아 나오는 레이어로 구성됩니다. 데이터 벡터 $\vec{x}$가 $N$개의 특성을 가진다고 할 때, 데이터 특성 $x_i$는 해당 큐비트와 1:1 매핑됩니다. 단일 반복의 ZFM 전체 회로 유니타리는 다음과 같습니다:
<p>
$$
\mathscr{U}_{\text{ZFM}}=\big(P(\vec{x}_1)\otimes\ldots P(\vec{x}_k)\otimes\ldots P(\vec{x}_N)H^{\otimes N}\big)=\left(\bigotimes_{k = 1}^N P(\vec{x}_k)\right)H^{\otimes N}
$$
</p>

$r$번의 반복은 다음과 같습니다:
<p>
$$
\mathscr{U}^{(r)}_{\text{ZFM}}\left(\vec{x}\right)=\prod_{s=1}^{r}\left[\left(\bigotimes_{k = 1}^N P(\vec{x}_k)\right)H^{\otimes N}\right]
$$
</p>

데이터 특성 $x_k$는 모든 $r$번의 반복에서 동일한 방식으로 위상 게이트에 매핑됩니다.

아래 Qiskit 코드는 아다마르 게이트와 위상 게이트를 명시적으로 사용하여 ZFM의 구조를 보여줍니다:

```python
qc0 = QuantumCircuit(1)
qc1 = QuantumCircuit(1)

qc0.h(0)
qc0.p(pi / 2, 0)

qc1.h(0)
qc1.p(pi / 3, 0)

# 회로 qc0와 qc1을 하나의 회로로 결합
qc = QuantumCircuit(2)
qc.compose(qc0, [0], inplace=True) # qc0의 qubit 0을 qc의 qubit 0에 연결
qc.compose(qc1, [1], inplace=True)

qc.draw("mpl", scale=1)
```

### 8. ZZ 피처 맵 (ZZ Feature Map)

$ZZ$ 피처 맵(ZZFM)은 2큐비트 얽힘 게이트, 특히 $ZZ$-회전 게이트 $R_{ZZ}(\theta)$를 포함시켜 ZFM을 확장한 것입니다. ZZFM은 ZFM과 달리 고전 컴퓨터에서 일반적으로 계산하기 어렵다고 추측됩니다.

ZZFM 회로의 단일 반복은 다음과 같습니다:
<p>
$$
\mathscr{U}_{\text{ZZFM}}(\vec{x}) = R_{ZZ,q,p}(\theta_{q,p})(\vec{x})\left(\bigotimes_{k = 1}^N P(\vec{x}_k)\right)H^{\otimes N},
$$
</p>

얽힘 방식은 선형(linear), 원형(circular), 완전(full) 등이 있습니다. 위상 레이어에서 큐비트 $p$와 $q$에는 각각 $\vec{x}_p$와 $\vec{x}_q$를 인코딩하는 위상 게이트가 있습니다. 기본적으로:

<p>
$$
\theta_{q,p} \rightarrow \phi(\vec{x}_q, \vec{x}_p) = 2(\pi-\vec{x}_q)(\pi-\vec{x}_p).
$$
</p>

하지만 자체 $\phi$를 정의할 수도 있습니다. 또한 $R_{ZZ}$를 다른 다중 큐비트 유니타리로 교체할 수도 있습니다.

```python
from qiskit.circuit.library import zz_feature_map

feature_dim = 2
zzfeature_map = zz_feature_map(
    feature_dimension=feature_dim, entanglement="linear", reps=1
)
zzfeature_map.draw("mpl", scale=1)
```

```python
feature_dim = 4
zzfeature_map = zz_feature_map(
    feature_dimension=feature_dim, entanglement="circular", reps=1
)
zzfeature_map.draw("mpl", scale=1)
```

### 9. Pauli 피처 맵 (Pauli Feature Map)

Pauli 피처 맵(PFM)은 ZFM과 ZZFM을 임의의 Pauli 게이트를 사용하도록 일반화한 것입니다. Pauli 피처 맵은 이전 두 피처 맵과 매우 유사한 형태를 가집니다. 데이터 벡터 $\vec{x}$의 $N$개 특성에 대해 $r$번의 인코딩 반복을 수행할 때:

<p>
$$
\mathscr{U}_{\text{PFM}}(\vec{x}) = \prod_{s=1}^{r} U(\vec{x}) H^{\otimes n}.
$$
</p>

PFM에서 $U(\vec{x})$는 Pauli 전개 유니타리 연산자로 일반화됩니다. 여기서 지금까지 고려한 피처 맵의 보다 일반화된 형태를 제시합니다:
<p>
$$
U(\bar{x}) = \exp\left(i \alpha \sum_{S\subseteq[n]} \phi_S(\bar{x}) \prod_{i \in S} \sigma_i \right)
$$
</p>

여기서 $\sigma_i$는 Pauli 연산자 $\sigma_i \in \{I,X,Y,Z\}$입니다. $\mathcal{I}$는 단일 큐비트 게이트가 작용하는 큐비트 집합을 포함하여, 피처 맵에 의해 결정된 모든 큐비트 연결성의 집합입니다. 그리고:
<p>
$$
\phi_S(\vec{x})=
    \begin{cases}
      x_i & \text{if } S= \{i\} \text{ (단일 큐비트)}\\
      \prod_{j\in{S}}(\pi-x_j) & \text{if } |S|\ge2 \text{ (다중 큐비트)}\\
    \end{cases}
$$
</p>

$\alpha$의 기본값은 $2$입니다. 예를 들어 $[0,4]$ 구간에서 그 값을 최적화함으로써, 양자 커널을 데이터에 더 잘 맞출 수 있습니다.

```python
from qiskit.circuit.library import pauli_feature_map
 
feature_dim = 3
pfmap = pauli_feature_map(
    feature_dimension=feature_dim, entanglement="linear", reps=1, paulis=["Y", "XX"]
)
 
pfmap.draw("mpl", scale=1.5)
```

### 10. Two Local

Two Local은 번갈아 나타나는 회전(rotation) 레이어와 얽힘(entanglement) 레이어로 구성된 매개변수화 회로를 생성합니다. 두 레이어 모두 매개변수화된 회로 블록이 정해진 방식으로 회로에 작용합니다. 회전 레이어에서 블록들은 서로 위에 쌓인 형태로 적용되고, 얽힘 레이어에서는 얽힘 전략에 따라 적용됩니다. 각 레이어는 여러 번 반복되며, 기본적으로 마지막에 최종 회전 레이어가 추가됩니다.

```python
from qiskit.circuit.library import TwoLocal

twolocal = TwoLocal(num_qubits=5, reps=1, rotation_blocks=['ry','rz'],

twolocal.decompose().draw('mpl')
```

### 2. 회로 구성 메서드 (Circuit Construction Methods)

#### 2. compose

이 회로를 다른 회로와 결합합니다. `append`와 달리 `compose`는 다른 회로를 하나의 불투명한 명령으로 추가하는 대신, 이 회로에 병합합니다.


#### 3. Append VS. Compose

`compose()`는 두 번째 회로를 첫 번째 회로에 병합하여 게이트들이 일렬로 나열되도록 합니다. `append()`는 다른 회로를 하나의 불투명한 명령(박스로 둘러싸인 게이트, 단일 게이트는 제외)으로 추가합니다. 시각적으로는 한쪽은 확장된 형태로, 다른 쪽은 하나의 박스로 표시됩니다. 아래 예시는 동일한 작은 회로에 `append()`를 적용한 결과를 보여주므로 두 표현을 비교해 볼 수 있습니다.

```python
# Demonstrate append() so it appears as an opaque instruction
qc1 = QuantumCircuit(1)
qc1.h(0)
qc2 = QuantumCircuit(1)
qc2.x(0)
# Make a copy so we don't mutate the previous example
qc_app = qc1.copy()
# Convert qc2 into a single gate/instruction and append it
qc_app.append(qc2.to_gate(), [0])
qc_app.draw('mpl')
```

#### 4. store

고전적인 값을 고전적인 비트에 저장합니다. 이는 Qiskit의 동적 회로 기능 중 하나입니다.

```python
# Import the necessary components from Qiskit
import qiskit
from qiskit import QuantumCircuit
from qiskit.circuit.classical import types, expr

# --- The Correct Way to use qc.store in Modern Qiskit ---

# 1. Create a circuit with classical bits
qc = QuantumCircuit(1, 1)
target_clbit = qc.clbits[0]

# 2. Define the classical value you want to store.
# We use expr.lift() to create a classical literal from a Python value and a Qiskit type.
# For example, let's store the integer value 1 into an 8-bit unsigned integer type.
uint_value_to_store = expr.lift(1, types.Uint(8))

# 3. Use the store instruction within a try...except block for safety
try:
    qc.store(target_clbit, uint_value_to_store)
    print("Successfully added a 'store' instruction to the circuit.")
    # Draw the circuit to see the 'Store' instruction
    display(qc.draw('mpl'))
except Exception as e:
    print(f"The 'store' instruction failed: {e}")

# --- Environment Info ---
print("\n--- Environment Info ---")
# The correct way to get the version in modern Qiskit
print('qiskit version:', qiskit.__version__)

# Inspect the circuit data to confirm the instruction was added
if qc.data:
    print('Last instruction in circuit data:', qc.data[-1])
else:
    print('No instructions in the circuit.')
```

#### 1. estimate_duration

#### 2. qubit_duration

특정 큐비트에 대한 연산의 지속 시간을 반환합니다.

```python

backend = GenericBackendV2(num_qubits=2)

pm = generate_preset_pass_manager(target=backend.target, optimization_level=1)
```

```python
# More illustrative scheduling demo: build a small circuit with gates and delays,
# then schedule it with a fake backend so qubit_duration/start/stop are non-zero.
from qiskit.providers.fake_provider import GenericBackendV2
from qiskit.transpiler import generate_preset_pass_manager
from qiskit import QuantumCircuit

backend = GenericBackendV2(num_qubits=3)
```

#### 3. qubit_start_time 및 qubit_stop_time

이 메서드들은 특정 큐비트에 대한 연산의 예약된 시작 시간과 종료 시간을 반환합니다.

중요한 점은, 이 메서드들은 회로가 'alap' 또는 'asap'과 같은 scheduling_method를 사용하여 트랜스파일링된 경우에만 0이 아닌 값을 반환한다는 것입니다. 이전 섹션의 qc_scheduled 객체는 스케줄링 과정 없이 생성되었으므로, 아래와 같이 타이밍 정보가 기본값인 0으로 유지됩니다. 이는 스케줄링 과정을 거쳐 유효한 타이밍 데이터를 포함하는 qc_demo_scheduled 객체와는 대조적입니다.

참고로, qc_scheduled 객체를 조회하면 시작 시간과 종료 시간이 여전히 0으로 표시됩니다. 타이밍 정보는 scheduling_method가 지정된 경우 트랜스파일링 결과로 반환되는 새 객체에만 존재하기 때문입니다.

```python
# Check start and stop times for Qubit 0
print(f"Qubit 0 start time: {qc_scheduled.qubit_start_time(0)}")
print(f"Qubit 0 stop time: {qc_scheduled.qubit_stop_time(0)}")
```

### 13. 명령어 수정자(Instruction Modifiers)

수정된 속성(예: 제어 또는 반전)을 가진 새로운 회로 버전을 반환하는 메서드입니다.

#### 1. `control()` 메서드

`.control()` 메서드는 기존 회로 또는 게이트를 입력받아 하나 이상의 제어 큐비트를 포함하는 새로운 버전을 반환하는 강력한 도구입니다. 이를 통해 복잡한 연산을 수동으로 분해하지 않고도 제어된 버전으로 쉽게 만들 수 있습니다.

아래 예제에서는 단일 하드마르 게이트를 포함하는 간단한 1큐비트 회로에서 시작합니다. 그런 다음 `.control(1)`을 사용하여 새로운 2큐비트 제어 하드마르 게이트를 생성합니다. 출력 다이어그램에서 원래의 `h` 게이트가 이제 `q0`의 제어점이 대상 `q1`에 작용하는 단일 블록으로 표현되는 것을 확인할 수 있습니다.

```python
from qiskit import QuantumCircuit

# Create a simple circuit for a single H-gate
qc_h = QuantumCircuit(1, name='H-Gate')
qc_h.h(0)

# Add one control qubit to the H-gate circuit
# The original qubit (0) becomes the target (now on wire 1)
# The new control is on wire 0
controlled_h_gate = qc_h.control(1)

# You can now use this new gate in another circuit
main_qc = QuantumCircuit(2)
main_qc.append(controlled_h_gate, [0, 1])
main_qc.draw('mpl')
```

#### 2. `inverse()` 메서드

`.inverse()` 메서드는 회로의 수학적 역행렬(켤레 전치, 또는 $U^\dagger$)을 계산합니다. 이 메서드를 사용하면 모든 게이트가 반대 순서로 적용되고 각 게이트가 자신의 역행렬로 대체된 새로운 회로가 반환됩니다.

이는 진폭 증폭이나 상태 역산과 같은 여러 알고리즘에 매우 유용합니다.

아래 예시에서, 반전된 회로에서 `Sdg`(S-dagger) 게이트와 `H` 게이트가 원래 회로와 반대 순서로 적용된 것을 확인할 수 있습니다.

```python
# Create a circuit with a non-symmetrical sequence of gates
qc = QuantumCircuit(1)
qc.h(0)
qc.s(0)

print("Original Circuit:")
display(qc.draw('mpl'))

# Now, compute the inverse
qc_inv = qc.inverse()

print("\nInverted Circuit:")
display(qc_inv.draw('mpl'))
```

#### 3. `power()` 메서드

`.power()` 메서드를 사용하면 게이트 또는 회로를 특정 거듭제곱으로 계산할 수 있습니다. 이는 게이트의 유니터리 행렬 $U$를 자신에게 *n*번 적용하는 것과 같습니다($U^n$).

이 메서드는 특히 소수 거듭제곱을 계산하거나 알려진 게이트의 특성을 보여줄 때 유용합니다. 예를 들어, T 게이트를 두 번 적용하는 것은 S 게이트를 한 번 적용하는 것과 같다는 것을 알고 있습니다($T^2 = S$). 아래 코드는 T 게이트를 생성하고 이를 2제곱하면 결과 회로도가 S 게이트를 정확하게 보여줍니다.

```python
from qiskit.circuit.library import TGate

# Create a circuit with a single T-gate
qc = QuantumCircuit(1)
qc.append(TGate().power(2), [0])

# The output diagram will show an S-gate (Phase Gate where theta=π/2), because T^2 = S
qc.draw('mpl')
```

#### 4. `reverse_ops()` 메서드

`.reverse_ops()` 메서드는 회로의 명령어 순서를 반전시킵니다. 중요한 점은 이 메서드가 **단지 순서만 반전시킬 뿐** 각 게이트의 역문을 취하지 않는다는 것입니다(`.inverse()` 메서드와는 다릅니다).

이 메서드는 상태 준비 회로를 만들거나 특정 알고리즘 단계를 수동으로 구성할 때 유용합니다. 아래 그림에서 원래 회로와 반전된 회로를 비교하여 그 효과를 확인해 보세요.

```python
# Create a circuit with a clear sequence of different gates
qc = QuantumCircuit(1)
qc.h(0)
qc.x(0)
qc.z(0)

print("Original Circuit:")
display(qc.draw('mpl'))

# Now, reverse the operations
qc_rev = qc.reverse_ops()

print("\nReversed Circuit:")
display(qc_rev.draw('mpl'))
```

### 14. 시각화: `reverse_bits`를 이용한 비트 순서 관리

`.reverse_bits()` 메서드는 회로 내 큐비트의 순서를 반전시킵니다. 이 메서드는 Qiskit의 비트 순서 규칙(흔히 엔디안이라고 함)을 관리하는 데 매우 중요한 도우미 메서드입니다.

* **Qiskit 기본값(리틀 엔디안):** Qiskit은 기본적으로 리틀 엔디안 순서를 사용합니다. 이는 양자 상태를 이진 문자열로 표현할 때 인덱스 0의 큐비트(`q_0`)가 최하위 비트(LSB)임을 의미합니다. 예를 들어, 상태 $\|101\rangle$은 `q_2=1`, `q_1=0`, `q_0=1`로 표현됩니다.

* **교과서적 규칙(빅 엔디안):** 하지만 많은 양자 알고리즘, 교과서 및 학술 논문에서는 빅 엔디안 규칙을 사용하며, 이 경우 `q_0`이 최상위 비트(MSB)입니다. 이 규칙에서 $\|101\rangle$은 `q_0=1`, `q_1=0`, `q_2=1`로 표현됩니다.

`reverse_bits()` 메서드는 회로를 재구성하지 않고도 시각화 또는 특정 연산을 수행하기 전에 이 두 가지 규칙 사이를 간단하게 전환할 수 있는 방법을 제공합니다.

아래 예시에서는 3큐비트 회로를 생성하고 `q_0`에 `X` 게이트를, `q_2`에 `Z` 게이트를 적용합니다. 규칙을 반전시킨 후, 새로운 회로도에서는 비트 객체의 내부 순서가 바뀌었기 때문에 `X` 게이트가 `q_2` 와이어에, `Z` 게이트가 `q_0` 와이어에 나타나는 것을 확인할 수 있습니다.

```python
# Create a 3-qubit circuit to clearly show the reversal
qc = QuantumCircuit(3)

# Apply a distinctive gate to the first and last qubits
qc.x(0)
qc.z(2)

print("Original Circuit (Default Little-Endian Ordering):")
print(qc)

# Now, reverse the order of the bits
qc_reversed = qc.reverse_bits()

print("\nCircuit After reverse_bits() (Big-Endian Ordering):")
print(qc_reversed)
```
